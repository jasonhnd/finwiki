ERC-7702 vs ERC-4337 · Ethereum AA デュアルトラック対照

ERC-7702 vs ERC-4337 · Ethereum AA デュアルトラック対照

Wiki route

This entry sits under systems index. Read it against ERC-7702 概観 · EOA が一時的に SCW 機能を獲得する Pectra アップグレード for peer / contrast context and fintech index for the broader system / regulatory boundary.

Key facts

  • 4337 = 新規 SCW 作成(2023-03 メインネットでアプリ層有効化)
  • 7702 = 既存 EOA のアップグレード(2025-05 Pectra でプロトコル層有効化)
  • 両者は EntryPoint インフラを共有
  • 4337 の浸透率 < 5%(2023-2025)· 7702 は急速な普及を予想
  • 主要推進者が分かれる:4337 = Privy / Coinbase CDP / ウォレット供給業者 · 7702 = Vitalik / Ethereum コア開発者 / MetaMask

Mechanism / How it works

デュアルトラックの詳細対照:

観点ERC-4337ERC-7702
対象ユーザー新規ユーザー / fintech embedded wallet既存 EOA ユーザー / DeFi 老舗ウォレット
移行コスト新規アドレス + 資産移行ゼロコスト · 原地アップグレード
浸透速度遅い(ユーザーが能動的に SCW を作成する必要)速い(ワンクリックでアップグレード可能)
主要推進者Privy / Coinbase CDP / ウォレット供給業者Vitalik / Ethereum コア開発者 / MetaMask
プロトコル層の改変なし(純粋にアプリ層)あり(新規 tx type)
適用シナリオ加盟店が顧客に新規ウォレットを発行ウォレットアプリが既存ユーザーに体験をアップグレード
リスクモデルSCW 契約自体のセキュリティ監査delegation 対象契約 + EOA 秘密鍵の二重リスク

補完性の根本理由:Ethereum にはすでに数千万人の EOA ユーザーが存在し · 履歴資産/ENS/NFT/DeFi ポジションを軽々と移行することはできない —— これが 4337 経路の難点である。一方 fintech 経由の新規ユーザー(Stripe / PayPal / Revolut のエンドユーザー)は EOA を持ったことがなく · 4337 で直接 SCW を発行するのが最適経路となる。

実際のシナリオでの棲み分け:

  • Stripe 加盟店が新規ユーザーに wallet を発行 → 4337 SCW(Privy 経由)
  • MetaMask ユーザーが DeFi で batch approve+swap を行いたい → 7702 一時 delegation
  • 機関向け multi-sig → Safe(4337 Modular ベース)
  • DeFi 上級ユーザーが session key で trading bot に授権 → 4337 または 7702 + ERC-7715

Origin & evolution

2022-2024 年の間 · Ethereum AA 陣営内では「プロトコル層 vs アプリ層」の議論があった —— 4337 はアプリ層先行に賭け · 7702 は後からプロトコル層で補完。Vitalik が 2024-05 に 7702 を主推進したことは重要な転換点であり · Ethereum コア開発者が「プロトコル層 AA は長期的方向性である」と正式に支持したことを示す。

Pectra 2025-05 有効化後 · 2026-Q1-Q2 に主流ウォレットがデフォルトサポートし · 7702 の浸透速度は 4337 の歴史的カーブを大きく上回ると予想される(EOA ユーザーがアドレス移行を必要としないため)。
