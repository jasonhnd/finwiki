組込み型ウォレットによる Fintech の脱仲介化 · 四強構図

Wiki route

This entry sits under fintech index. Read it with 日本金融規制 — トークン・暗号資産・決済に関する法体系 for adjacent context and 日本 Stablecoin 法制度の三層構造（JPYC・USDC・Project Pax） for the broader system boundary.

[!info] TL;DR 組込み型ウォレット(Embedded wallet)は 2025-2026 fintech における逆方向の脱仲介化を担う重要なキャリアである —— Privy(Stripe が 2025.06 に ~$1.1B で買収)/ Coinbase CDP / Magic / Web3Auth の四強が fintech 既存顧客基盤を crypto エントリーポイントに変え、ユーザーに意識させずに保有/送金/署名を可能にする。プロトコル基盤は ERC-4337/7702/7715 三点セット。

Key facts

  • Stripe が 2025.06 に Privy を ~$1.1B で買収 · 同時期に Bridge を $1.1B で買収
  • Coinbase CDP 2024 ローンチ · onchainkit + smart-wallet 一体型 · Coinbase 1 億人超ユーザーへの流通
  • Magic 2018 創業 · メール magic link から出発 · のち MPC へ転換 · $99/mo + 従量課金
  • Web3Auth(旧 Torus Labs)はオープンソース + エンタープライズの二軌 · BNB Labs 投資
  • Privy 顧客:OpenSea · Friend.tech · Hyperliquid + Stripe 5M+ 加盟店という潜在ディストリビューション

「ユーザー無感」UX の五要素:

  • Onboarding: Google/Apple/Email ワンタップ · TSS キーはバックエンドで · seed なし
  • 署名: session key(7702/7715)で 1 回認可 · session 内ではポップアップなし
  • Gas: Paymaster による代理支払(4337)· USDC 支払いまたは完全 sponsor 化
  • Recovery: 複数 share(device + cloud + server)で分散
  • クロスチェーン: Hyperlane / CCIP がバックエンドで調整 · ユーザーは単一残高として確認

プロトコル基盤:ERC-4337(AA SCW)+ ERC-7702(EOA → SCW 一時アップグレード)+ ERC-7715(granular permissions)の三点セットを組み合わせると = ユーザー無感 + シームレスアップグレード + 細粒度認可。

Origin & evolution

2018-2022 = MetaMask 主導 · ユーザーが能動的にインストール · seed phrase 自己管理 · crypto-native 境界が明確。2023-2024 = Privy / CDP / Magic が b2b SaaS に切り替え · 顧客が dApp から fintech へ拡大。2025.06 = Stripe が同時に Privy + Bridge を買収 · fintech が正式に crypto エントリーポイントに組み込まれる象徴。2025.07 = AWS Bedrock AgentCore Payments が Privy をデフォルトに選定(詳細は Privy × AgentCore)· 組込み型ウォレットが crypto ツールから汎用決済インフラへ格上げ。
