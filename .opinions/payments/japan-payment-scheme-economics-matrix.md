# 主観内容の隔離 — Japan payment scheme economics matrix

> 出典エントリー: `payments/japan-payment-scheme-economics-matrix.md`

## Counterpoints

- "Code payment will replace card." Code-payment **transaction count** grows fast but **transaction value** is still dominated by card class (82.7% of cashless value vs 10.2% for code in 2025). High-ticket EC, travel, and B2C subscription default to card because of chargeback strength and credit float.
- "A2A is cheaper, so it will win." Bank Pay and J-Debit acceptance footprint remains narrow because they don't run a point program, don't extend credit, and have weaker consumer-dispute mechanics than card.
- "Prepaid e-money is legacy." Suica / PASMO retain dominant transit-zone position; WAON and nanaco retain dominant in-group retail position. They are not growth stories but the installed base is sticky.
- "Stablecoin / EPI will disrupt all four." Possibly — but as of 2026 there is no merchant-acceptance footprint or chargeback regime that makes stablecoin a retail payment scheme. It is closer to a settlement-rail innovation than a retail-scheme replacement.

## Open questions

- Will JFTC merchant-fee disclosure pressure extend from card class to code class, and what effect would forced fee transparency have on the wallet platforms' subsidy economics?
- Can A2A acceptance grow without a loyalty layer, or does the absence of a point program permanently cap its retail footprint?
- How will Cotra interoperability evolve — will it become a true cross-scheme A2A rail or remain a small-value P2P utility?
- Will stablecoin / EPI develop a defined chargeback / dispute regime, and if so, would that allow it to compete with card class on consumer protection grounds?
- How sustainable is platform-funded point economics in the code class once campaign budgets normalize?
