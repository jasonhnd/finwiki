# 国内机构托管三强格局 — Komainu / Ginco / Fireblocks Japan 比较

## 概述

国内机构托管市场沿 **技术（冷钱包 vs MPC）× 母公司（国内 vs 境外）** 两轴分化。与 VASP 自营托管（bitFlyer / Coincheck / bitbank 等）并行，B2B 供应商层（面向机构、其他 VASP、金融机构的基础设施）已成形，其核心为 Komainu / Ginco / Fireblocks Japan 三家。

## 三家公司简介

### Komainu Holdings（英国 Jersey）
- 2018 设立 · 野村 HD + CoinShares + Ledger 三方合资成立的 [[exchanges/jp-custody-komainu]]
- 专注机构及主权国家客户的 **冷存储 + 离线签名** 托管
- 客户包括阿联酋政府等，2024-10 收购 Propine 加速亚洲布局
- 董事会设有 [[exchanges/jp-exchange-laser-digital-japan]]（野村系）高管 + Blockstream 的双锚结构

### 株式会社 Ginco（国内独立系）
- 2017 设立 · 创始人 森川夢佑斗 · 2026-04 更换代表董事 → 坂根遼 [[exchanges/jp-custody-ginco]]
- 提供 **Ginco Enterprise Wallet（MPC）** + Ginco Node
- B2B SaaS · 服务 VASP / 银行 / 信托 / 加密资产交换业者
- 国内独立系、无资本背景，为国内 VASP 基础设施最大服务商

### Fireblocks Japan（境外供应商日本法人）
- 全球母公司 Fireblocks Inc.（总部纽约 · 2018 以色列起源）[[exchanges/jp-custody-fireblocks-japan]]
- **MPC 托管 + 策略引擎 + 资金管理工作流**
- 全球 2,400+ 机构客户 · 累计交易额 $10T+ · E 轮融资 $550M（估值 $8B）
- 日本东京办公室 5 名规模 · 支持三井物产 MDC（Zipangcoin RWA）+ SMBC/Ava Labs/TIS SC（2025-04）+ みんなの銀行 + CoinTrade 等

## 技术 / 商业模式比较

| 维度 | Komainu | Ginco | Fireblocks |
|---|---|---|---|
| 技术 | 冷存储 + 离线签名 | MPC SaaS + 节点服务 | MPC + 策略引擎 + 工作流 |
| 主要客户 | 机构·主权国家 | 国内 VASP / 金融机构 | 数千家机构（全球） |
| 收费模式 | 月费 / AUM 比例 | 许可费 / 按交易 | 订阅制 |
| 母公司 | 野村 HD 系（境外法人） | 国内独立系 | 境外供应商（美国纽约） |

## 监管定位

三家均 **未持有 FSA 加密资产交换业登记**（作为 B2B 基础设施无需持牌）· [[exchanges/fsa-vasp-registration-system|JVCEA 非加盟]] · 作为依赖客户方 VASP 牌照的供应商层运营。

## 竞争层（海外对比）

Anchorage / Coinbase Custody / BitGo / Fidelity Digital Assets — 均未进入国内市场，或通过合作形式进入（如 BitGo × 三井）。机构纯托管领域仍与 [[exchanges/jp-exchange-crypto-garage]] 等大和 / SBI 系证券型代币接入业务存在重叠。
