# ERC-7702 vs ERC-4337 · Ethereum AA 双轨对照

## Wiki route

本条目位于 [[systems/INDEX|systems index]] 之下。请结合 [[systems/erc-7702-overview|ERC-7702 概観 · EOA が一時的に SCW 機能を獲得する Pectra アップグレード]] 阅读同业 / 对照背景，并结合 [[fintech/INDEX|fintech index]] 阅读更广泛的系统 / 监管边界。

## Key facts

- 4337 = 创建新 SCW（2023-03 主网上应用层启用）^[extracted]
- 7702 = 升级既有 EOA（2025-05 Pectra 在协议层启用）^[extracted]
- 两者共享 EntryPoint 基础设施 ^[extracted]
- 4337 渗透率 < 5%（2023-2025）· 7702 预计会快速普及 ^[extracted]
- 主要推动者分化: 4337 = Privy / Coinbase CDP / 钱包供应商 · 7702 = Vitalik / Ethereum 核心开发者 / MetaMask ^[extracted]

## Mechanism / How it works

**双轨详细对照**:

| 视角 | ERC-4337 | ERC-7702 |
|---|---|---|
| 目标用户 | 新用户 / fintech embedded wallet | 既有 EOA 用户 / DeFi 老牌钱包 |
| 迁移成本 | 新地址 + 资产迁移 | 零成本 · 原地升级 |
| 渗透速度 | 慢（用户需要主动创建 SCW） | 快（可一键升级） |
| 主要推动者 | Privy / Coinbase CDP / 钱包供应商 | Vitalik / Ethereum 核心开发者 / MetaMask |
| 协议层改动 | 无（纯应用层） | 有（新 tx type） |
| 适用场景 | 商户向客户发行新钱包 | 钱包应用为既有用户升级体验 |
| 风险模型 | SCW 合约本身的安全审计 | delegation 目标合约 + EOA 私钥的双重风险 |

**互补性的根本原因**: Ethereum 已经存在数千万 EOA 用户 · 历史资产 / ENS / NFT / DeFi 仓位无法轻易迁移 —— 这是 4337 路径的难点。另一方面，经由 fintech 进入的新用户（Stripe / PayPal / Revolut 的终端用户）从未持有过 EOA · 通过 4337 直接发行 SCW 是最优路径。

**实际场景中的分工**:
- Stripe 商户向新用户发行 wallet → 4337 SCW（经由 [[agent-economy/privy-embedded-wallet-overview|Privy]]）
- MetaMask 用户想在 DeFi 中 batch approve+swap → 7702 临时 delegation
- 机构 multi-sig → Safe（基于 4337 Modular）
- DeFi 高级用户用 session key 授权 trading bot → 4337 或 7702 + [[agent-economy/erc-7715-overview|ERC-7715]]

## Origin & evolution

2022-2024 年期间 · Ethereum AA 阵营内部曾有“协议层 vs 应用层”的争论 —— 4337 押注应用层先行 · 7702 随后在协议层补充。Vitalik 在 2024-05 主推 7702 是重要转折点 · 表明 Ethereum 核心开发者正式支持“协议层 AA 是长期方向”。

Pectra 2025-05 启用后 · 2026-Q1-Q2 主流钱包预计默认支持 · 7702 的渗透速度将显著超过 4337 的历史曲线（因为 EOA 用户不需要迁移地址）。
