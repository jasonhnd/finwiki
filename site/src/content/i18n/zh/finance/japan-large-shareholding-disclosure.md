# 日本大量持股披露

## 概述

日本的大量持股披露制度是金融商品交易法（FIEA）用于揭示大宗持股、影响力和市场供给信息的「5%规则」。一旦持有人的持股比例超过 5%，原则上须在五个营业日内提交大量持股报告书。此后发生重大变更，包括 1% 或以上的持股比例变动，原则上须在五个营业日内提交变更报告书。提交通过 EDINET 进行。

本页是基于 1 年 2026  月修订后的制度撰写的。FSA 的提交页面于 2026-05-01 更新，并说明修订后的大量持股制度自当日起生效，改变了持股比例的计算、共同持有人范围和表格。请谨慎对待出处日期。

## 制度速览

| 主题 | 经验法则 |
|---|---|
| 触发条件 | 上市股份等的持股比例超过 5%。 |
| 初次报告 | 大量持股报告书，原则上在五个营业日内。 |
| 变更报告 | 在 1% 或以上的变动或其他重大变更之后，原则上在五个营业日内。 |
| 提交渠道 | EDINET 电子提交；纸质提交不是通常的途径。 |
| 目的 | 围绕经营影响力和市场供需的透明度。 |
| 2026 后须注意 | 2026-05-01 生效的改革改变了计算、共同持有人范围和表格。 |

## 它不是什么

大量持股披露不是要约收购文件。它披露持有人、持股比例、目的、共同持有人以及相关的大宗持股信息，但其本身并不披露要约价格、要约期间、收购条件、资金证明或目标公司董事会意见。

| 提交文件 | 公开记录字段 |
|---|---|
| 大量持股报告书 | 谁以何种持股结构、出于何种目的越过了 5%？ |
| 变更报告书 | 自最新报告以来发生了什么变化？ |
| 要约收购说明书 | 要约条款、资金、条件以及 TOB 后计划是什么？ |
| TDnet 披露 | 通过适时披露传播的上市公司事件。 |

[[finance/japan-tender-offer-process|Japan tender offer process]] 与本页是相互独立但又彼此关联的途径。

## 持有人分析

在 2026 后的制度下，公开提交文件包含诸如以下的持有人分析字段：

- 直接持有；
- 投资权限；
- 表决权或指示权限；
- 共同持有人；
- 修订规则纳入时的衍生品或股票挂钩敞口；
- 持有目的；
- 重要提案行为；
- 披露时的质押 / 借款 / 融资背景；
- 快速增持或抛售模式。

汇总字段可包括基金、顾问主体、关联方、衍生品交易对手或共同行动人。提交人、共同持有人、目的和变更历史在披露时一并记录。

## 维权 / 控制权解读

大量持股提交文件是 [[finance/japan-activist-investor-playbook|activist investor playbook]] 页面的一部分。它们可以显示进入时点、增持、共同持有人变更、目的修订，以及提交人是否发出重要提案行为的信号。相关的公开记录包括：

- 股东提案；
- 股东大会表决结果；
- 公开信 / 维权活动网站；
- TDnet 公司回应；
- 如有则包括 TOB 提交文件；
- 发行人治理改革；
- 市场价格与成交量背景。

对于银行、券商、保险和金融科技主体，仅在公开出处支持主体层面的相关性时，才将相关的控制权案例链接回 [[finance/INDEX|finance INDEX]] 页面。

## EDINET 工作流

1. 在 EDINET 中搜索发行人和提交人。
2. 调取大量持股报告书和最新的变更报告书。
3. 检查该报告是普通报告书、特例（简式）报告书、修订还是订正。
4. 提取持股比例、共同持有人、目的、重要提案行为，以及存在时的担保 / 融资披露。
5. 建立带日期的持股时间线。
6. 就公司回应或相关公司行动，与 TDnet / 上市公司披露进行交叉核对。
7. 如存在 TOB，则单独阅读要约收购说明书。

## 阅读核对清单

| 字段 | 原因 |
|---|---|
| 提交人 / 持有人 | 识别法律上的报告人，未必是最终的经济集团。 |
| 共同持有人 | 决定汇总以及可能的协同影响力。 |
| 持股比例 | 核心阈值和变更报告的依据。 |
| 持有目的 | 区分纯投资、政策投资、重要提案、控制或其他目的。 |
| 收购资金 / 借款 | 可揭示杠杆、融资和交易意图。 |
| 近期交易 | 显示增持 / 处置的节奏。 |
| 衍生品 / 合约权利 | 改革后以及隐性敞口分析的重要内容。 |
| 修订历史 | 订正先前的提交文件，并可能改变解读。 |
