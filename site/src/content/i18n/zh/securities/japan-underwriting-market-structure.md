# 日本承销市场结构

## Overview

日本承销是连接发行人、证券公司、投资者、交易所、披露系统和清算结算基础设施的融资桥梁。主承销商 / 主办券商的角色不只是销售分销；它还锚定尽职调查、上市准备、定价、簿记建档、承销团协调和投资者保护控制。

## Functional Chain

| 阶段 | 主要参与者 | 分析解读 |
|---|---|---|
| 发行人准备 | 发行人、审计师、法律顾问、主承销证券公司 | 治理、会计、内部控制、股权故事、目标市场适格性。 |
| 承销尽调 | 主承销商和承销团成员 | JSDA 承销规则和公司内部控制塑造尽调和审批流程。 |
| 上市审查 | [[securities/tokyo-stock-exchange|Tokyo Stock Exchange]] / JPX-R 及相关方 | 交易所上市标准和审查流程不同于证券公司的承销判断。 |
| 披露 | EDINET、招股说明书、法定 / 交易所披露 | 公开投资者信息和责任界面。 |
| 簿记建档 / 定价 | 主办券商、承销团、机构 / 零售投资者 | 需求发现、分配政策、估值、适用时的稳定操作。 |
| 交易 / 结算 | 交易所 / PTS、[[securities/japan-securities-clearing-corp|JSCC]]、[[securities/japan-securities-depository-center|JASDEC]] | 新发行或出售的证券进入市场基础设施层。 |

## IPO / PO Reading

进行 IPO 和公开发行分析时，区分四个问题：

1. **上市准备度**:发行人能否满足交易所上市标准和披露预期？
2. **承销判断**:主承销商能否完成尽职调查并形成可辩护的承销观点？
3. **分销能力**:谁能向机构和零售投资者配售股票？
4. **售后市场质量**:上市后会发生什么，包括流动性、锁定、稳定操作、研究覆盖和投资者关系？

JPX 的上市前和上市指南页面有助于理解交易所侧流程。JSDA 的规则和自律材料有助于理解证券公司行为层。EDINET 和招股说明书材料属于公开披露层。

## DCM / Bond Underwriting Reading

债务资本市场分析增加另一组约束：

- 发行人评级 / 信用利差；
- 银行关系和投资者基础；
- 合格投资者类型；
- JGB 收益率和 swap curve 环境；
- 通过 [[securities/japan-securities-depository-center|JASDEC]] 的结算路径和现金结算管线；
- 披露和持续义务画像。

这就是为什么即使在证券承销页面，[[money-market/jgb-repo-market-japan|Japan JGB repo market]] 和 [[money-market/INDEX|money-market domain]] 也可能重要。

## League Table Boundary

[[finance/japan-ib-league-table|Japan IB league table]] 被记录为排名视角，而不是市场结构本身。League table 会随以下因素变化：

- 交易日期范围；
- 是否纳入 IPO、PO、CB、债券、M&A 或贷款相关角色；
- 日本国内 vs 跨境交易定义；
- 承销商 credit-splitting 方法；
- 母集团 vs 法律实体归属。

使用 league table 来问“谁在这个切片中活跃”，然后使用本页来问“该公司履行了什么角色，以及哪个监管 / 基础设施层重要？”

## Risk Controls and Failure Modes

| 风险 | 出现位置 |
|---|---|
| 发行人尽调薄弱 | 主承销商审查、审计师协调、JPX 上市审查。 |
| 定价过于激进 | 簿记建档、估值、售后市场表现。 |
| 分配冲突 | 机构 / 零售分配、集团关系、基石 / 母公司客户顾虑。 |
| 披露缺口 | 招股说明书、EDINET、TDnet、风险因素完整性。 |
| 市场滥用 / 稳定操作边界 | JSDA 规则、交易所规则、内部合规、监控。 |
| 结算 / 运营失败 | JSCC、JASDEC、过户代理、券商后台。 |
