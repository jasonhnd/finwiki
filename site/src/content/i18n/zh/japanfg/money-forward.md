## 摘要

Money Forward 是日本 PFM（Personal Financial Management）和中小企业会计 SaaS 的代表性公司。核心双轴是**个人家庭账簿应用“Money Forward ME”**（超过 1500 万 用户）与法人向 **“Money Forward Cloud”**。创始人**辻庸介**来自原 Monex 集团，于 2012-05-18 创业；2017-09-29 在东证 Mothers 上市，2022  转入东证 Growth，之后进入东证 Prime（3994）。其**与超过 1500 行 家金融机构 API 连接**是技术护城河。2025-04-15 将原本与 MUFG 的 JV “Biz Forward”（保理）独立纳入旗下，强化了不依赖大型银行的独立性。中期黑字化计划延期曾压低股价，但 SaaS ARR 扩大仍是主要阅读线索。^[extracted]

## 2. 业务分部地图

| 分部 | 主要服务 | 目标客户 | 竞争 |
|---|---|---|---|
| 个人 PFM | Money Forward ME | 个人用户（超过 1500 万 用户） | Zaim / OsidOri / 各银行 app |
| 法人 SaaS | Money Forward Cloud（会计、工资、经费、发票） | 中小企业 | freee / Yayoi / TKC / Bugyo |
| 大企业 | Money Forward PFP | 大企业 CFO 部门（资金 / 财务 SaaS） | Oracle / SAP / 国内大型 ERP |
| 法人整合 | Money Forward Business Company | 中堅 / 大企业 | freee / 各类 ERP |
| 保理 | Money Forward Biz Forward | 中小企业（发票收购） | OLTA / Yup / 银行系平台 |
| 金融机构 BPaaS | API 连接基础（超过 1500 行 家机构） | 金融机构 | TIS / NTT Data / Plaid |

### 个人 PFM x 法人 SaaS 双轴战略

- **Money Forward ME**：超过 1500 万 用户，是日本国内最大 PFM 基础，结合家庭账簿、资产可视化和金融机构批量连接。
- **Money Forward Cloud**：中小企业 SaaS 主战场，ARR 扩大是对照 freee 同规模竞争的主要指标。
- **双轴互补**：BtoC 个人数据带来信任和连接，BtoB 法人会计提供收入支柱。

### 金融机构 API 连接（技术护城河）

- **API hub 地位**：对金融机构来说，是 PFM 连接事实上的中间层。
- **竞争进入壁垒**：连接实施和长期运维成本会抑制新进入者。

### 大企业 BPaaS（Business Process as a Service）

- **Money Forward PFP**：面向大企业的资金 / 财务 SaaS，目标为 CFO 部门。
- **与既有 ERP 互补**：不是替代 Oracle / SAP，而是前端资金管理层。
- **下一收入支柱**：是突破中小企业 SaaS 客单价上限的方向。

### 合作 / 收购战略

- **与 Monex 的关系**：创始人辻庸介来自原 Monex，创业早期即有持续关系。
- **SBI HD 业务合作**（2018）：强化金融机构 API 连接基础，并与 SBI 经济圈联动。
- **MUFG JV “Biz Forward”**（2021-04  成立、2025-04-15 解散 笘・）：在 4  年整合保理合资业务，确保摆脱大型银行依赖后的独立性。
- **AS Capital 合作**：创业早期以来的 VC 伙伴。

### 竞争与差异化

| 竞争方 | 强项 | 与 Money Forward 的差异 |
|---|---|---|
| freee | 同等规模 SaaS，税理士渠道强 | freee 是完全云原生，Money Forward 更强调与既有业务的连接 |
| Yayoi | 国内份额领先，桌面端基础强 | Yayoi 以本地 / 税理士渠道为中心，Money Forward 更强于移动端 / 个人连接 |
| TKC | 会计事务所 BtoB 特化 | TKC 更接近税理士 OS，Money Forward 直接销售给企业 |
| 各银行 app | 银行强化自有 PFM | 银行 app 限于自家账户，Money Forward 提供跨机构连接 |

## 4. 监管 / 政策

- **主管**：金融厅（FSA），作为电子支付等代行服务提供者。
- **电子支付等代行服务**：2017 年银行法修正形成的登记类别，是 API 连接的法律基础。
- **个人信息保护**：大量持有个人金融数据，受个人信息保护委员会（PPC）监督。
- **近期政策议题**：
  - 2024 起：银行 API 开放义务推进，对 Money Forward 构成顺风。
  - 2025 起：电子账簿保存法和发票制度对应，拉动法人 SaaS 需求。
  - 2026 起：稳定币制度可能使 PFM 连接转化为信任 / 合规层。
