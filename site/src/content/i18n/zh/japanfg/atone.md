# atone（Net Protections BNPL 钱包）

## Wiki 路线

本条目位于 [[JapanFG/INDEX|JapanFG 索引]] 之下，是消费者侧 **BNPL 钱包**品牌页面。应与 [[JapanFG/bnpl-landscape|日本 BNPL 格局]]、母公司 [[JapanFG/net-protections-hd|Net Protections HD]] 以及直接竞品 [[JapanFG/paidy|Paidy]] 对照阅读。[[payments/japan-bnpl-credit-purchase-boundary|BNPL 与信用购买边界]] 提供监管框架，[[payments/credit-purchase-card-operators-japan-index|METI 信用购买注册表]] 提供登记证据，[[payments/japan-payment-scheme-economics-matrix|日本支付方案经济性矩阵]] 说明 atone 在卡支付和码支付之间的位置。

## TL;DR

atone 是 [[JapanFG/net-protections-hd|Net Protections HD]] 运营的**消费者侧 BNPL 钱包**。用户以账户注册为基础跨多个加盟店使用，按月结算，并在次月通过便利店收款、银行转账、账户扣款或相关卡产品一次性支付。相对于以发票随商品寄送为特征、由加盟店主导的 NP 后付款，atone 构成 Net Protections 双品牌战略中的**消费者主导 BNPL**一翼，与 [[JapanFG/paidy|Paidy]] 形成最直接竞争。

## 2. 业务角色

- 作为跨加盟店的**消费者账户型 BNPL**，与 NP 后付款共同构成 Net Protections 的两大核心品牌。^[extracted]
- 由于不依赖单一加盟店的发票流程，可向订阅制电商、移动电商和跨境电商扩展。

## 3. 独立页面的意义

[[JapanFG/net-protections-hd|Net Protections HD]] 主页面覆盖控股公司、IR 和海外业务，因此需要单独页面从消费者品牌角度分析 atone。该页面也为与 [[JapanFG/paidy|Paidy]] 对比 BNPL 消费者体验提供独立轴线。

## 4. 监管和政策

- **《分期销售法》**：综合 / 个别信用购买中介分类边界、授信判断模型和投诉处理仍是持续论点。
- **《个人信息保护法》**：账户注册所伴随的授信数据处理和第三方提供透明性是核心关注点。
