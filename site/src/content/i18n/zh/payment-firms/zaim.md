# Zaim (ザイム)

## Wiki 路径

本条目位于 [[payment-firms/INDEX|payment-firms INDEX]] 之下，作为一款紧邻规模大得多的 [[payment-firms/money-forward|Money Forward]] ME 同业的 **PFM (Personal Financial Management)** 应用。就邻接的消费金融语境，请与 [[payment-firms/bnpl-landscape|Japan BNPL landscape]] 一并阅读；就兄弟的家庭财务管理组合 [[payment-firms/freee|freee]] / [[payment-firms/smartbank|SmartBank]]，以及 KDDI 系列的分支故事，请与 [[megabanks/au-fh|au フィナンシャル HD]] 的企业语境一并阅读。[[payments/japan-bnpl-credit-purchase-boundary|BNPL and credit-purchase boundary]] 与 [[payments/credit-purchase-card-operators-japan-index|credit-purchase registry]] 页面位于监管 stack 的下一层，而 [[payments/japan-payment-scheme-economics-matrix|Japan payment scheme economics matrix]] 解释了 PFM 应用为何依赖账户 API 的经济性。

## TL;DR

Zaim 是 2011 年公开的国内最元老级别的 **记账应用**。作为与 [[payment-firms/money-forward|Money Forward]] ME 并列形成个人 PFM 类别的 fintech 余量，于 2022-04 被 **KDDI 子公司化**，并被重新配置为 au 经济圈 ([[megabanks/au-fh|au フィナンシャル HD]]) 的数据起点资产。其特征是收据 OCR + 银行・卡 API 联动 + 自治体给付金检索这一功能集。

- 通过收据拍摄 + OCR + 自动类别分类 + 银行・卡明细自动联动来可视化家庭财务的 PFM 应用。^[extracted]

## 3. 为何需要这个独立页面

PFM 类别常被看作 [[payment-firms/money-forward|Money Forward]] ME 一家独大，但 Zaim 走过 **独立创业 → KDDI 完全子公司化** 这一稀有的退出经路，作为通信运营商经济圈内 fintech 余量的典型例，需要独立页面。
