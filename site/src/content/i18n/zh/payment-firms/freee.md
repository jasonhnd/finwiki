# freee 株式会社（freee K.K.）

## Wiki 路径

本条目归属于 [[payment-firms/INDEX|payment-firms INDEX]]。请对照 [[payment-firms/money-forward|株式会社マネーフォワード (Money Forward)]] 了解同类 / 对比背景，并对照 [[fintech/japan-stablecoin-regulatory-landscape|日本 Stablecoin 法制度の三層構造（JPYC・USDC・Project Pax）]] 了解更广泛的系统 / 监管边界。

## TL;DR

与 [[payment-firms/money-forward|Money Forward]] 一道开拓了日本云会计・人事劳务 SaaS 行业的**面向小微企业的一体化后台平台运营商**（东证 Growth 4478）。由**佐佐木大辅**（前 Google Japan・SMB 营销出身）于 2012-07-09 创立，作为起源于工程的初创企业，在金融周边软件市场中占据独特的站位。于 2019-12-17 在东证 Mothers 上市（现 Growth），并于 2023 年实现扭亏为盈。聚焦中小企业 / 个体工商户 / 自由职业者，坚守**专攻法人 / 自营业者的路线**，不涉足家庭记账 App 领域（Money Forward ME 式的个人领域）。以基于 AI 的记账自动化以及对电子账簿保存法与发票制度（インボイス制度）的支持作为差异化轴心，志在打造将**会计 + 人事劳务 + 电子合同 + 法人卡 + 融资撮合**一体运营的「一体化 BPaaS（Business Platform as a Service）」模式。^[extracted]

## 1. 公司概况

**正式名称**：freee 株式会社
**英文名**：freee K.K.
**证券代码**：东证 Growth 4478（2019-12-17 上市，当时为 Mothers）
**成立**：2012-07-09 ^[extracted]
**创始人・代表**：佐佐木大辅（代表取缔役 CEO，前 Google Japan SMB 营销负责人）^[extracted]
**总部**：东京都品川区（大崎区域，基于公开企业信息）
**业态**：云会计・人事劳务 SaaS，及相关金融科技服务（电子合同、法人卡、融资撮合）

### 业态

它是纯粹的 SaaS 供应商，不持有银行业・证券业・保险业中任何一种牌照。金融功能（发卡・放贷・电子合同）通过合作金融机构・卡公司・电子合同事业者提供，属于**非银行型金融科技平台运营商**。它与拥有银行・证券・卡业务的综合金融机构——如超大型银行 FG（[[megabanks/mufg]] / [[megabanks/smfg]] / [[megabanks/mizuho-fg]]）或 [[megabanks/ndfg]] / [[megabanks/paypay-fg]]——形成两极对照，其设计理念是**以 SaaS 为枢纽，将既有金融机构的 API・卡・贷款捆绑起来**。

### 「中小企业 + 自由职业者」专攻

公开资料中反复提出的战略是**「小微企业一体化平台」**。具体而言：

- **客户群**：中小企业 / 个体工商户 / 自由职业者
- **业务领域**：会计・人事劳务・申告・电子合同・卡・融资这一整套**法人后台**
- **回避领域**：个人家庭记账（Money Forward ME 式）・面向大企业的 ERP（OBC 奉行 / SAP / Oracle 等领域）

### 法人 BPaaS 化

在历经对电子账簿保存法（2022 年修订）・发票制度（インボイス制度，2023-10 起施行）的应对之后，freee 正从**单品会计 SaaS**向**法人后台 BPaaS**扩展产品线。电子合同（freee 签名）・法人卡（freee 卡）・融资撮合（freee 融资），都采用**以会计数据为起点、捆绑金融・合同流程**的设计。^[extracted]

### 竞争格局

- **[[payment-firms/money-forward|Money Forward]]**：云会计领域最大的竞争对手，采用个人家庭记账（Money Forward ME）+ 法人会计（Money Forward 云）的双线打法。在法人领域是正面竞争对手。
- **弥生**：套装会计时代的老牌厂商，近年以云版（弥生会计 Online 等）转向 SaaS。面向中小・个体工商户会计的传统王者。
- **TKC**：经由税理士事务所网络的法人会计平台，税理士渠道主导的模式。
- **面向大企业的 ERP（OBC 奉行 / SAP / Oracle 等）**：freee 基本不进入的领域。

### 合作生态

与银行・卡公司・税理士事务所的 API 连接是事业基础。通过与超大型银行・网络银行・主要发卡机构的明细对接，以及面向税理士事务所的「freee 认定顾问」计划等，构建**共享渠道＝客户触点**的模式。^[extracted]

## 4. 监管・政策

- **主管**：金融厅（FSA）并非直接的监管机关，但受相关法令的强烈影响
- **会计・税务相关**：国税厅（e-Tax / 电子账簿保存法）、会计准则（J-GAAP / IFRS）
- **电子合同（freee 签名）**：电子签名法、电子账簿保存法
- **法人卡（freee 卡）**：分期销售法、资金结算法（部分）、合作卡公司（Mastercard 品牌）规制
- **融资撮合（freee 融资）**：贷金业法（合作金融机构一侧）、银行法（合作银行一侧）
- **个人信息保护**：个人信息保护法，财务数据处理的 API 连接中安全标准成为论点
- **上市规制**：东证 Growth 市场的信息披露・治理标准
- **近期政策论点**：
  - 2022〜 **电子账簿保存法**修订应对（电子交易数据保存义务化）
  - 2024〜 **电子发票（Peppol）应对**、政府数字化（e-Gov / GビズID 连接）
  - 2025〜 中小企业 DX 补助金、IT 导入补助金等制度联动
