# 日本的资金移动 vs 预付边界

## Wiki 路径

本条目位于 [[payments/INDEX|payments index]] 之下。请将其与 [[payments/card-acquiring-japan-stack|Japan card acquiring stack]] 一并阅读以了解邻接背景，并与 [[fintech/japan-stablecoin-regulatory-landscape|日本 Stablecoin 法制度の三層構造（JPYC・USDC・Project Pax）]] 一并阅读以了解更广泛的体系边界。

## TL;DR

实务上的边界是**可转移性**。资金移动服务在银行以外处理外汇交易，并使价值得以转移给他人。预付式支付手段主要是用于向发行人或参与加盟店付款的储值，并非设计为开放的个人间汇款轨道。

这一区分对于码支付、钱包、储值余额、游戏点数、交通系金钱、加盟店礼品价值以及类点数产品而言至关重要。产品的品牌标识并不可靠：同一应用可在彼此独立的法律层中包含资金移动余额、预付价值、点数以及卡关联支付。

## 边界测试

| 问题 | 资金移动信号 | 预付信号 |
|---|---|---|
| 用户能否将价值发送给他人？ | 能，尤其是当服务接受汇款指令时。 | 通常不能；余额主要用于用户自身的付款。 |
| 服务是否在银行以外进行外汇交易？ | 这指向资金移动业登记。 | 通常并非核心功能。 |
| 价值是否用于向发行人或参与加盟店购买商品 / 服务？ | 可能，但仅此本身并不充分。 | 核心使用场景。 |
| 当手段终止时，未使用价值是否受保护并可退款？ | 用户资金的保全取决于资金移动的类别。 | 当使用在资金结算法规则下终止时，适用预付退款手续。 |
| 余额是高额汇款还是低额汇款？ | 类别很重要：第一种、第二种或第三种资金移动业。 | 比起汇款限额，预付规则与发行人义务更为重要。 |

## 资金移动类别

金融厅资料描述了 2021 之后的三种资金移动类别：

| 类别 | 功能 | 主要限制 |
|---|---|---|
| 第一种资金移动业 | 超过 1 万日元的高额转账。 | 认可；一般限制接受不附特定汇款指令的资金。 |
| 第二种资金移动业 | 1 万日元以下的转账。 | 登记；可能需要削减无关联的超额余额。 |
| 第三种资金移动业 | 50,000 以下的低额转账。 | 登记；收款人账户余额即使临时也不得超过 50,000 ，并在符合条件时允许以分别存款方式保全。 |

这一分类之所以重要，是因为对用户而言看似"余额"的钱包，可能因其是否执行汇款以及可发送或持有多少价值，而受到不同的监管。

## 预付层

金融厅将预付式支付手段解释为一个广义类别，包括礼券、礼品卡、磁性 / IC 预付卡以及服务器型预付手段。预付发行人可能需要申报或登记，必须提供用户信息，并必须保全所需额度的客户价值。当预付手段停止使用时，发行人在资金结算法下进行退款流程。

分析规则为：

- **封闭式或加盟店使用的储值**倾向于预付分析。
- **个人间可转移的价值**倾向于资金移动分析。
- **现金等价赎回或广泛的可转移性**应触发更高风险的法律审查，而非简单的"点数"或"预付"标签。

对于已登记的第三方型预付发行人群体，请使用 [[payments/prepaid-payment-instrument-issuers-japan-index]]，而非创建长尾发行人页面。

## 产品分解

| 产品面 | 可能的法律层 |
|---|---|
| 码支付应用 | 资金移动余额、预付余额、信用卡、银行借记、点数、优惠券（例如 [[payment-firms/paypay|PayPay]]、[[payment-firms/kyash|Kyash]]）。 |
| 零售电子货币 | 预付式支付手段、忠诚点数、加盟店结算合同。 |
| 市场平台钱包 | 用于卖家结款的资金移动余额、用于购买的预付额度、托管 / 结算功能（与 [[payment-firms/merpay|Merpay]] 比较）。 |
| 游戏 / 应用点数 | 服务器型预付式支付手段、促销点数、平台手续费经济学。 |
| 稳定币 / EPI 桥接 | 电子结算手段、资金移动途径、信托途径、加密资产 / ECISB 边界（参见 [[fintech/japan-ecisb-license|Japan ECISB license]]）。 |

## 为何重要

- **消费者保护**：退款、保全、披露以及防滥用管控取决于法律层。
- **业务经济学**：预付价值、汇款余额、奖励点数以及加盟店结算均产生不同的负债与浮存金状况。
- **牌照战略**：企业可能为留在类预付结构中而避免 P2P 转移，或为实现钱包网络效应而取得资金移动业登记。
- **JapanFG 路由**：银行集团、支付集团、零售商以及稳定币项目应按法律余额层而非应用 UI 进行比较。
