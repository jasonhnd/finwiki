# 日本卡收单堆栈

## 长话短说

日本卡收单应被解读为**商户签约和卡数据控制堆栈**，而不仅仅是支付处理功能。日本经济产业省的《分期付款销售法》材料将注册商户承包角色定义为签订允许商户处理信用卡号码的合同的业务，并根据谁拥有批准商户接受的最终权力进行注册。

本页路由有关收单机构、PSP、商户入职、PCI DSS、欺诈控制以及注册商户签约方与另一收单机构下工作的服务提供商之间的法律差异的问题。

## 核心堆栈

| 图层 | 功能 | 主要问题 |
|---|---|---|
| 持卡人/发卡机构 | 发行卡并管理持卡人信用/支付关系。 | 消费信用和还款关系归谁所有？ |
| 网络/品牌 | 提供卡网络规则、品牌认可、清算约定和安全要求。 | 哪些网络规则适用于交易？ |
| 收单方/商户签约方 | 签订商户合同并负责商户管理。 | 谁有最终权力让商户接受银行卡？ |
| PSP/支付网关 | 集成商家结账、标记化、欺诈工具、路由和结算报告。 | 是否只是处理/初筛，还是拥有商家审批权限？ |
| 商人 | 销售商品/服务并获得卡受理能力。 | 商户是否遵守卡数据和防欺诈要求？ |

## 登记范围

日本的关键法律界限是《分期付款销售法》中的“信用卡号码处理合同签订业务”。

| 情况 | 可能阅读 |
|---|---|
| 收单方签署商户合同并保留最终审批权。 | 收单方是注册的商户缔约方； PSP 工作可能会得到委托支持。 |
| PSP从收单机构获得全面的权力，做出实际的最终商户批准决定，并对商户进行管理。 | PSP 本身可能需要注册为商户缔约方。 |
| PSP仅进行初筛、技术对接或运营支持，注册收购方保留最终决定权。 | PSP 角色本身可能不需要注册，但合同事实很重要。 |
| 外国公司在日本开展业务。 | 日本经济产业省表示，相关业务需要在国内营业所注册。 |

## 运营职责

METI 的常见问题解答强调了对 FinWiki 分析至关重要的三项运营职责：

- **商户调查**：初步和持续的商户审查，以纠正或排除有问题的商户。
- **卡号管理**：控制信用卡号码和相关客户/交易信息的泄漏。
- **欺诈预防**：分层控制，例如 PCI DSS、卡数据不保留、EMV / 3-D Secure、授权监控和基于风险的商户控制。

对于公司分析来说，这意味着 PSP 的护城河不仅仅是 API 的便利性。更有价值的部分是其合规性和风险引擎：承保商户、监控欺诈、减少退款以及满足收单方/网络/监管期望。

## 商业模式镜头

| 收益/风险项目 | 为什么这很重要 |
|---|---|
| 商户折扣/手续费 | 核心支付率池，但经常受到竞争和大商户谈判的压缩。 |
| 网关及增值服务 | 令牌化、定期计费、欺诈工具、订阅、多重支付编排和数据服务。 |
| 结算时间 | 创建营运资金、信贷、储备和运营风险问题。 |
| 退款/欺诈风险 | 确定承保标准、准备金政策和商户监控成本。 |
| 注册和安全负担 | 提高固定成本并有利于具有合规基础设施的规模化提供商。 |
