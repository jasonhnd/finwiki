# Rakuten Group 移动-金融捆绑案例 —— 电信现金消耗由卡 / 银行 / 证券利润交叉补贴

## Wiki route

本条目归于 [[business/INDEX|business INDEX]] 之下,作为上市公司战略案例。请将其与 [[business/ntt-docomo-d-point-telco-finance-case|NTT Docomo d-Point telco-finance case]] 作为电信补贴金融而非相反的对照模式、与 [[business/gmo-internet-group|GMO Internet Group]] 作为互联网到金融集团平行案例,以及 [[business/kitao-yoshitaka-sbi-independent-strategy-case|Kitao Yoshitaka SBI independent strategy]] 作为对照的独立 FG 路径一并阅读。配合 [[business/INDEX|business INDEX]] 以了解实体覆盖。

## TL;DR

Rakuten Group(TSE 4755)作为一个集团运营,其中**盈利的金融子公司交叉补贴一个深度亏损的移动网络运营商**。核心引擎——[[card-issuers/rakuten-card|Rakuten Card]](按交易量计日本最大的信用卡发行方)、[[banking/rakuten-bank|Rakuten Bank]](日本最大的互联网银行,于 2023-04IPO)、[[securities-firms/rakuten-securities|Rakuten Securities]](顶 3 在线券商)及 Rakuten Insurance / Rakuten Edy——产生为 Rakuten Mobile 网络铺设提供资金的经营现金(自 2019以来累计亏损估计超过 ¥1万亿)。

2023-2025 再资本化举措——Rakuten Bank IPO、Rakuten Securities 部分出售给 Mizuho,以及多次直接 / 可转换债券配售——揭示了一个母公司在慢动作去杠杆中,同时仍试图保全集团逻辑,即移动 / 电商 / 支付 / 金融超级应用能复合出独立竞争者无法获得的网络效应。

## 1. 集团架构

| 板块 | 代表性实体 | 利润状况(2024-2025) |
|---|---|---|
| 互联网服务 | Rakuten Ichiba(电商)、Rakuten Travel | 盈利,成熟增长 |
| 移动 | Rakuten Mobile(RMK)、Rakuten Symphony | 重度经营亏损,基础设施资本支出 |
| 金融 | [[card-issuers/rakuten-card|Rakuten Card]]、[[banking/rakuten-bank|Rakuten Bank]]、[[securities-firms/rakuten-securities|Rakuten Securities]]、Rakuten Pay | 高度盈利;增长 + 交叉销售引擎 |
| 保险 | Rakuten Life、Rakuten General Insurance、Rakuten Insurance HD | 微利 |
| 忠诚度 | Rakuten Points(约 120百万会员) | 跨板块黏合剂 |

结构模式:金融板块利润为移动板块亏损买单,而母公司股权经由资本募集提供时机桥梁。

## 2. 移动现金消耗 —— 交叉补贴约束

Rakuten Mobile 于 2019-2020 作为第 4家运营商进入日本 MNO 市场,以软件虚拟化核心(Open RAN)从零构建自己的 4G/5G 网络。这需要:

- 跨基站铺设的数万亿日元资本支出
- 客户获取成本(免费层促销定价)
- 用户基础规模低于 5-6百万用户时的经营亏损(依 Rakuten 披露的估计盈亏平衡规模)

到 2024-2025, Rakuten Mobile 已达约 7-8百万线,但仍亏损。自 2019 以来移动板块累计亏损被广泛估计超过 ¥1万亿。

## 3. 金融利润引擎

| 子公司 | 角色 | 交叉补贴贡献 |
|---|---|---|
| **[[card-issuers/rakuten-card|Rakuten Card]]** | 按交易量计日本第 #1 发行方 | 商户交换费 + 循环利息 + Rakuten Points 引擎;估计年处理量 >¥20万亿 |
| **[[banking/rakuten-bank|Rakuten Bank]]** | 第 #1 互联网银行,于 2023-04 IPO(TSE 5838) | 净利息收入、手续费收入;经由 IPO 部分出售募集现金 |
| **[[securities-firms/rakuten-securities|Rakuten Securities]]** | 顶 3 在线券商(相对 SBI、Matsui) | 经纪费、保证金贷款;部分出售给 Mizuho |
| Rakuten Pay / Rakuten Edy | QR 码 + 预付电子货币 | 无现金生态系统联结,参见 [[payments/cashless-jp-landscape|cashless landscape]] |
| Rakuten Insurance HD | 人寿 + 非人寿 | 较小的交叉补贴贡献 |

经济飞轮:电商驱动卡交易 → 卡交易驱动积分发行 → 积分锁定驱动银行存款获取 + 证券账户开立 → 所有这些产生为移动提供资金的手续费 / 利息利润。

## 4. 2023-2025 再资本化举措

| 日期 | 行动 | 目的 |
|---|---|---|
| 2023-04 | Rakuten Bank IPO(TSE 5838) | 募集现金、标记股权价值、保留控制 |
| 2023-Q4 | Rakuten Securities 部分出售给 Mizuho(约 19.99%) | Mizuho 战略股权 + 现金给母公司 |
| 2024-Q1 | 多次债券配售(直接 + 可转换) | 再融资债券到期墙 |
| 2024 | Rakuten Symphony(5G 软件供应商)外部客户中标 | 在 Rakuten 之外将技术堆栈货币化 |
| 2024-2025 | Rakuten Card 债务发行 / 证券化 | 营运资本优化 |
| 2025 | 持续移动用户增长、减少(但仍为负)的移动亏损 | 接近盈亏平衡 |

每次再资本化都进一步推动集团从净亏损转向净正,而不打破交叉补贴链。

## 5. 比较交叉补贴模式

| 集团 | 交叉补贴方向 | 示例流向 |
|---|---|---|
| **Rakuten** | 金融 → 移动(本案例) | 卡 / 银行 / 证券利润填补移动亏损 |
| **NTT Docomo** | 电信 → 金融(下一条目) | 移动用户基础驱动 d-Point / d-Card / d-Barai 增长,参见 [[business/ntt-docomo-d-point-telco-finance-case|d-Point case]] |
| **KDDI / au** | 电信 → 金融 | au PAY / au Jibun Bank / au Insurance 捆绑入移动套餐 |
| **SoftBank / PayPay** | 电信 + 资本 → 金融 | [[megabanks/paypay-fg|PayPay FG]] 带 SoftBank 支持,参见 [[megabanks/paypay-fg|PayPay FG]] |
| **GMO Internet Group** | 互联网基础设施 → 金融 | [[business/gmo-internet-group|GMO Internet Group]] 支付 / 银行 / 外汇 叠加于互联网基础 |
| **SBI Holdings** | 独立战略 | [[business/kitao-yoshitaka-sbi-independent-strategy-case|Kitao Yoshitaka SBI]] —— 无集团交叉补贴,金融纯业务 |

Rakuten 在补贴方向上独特:金融利润补贴一个亏损的电信单元,与全球大多数超级应用结构相反。

## 6. 为何交叉补贴持续

1. **网络效应论点**——移动 + 电商 + 卡 + 银行 + 证券 + 旅行 + 保险 + 积分在一个 ID 上使复合效应成为可能,这是独立竞争者无法获得的。(经验证据参半。)
2. **沉没资本支出**——移动网络建设成本基本已沉没;放弃将减记资产并发出资本不自律的信号
3. **忠诚度 / 品牌凝聚**——Rakuten Points 作为跨服务的单一货币绑定用户;移动是驱动积分参与的「永远在线」服务
4. **创始人承诺**——三木谷浩史(CEO / 创始人)已公开多年承诺移动成功
5. **可选性**——若移动达到盈亏平衡加利润,合并实体的估值倍数将显著扩张

## 7. 交叉补贴模型的风险

- **金融子公司现金流中断**——对卡 / 银行 / 证券的监管或竞争冲击(如 FSA 的资本充足收紧——参见 [[exchanges/fsa-vasp-registration-system|FSA reach]] 模板)可能打破资金链
- **债券到期墙**——若利率维持高位,母公司层债务的再融资风险
- **移动竞争**——Docomo / KDDI / SoftBank 拥有优越覆盖;Rakuten Mobile 依赖漫游协议经济
- **交叉违约情境**——若移动进一步表现不佳,集团范围债券契约可能触发
- **强制剥离**——若再资本化举措持续,金融子公司可能被逐步出售(Securities → Mizuho 已发生),使集团被掏空

## 8. 比较:Rakuten Bank IPO vs 其他剥离

| 交易 | 载体 | 母公司保留 | 税务处理 |
|---|---|---|---|
| **Rakuten Bank 2023** | IPO 部分减持 | IPO 后约 63%(锁定逐步释放) | 出售部分的资本利得 |
| [[business/sony-fg-partial-spinoff-case|Sony FG 2025 (planned)]] | 部分分拆(株式分配) | <20% | 在 [[corporate-strategy/partial-spinoff-tax-deferral|partial-spinoff regime]] 下递延 |
| [[business/softbank-vision-fund-arm-ipo-template|Arm 2023]] | IPO 部分减持 | 约 90% | 出售部分的资本利得 |

Rakuten 选择 IPO 而非部分分拆,因为现金募集对母公司再资本化需求至关重要,而不仅是估值实现。完整选项集参见 [[corporate-strategy/spinoff-decision-tree-japan|spinoff decision tree Japan]]。

## 9. 反论点

- 「超级应用」论点被主张但未在证明此等量级移动亏损所需的水平上得到经验确立
- 减持金融子公司(银行 IPO、证券部分出售)随时间逐步削弱交叉补贴引擎
- 移动盈亏平衡规模被反复推迟——曾预测为 6百万用户变成 7-8百万再变成更多
- 金融子公司的竞争护城河取决于积分与电商流量;独立估值会更低
- 创始人 CEO 固守限制了理性终止移动的能力

## 10. 悬而未决的问题

- Rakuten Mobile 何时(若有)将达到经营盈亏平衡?
- 母公司最终是否将不得不剥离移动(出售给 NTT / KDDI / SoftBank)并承认押注失败?
- 逐步的金融子公司 IPO / 部分出售举措将如何影响 Mizuho 在 [[securities-firms/rakuten-securities|Rakuten Securities]] 中的战略地位?
- 若任何主要板块被剥离,积分忠诚度引擎能维持跨板块黏合吗?
- 鉴于母公司层信用压力,FSA 会增加对卡 / 银行资本充足的监管审查吗?
