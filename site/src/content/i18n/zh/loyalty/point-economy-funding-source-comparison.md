# 日本积分经济原资比较（V／楽天／d／PayPay／Ponta 锚点）

## 维基路径

本条目位于 [[loyalty/INDEX|loyalty index]] 之下，作为**跨系统比较**页面：它沿着真正决定其经济性的维度——**什么锚定该计划、谁为积分买单**——把五大共通积分系统*并排*阅读，而非逐运营商的深度。它是单一运营商深度解析 [[loyalty/rakuten-point-ecosystem-deep|Rakuten Point ecosystem deep]]、[[loyalty/v-point-smbc-ccc-case|V Point (SMBC × CCC) case]]、[[loyalty/d-point-detailed-ecosystem|d Point detailed ecosystem]]、[[loyalty/sb-yahoo-paypay-points-unified-ecosystem|SoftBank / Yahoo / PayPay unified points]] 与 [[loyalty/ponta-points-deep-dive|Ponta points deep dive]] 的比较性对应。经济脊柱是 [[loyalty/point-program-unit-economics|point program unit economics]]；计划清单是 [[loyalty/japan-points-landscape|Japan points and loyalty landscape]]。

## 要点速览

日本的「五巨头」共通积分——**V Point、楽天积分、d Point、PayPay Point、Ponta**——在消费者看来可互换（约 1% 回馈、广泛受理），却是结构上不同的经济机器。区分它们的变量是**锚点**：计划所构建于其上的高频资产。V Point 锚定于**银行／卡**（SMBC、Olive）；楽天 锚定于 **EC 习惯**（楽天市場 + SPU）；d Point 锚定于**移动线路**（docomo）；PayPay Point 锚定于**码支付钱包**；Ponta 锚定于**多赞助方联盟**（Lawson / KDDI / Recruit）。锚点决定**原资构成**（商户出资 vs 运营商出资 vs 发卡方出资）、**交叉销售去向**（积分喂养哪些金融产品）以及**数据形态**（ID 图谱能看到什么）。比较回馈率毫无意义；比较锚点与原资才是分析的全部。

| 系统 | 锚定资产 | 主要赞助方／运营商 | 金融去向 | 比较页面 |
|---|---|---|---|---|
| **V Point** | 银行 + 信用卡 | SMBC / CCCMK（SMFG-CCC JV） | [[megabanks/smfg|SMFG]] / Olive | [[loyalty/v-point-smbc-ccc-case|V Point case]] |
| **楽天积分** | EC 习惯（楽天市場） | 楽天グループ | [[payment-firms/rakuten-fg|Rakuten FG]] | [[loyalty/rakuten-point-ecosystem-deep|Rakuten Point deep]] |
| **d Point** | 移动线路（docomo） | NTT docomo | [[megabanks/ndfg|NDFG]] | [[loyalty/d-point-detailed-ecosystem|d Point ecosystem]] |
| **PayPay Point** | 码支付钱包 | PayPay / SoftBank / LY | [[megabanks/paypay-fg|PayPay FG]] | [[loyalty/sb-yahoo-paypay-points-unified-ecosystem|PayPay unified points]] |
| **Ponta** | 联盟（Lawson / KDDI / Recruit） | 株式会社ロイヤリティ マーケティング | au PAY / KDDI 路径 | [[loyalty/ponta-points-deep-dive|Ponta deep dive]] |

## 按锚点划分的原资

锚点预示原资构成，因为每个锚点以不同方式变现积分。映射到 [[loyalty/point-program-unit-economics|unit economics]] 中的原资模式框架：

| 系统 | 商户出资型核心 | 运营商出资型叠加 | 发卡方出资型（卡） | 自筹型（集团零售） |
|---|---|---|---|---|
| V Point | 共通积分受理 | 促销 | SMBC Card / Olive 回馈 | — |
| 楽天积分 | 商城卖家费 | **SPU 乘数 + 活动** | 楽天カード 回馈 | — |
| d Point | 伙伴受理 | d払い 活动 | d Card 回馈 | — |
| PayPay Point | 商户受理 | **大额活动还元** | PayPay Card 回馈 | — |
| Ponta | 联盟伙伴费 | au PAY / Recruit 活动 | — | Lawson 门店利润 |

两个模式凸显出来。**钱包锚定型**系统（PayPay，及楽天的 SPU 层）最重地倚赖运营商出资型活动——它们在买频次与一张 ID 图谱，在金融交叉销售与数据变现被计入之前看起来「亏损」（[[loyalty/retail-media-points-data-loop|the data-loop page]] 中的对账）。**联盟锚定型**的 Ponta 把原资分散到多个无关赞助方，这降低了任一单一赞助方的成本，却稀释了由谁拥有所得图谱。

## 为何锚点决定经济性

锚点不是品牌——它同时设定三件事：

1. **原资杠杆。** 银行／卡锚点（V Point）可从交换费与放贷利润中为回馈出资；钱包锚点（PayPay）必须先期花营销预算、之后再回收。同样的醒目费率，相反的现金流形态。
2. **交叉销售梯度。** 积分是通往锚点接下来所售之物的漏斗：V Point → 存款 / Olive；楽天 → 证券 / 银行 / 移动；d-point → docomo 金融 / [[megabanks/ndfg|NDFG]]；PayPay → [[megabanks/paypay-fg|PayPay FG]]；Ponta → au PAY。该梯度越陡，一个「不盈利」的积分就越说得通。
3. **数据形态。** 移动线路锚点看到身份 + 位置 + 账单；EC 锚点看到 SKU 级购物篮；钱包锚点看到支付频次；联盟看到广度但共享之。这是 [[loyalty/retail-media-points-data-loop|the data-loop page]] 中的 ID 图谱维度。

所以同样的 1% 回馈，在这五者之间，是一件银行交叉销售工具、一台 EC 留存引擎、一根电信捆绑杠杆、一项支付频次补贴，以及一套联盟受理方案——带着五种不同的资产负债表与数据足迹。

## 共享的负债，不同的会计压力

五者都生成一项**积分负债**，跑过同一套会计机制——ASBJ 企业会计准则第29号（IFRS 15）之下的合约负债／失效沉淀，编目于 [[loyalty/point-liability-accounting-boundary|point liability accounting boundary]]。但该负债上的*压力*各异：

- **活动密集**的系统（PayPay；楽天的 SPU／用途限定层）生成大额运营商出资型授予 → 损益近期看着更差，失效沉淀估计更要紧，用途限定积分（期間・用途限定）被用来引导兑换并抬高失效沉淀。
- **卡锚定型**系统（V Point）把回馈负债与交换费经济性交织，故积分成本紧挨发卡方自身的回馈算法。
- **联盟**系统（Ponta）必须在赞助方*之间*结算价值，这加入了 [[loyalty/point-exchange-network-risk|point exchange network risk]] 中的运营商间转移机制。

让所有这些系统的失效沉淀保持诚实的披露规范，如 [[loyalty/point-program-unit-economics|unit economics]] 所引，来自无现金推进协议会／ペイメントジャパン一侧。

## 为何这对 JapanFG／金融分析重要

- **比较锚点，而非回馈率。** 同类对比的「1% 回馈」是噪声；真正的差异是原资杠杆、交叉销售梯度与数据形态——全部由锚点设定。
- **锚点点名受益的金融集团。** V Point → [[megabanks/smfg|SMFG]]；楽天 → [[payment-firms/rakuten-fg|Rakuten FG]]；d-point → [[megabanks/ndfg|NDFG]]；PayPay → [[megabanks/paypay-fg|PayPay FG]]。读一个积分计划，便告诉你哪个集团正在降低其获客成本。
- **「亏损」取决于锚点。** 钱包锚定型与 EC 锚定型活动在损益上看起来最差，却可能在买最好的图谱与最陡的金融漏斗；联盟积分按赞助方看更便宜，却分散了资产。
