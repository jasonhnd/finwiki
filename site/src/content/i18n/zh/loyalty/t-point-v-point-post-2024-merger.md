# T-Point + V-Point 2024 合并之后 —— 双品牌过渡、CCCMK Holdings、积分负债转移

## Wiki 路由

本条目归于 [[loyalty/INDEX|loyalty index]]，作为 T-Point + V-Point 统一的 **2024-04-22 之后运营细节页面**，与记录结构性框架的战略性 [[loyalty/v-point-smbc-ccc-case|V Point (SMBC × CCC) bank-led common-point case]] 条目互为补充。请与 [[loyalty/japan-points-landscape|Japan points and loyalty landscape]]（生态地图）、[[loyalty/point-liability-accounting-boundary|point liability accounting boundary]]（IFRS 15 / ASBJ 29 负债转移机制）、[[loyalty/d-point-detailed-ecosystem|d Point detailed ecosystem]]（以电信为锚的同类）、[[loyalty/sb-yahoo-paypay-points-unified-ecosystem|SoftBank / Yahoo / PayPay points unified ecosystem]]（以钱包为锚的同类）、[[loyalty/ponta-points-deep-dive|Ponta points deep dive]]（第三种联盟模式）、[[JapanFG/smfg|SMFG]]（母公司金融控股背景）以及 [[JapanFG/smbc-card|SMBC Card (SMCC)]]（发卡实体）配合阅读。

## TL;DR

**2024-04-22 的 T-Point + V-Point 合并**是迄今日本最大的通用积分整合。**CCC Marketing 的 T-Point 通用积分业务与 SMBC 集团的 V-Point 信用卡奖励计划在 V-Point 品牌下统一**，由 **CCCMK Holdings（2023-06设立的 SMFG-CCC 合资公司）** 运营。战略意图是将 T-Point 约 70 百万卡的实体商户网络（TSUTAYA、FamilyMart 遗留、ENEOS、药妆店、餐厅）与 SMBC 的银行发卡骨干（SMBC Card / SMCC）结合，创造一个对抗 Rakuten（电商）、dポイント（电信）与 PayPay Points（钱包）的**银行主导通用积分制衡力量**。该统一确实是运营性的 —— 通用积分业务的 T-Point 品牌于 2024-04-22 终止 —— 但**双品牌过渡现实持续存在**：TSUTAYA 品牌的 T-card 在迁移窗口期内继续作为 V-Point 赚取卡运作，且 TSUTAYA 发行的卡在参与商户处仍可读取。SMBC 发行的 V-card（Olive一体型、SMBC Card）与遗留 CCC 发行的卡在同一 V-Point 品牌下共存。统一时的**积分负债转移机制**遵循 IFRS 15 / ASBJ 29 框架：遗留 T-Point 负债坐落于各发行合作方自身的资产负债表，兑换比率按合约约定，接收运营方（CCCMK 下的 V-Point）随遗留余额迁移而确认新负债。PayPay Points 从结构上不同的锚点（钱包而非卡 / 银行）运作，这一比较揭示了为何 V-Point 的竞争上限与下限形态有别。

## 合并与双品牌过渡的时间线

| 日期 | 事件 |
|---|---|
| 2003-10 | T Point 作为 CCC 的全国通用积分推出，以 TSUTAYA、FamilyMart、ENEOS 为锚点合作方 |
| 2020 | Vポイント 作为 SMBC Card（SMCC）专属信用卡奖励积分推出（对许多持卡人而言是 ワールドプレゼント 的替代） |
| 2022-10 | CCC 与 SMFG / SMBC 宣布有意将 T Point 与 V Point 统一于一个通用积分品牌之下 |
| 2023-03 | CCC 在 SMFG-CCC 合资载体内将 T Point 运营重组进 CCC Marketing 结构 |
| 2023-06 | CCCMK Holdings 设立（SMFG 一侧对 CCC Marketing 运营的资本参与） |
| 2023-10 | Olive Account 以 V-Point 为核心忠诚度层推出；为统一奠定基础 |
| 2024-01-09 | CCC / SMBC 联合新闻稿确认 2024-04-22 统一日期及 V-Point 下的品牌整合 |
| **2024-04-22** | **T Point 与 Vポイント 在「Vポイント」品牌下统一**；T-Point 通用积分品牌终止；合作商户网络迁移至 V-Point 受理 |
| 2024-06 起 | TSUTAYA 继续发行在参与商户赚取 V-Point 的实体「T-card 设计」卡 —— 双品牌现实 |
| 2025 | 遗留 T-card 持有者向 V-card 设计的滚动迁移；合作商户受理拓宽 |
| 2026 （持续中） | 长尾 T-card 持有者继续赚取 V-Point；完整设计迁移延伸至初始时间线之外 |

2024-04-22 日期标志着**品牌层统一**，但**运营现实是双品牌长尾**：TSUTAYA 品牌在遗留及新发行的 T-card 设计卡上持续存在，因为 TSUTAYA / CCC 运营在 T-card 设计中具有品牌资产的零售业务，而 SMBC 发行的卡（V-Point 品牌的 Olive一体型 与 SMBC Card）带有 V-Point 品牌。两种卡类型皆赚取进同一统一 V-Point 账本。

## CCCMK Holdings —— 合资载体

**CCCMK Holdings（CCC Marketing Holdings）** 是统一 V-Point 业务的运营载体。公开结构：

| 层 | 角色 |
|---|---|
| CCC（Culture Convenience Club） | 创始方与历史通用积分运营方；遗留 T-Point 运营方 |
| SMFG / SMBC | 银行侧资本合作方；带来 SMBC Card（SMCC）发行与 Olive Account 整合 |
| CCCMK Holdings | 持有统一 V-Point 通用积分运营的合资载体，2023-06 设立 |
| Vポイント株式会社 | CCCMK Holdings 下 V-Point 的具体运营公司 |

资本合作结构将 CCC 数据营销运营的一部分转移进 SMFG-CCC 伞下，同时保留 CCC 在零售（TSUTAYA 等）的品牌存在。SMFG 对 CCCMK Holdings 的资本贡献在 SMFG IR 材料中公开披露；要点是银行侧锚点在数据营销权利之外获得了对忠诚度运营的实质性经济利益。

## 双品牌机制 —— 何为 TSUTAYA 品牌 vs SMBC 品牌

2024-04-22 统一是一次**品牌层**整合；运营现实是面向客户的卡保留传承品牌：

| 卡类型 | 发行方 | 设计 / 品牌 | 赚取界面 |
|---|---|---|---|
| 遗留 T-card（2024之前） | CCC（TSUTAYA / FamilyMart 历史等） | T-card 设计 | 统一后在参与商户赚取 V-Point |
| 新 TSUTAYA T-card（2024之后） | CCC / TSUTAYA | T-card 设计 | 赚取 V-Point |
| V-Point Card（CCCMK 发行） | CCCMK Holdings | V-Point 设计 | 赚取 V-Point |
| Olive一体型（SMBC 发行） | SMBC | SMBC / Olive 设计 | 以银行账户整合赚取 V-Point |
| SMBC Card / SMCC | SMBC Card | SMBC Card 设计 | 以信用卡费率赚取 V-Point |

其含义是客户可持有上述任一卡形态，并将积分累积进同一 V-Point 账本。这一双品牌现实反映了 CCC 持续运营 TSUTAYA 零售，以及在 70百万卡基础上更换遗留卡需历时数年的实际情况。

## 与 PayPay Points 的比较

| 维度 | V-Point（2024 合并之后） | PayPay Points |
|---|---|---|
| 锚点类型 | 银行主导 + 卡主导 + 遗留 CCC 零售网络 | 钱包主导 + LY 数据 + SoftBank 电信 |
| 运营方 | CCCMK Holdings（SMFG-CCC 合资） | PayPay 株式会社（SoftBank + LY） |
| 身份起源 | 卡 / 银行账户（SMBC Card、Olive、CCCMK 卡） | PayPay app 登录（SMS-OTP，常与电信关联） |
| 钱包整合 | V NEAR PAY（有限 NFC）、Olive 整合 | PayPay（主导 QR 钱包；见 [[payments/japan-code-payment-operator-2025-market-share-matrix|2025 share matrix]]） |
| 信用卡整合 | SMBC Card、V-Point Card、Olive一体型 | PayPay Card（SoftBank / LY） |
| 银行整合 | SMBC + Olive | PayPay Bank（前 JNB） |
| 证券整合 | SBI証券 合作（SMBC × SBI 联盟） | PayPay Securities |
| 会员 ID 基数 | 130M+ 统一（整合时 CCC + SMBC 两侧） | 70M+ PayPay 注册用户 |
| 零售受理 | 继承的 CCC 实体商户网络 | 经 PayPay 商户基础的全国 QR 受理 |
| 自有产品赚取经济学 | 刷卡费率；Olive 奖励层 | 钱包使用率（活动出资） |
| 投资产品附着 | 以 V-Point 在 SBI証券 购买基金 | 以 PayPay Points 的 PayPay 証券 |
| 活动风格 | 银行式结构化促销（名义率较低，经济学持久） | 钱包式频繁活动（名义率较高，breakage 出资） |
| 数据图谱 | CCC 零售数据库 + SMBC 客户流水 | LY（Yahoo Japan + LINE）数据资产 + PayPay 交易流水 |

这一结构性比较揭示了两种不同的金融分发原型。**V-Point 的优势**是经 SBI 联盟的**投资 / NISA 交叉销售**，以及经 Olive 的**银行账户主账户锚点**。**PayPay Points 的优势**是以频繁活动驱动高 MAU 的**钱包活动护城河**。两套系统在**零售商户赚取界面** —— 便利店、药妆店、餐厅 —— 竞争最为直接，V-Point 的 CCC 传承网络在此与 PayPay 的全国 QR 覆盖重叠。

PayPay Points 与 Yahoo 积分及 SoftBank Smart Login 的更广统一见 [[loyalty/sb-yahoo-paypay-points-unified-ecosystem|SB/Yahoo/PayPay unified points]]。

## 积分负债转移机制 —— IFRS 15 / ASBJ 29 框架

2024-04-22 统一引出了一个非微不足道的**积分负债转移问题**。每个持有统一前 T-Point 或 V-Point 负债的运营方都须处理迁移：

| 事件 | 会计处理 |
|---|---|
| CCC 账面上的统一前 T-Point 负债 | 在 CCC / 合作方资产负债表上确认为合约负债（IFRS 15下）或计提负债（ASBJ 29下） |
| SMBC Card 账面上的统一前 V-Point 负债 | 在 SMBC Card / SMCC 账面上确认为合约负债 |
| 统一事件（2024-04-22） | 负债从遗留账本迁移至 CCCMK Holdings 下的统一 V-Point 账本；既定兑换比率与结算机制 |
| 接收运营方（CCCMK） | 随遗留余额迁移确认新负债；此后承担兑换义务 |
| 迁移窗口 | 就所转移负债余额，CCC / 合作方与 CCCMK / SMBC 之间的双边净额结算 |
| 跨计划兑换 | V-Point ↔ ANA 里程、V-Point ↔ 合作计划 以调整后的兑换率继续；结算环节机制描述于 [[loyalty/point-liability-accounting-boundary]] |

迁移的清晰版本是**1 T-Point = 1 V-Point 兑换**，由接收的 CCCMK / SMBC 实体承担兑换义务。摩擦版本是某些遗留合作方重新协商参与条款、某些长尾商户未迁移，以及某些活动授予余额具有不同的失效规则而需要规则协调。

分析师阅读统一后 SMCC / SMFG IR 材料的披露线索，是追踪**「ポイント引当金」是否仍作为 JGAAP 科目出现，抑或已迁移至「契約負債」（合约负债）** —— 这一迁移告诉你该公司是否已完全采用 IFRS 15 / ASBJ 29 递延收入模型。CCCMK Holdings 的披露方式较为有限，因为它不是单独上市实体，但 SMFG 层分部附注提供了部分可见性。

## SBI 联盟 —— 作为银行主导差异化的投资产品附着

**SMBC × SBI 联盟**是 V-Point 最独特的交叉销售路径。客户可使用累积的 V-Point 在 **SBI証券 购买投资信托份额**，这在 dポイント 或 PayPay Points 生态中对 SMBC 对齐客户而言无直接对等。其机制：

| 机制 | 描述 |
|---|---|
| 合格客户 | 关联至 V-Point 的 SBI証券 账户持有者 |
| 合格产品 | 选定的投资信托份额（主要な投資信託銘柄） |
| 兑换 | 1 V-Point = 1 日元 用于基金购买 |
| 税务处理 | 购买后适用标准 NISA / 特定口座 处理 |
| 结算 | V-Point 负债对应 SBI証券 基金购买订单注销 |

战略意图是使 V-Point 成为一种**近现金投资出资货币**，提升 V-Point 对较高收入 / 财富积累家庭的吸引力，这些家庭原本会置身于高 MAU 钱包积分细分之外。这是银行主导通用积分相对于电信主导与钱包主导同类的结构性差异化。
