# 日本のカード加盟店管理スタック

## 要約

日本のカード加盟店管理は、単なる決済処理機能ではなく、**加盟店契約とカードデータ管理のスタック**として読むべきである。METI の割賦販売法資料は、登録対象となる加盟店契約締結の役割を、加盟店がクレジットカード番号を取り扱えるようにする契約を締結する事業として定義し、加盟店受入を承認する最終権限を誰が持つかによって登録要否が決まるとしている。

このページは、アクワイアラ、PSP、加盟店オンボーディング、PCI DSS、不正対策、登録済み加盟店契約締結主体と、他のアクワイアラの下で動くサービス提供者との法的差異に関する問いをルートする。

## 中核スタック

| レイヤー | 機能 | 主な問い |
|---|---|---|
| カード会員 / イシュア | カードを発行し、カード会員の信用 / 支払関係を管理する。 | 消費者の信用供与と返済関係を誰が持つのか。 |
| ネットワーク / ブランド | カードネットワーク規則、ブランド受入、清算慣行、セキュリティ要件を提供する。 | 取引にどのネットワーク規則が適用されるのか。 |
| アクワイアラ / 加盟店契約締結主体 | 加盟店契約を締結し、加盟店管理に責任を負う。 | 加盟店にカード受入を認める最終権限を誰が持つのか。 |
| PSP / 決済ゲートウェイ | 加盟店チェックアウト、トークン化、不正対策ツール、ルーティング、精算レポートを統合する。 | 単なる処理 / 一次審査なのか、加盟店承認権限を持つのか。 |
| 加盟店 | 商品 / サービスを販売し、カード受入能力を得る。 | 加盟店はカードデータと不正防止要件を満たしているか。 |

## 登録境界

日本法上の重要な境界は、割賦販売法上の「クレジットカード番号等取扱契約締結事業」である。

| 状況 | ありうる読み方 |
|---|---|
| アクワイアラが加盟店契約に署名し、最終承認権限を保持している。 | アクワイアラが登録済み加盟店契約締結主体であり、PSP 業務は委託支援でありうる。 |
| PSP がアクワイアラから包括的権限を受け、実質的な最終加盟店承認判断を行い、加盟店を管理している。 | PSP 自身が加盟店契約締結主体として登録を要する場合がある。 |
| PSP が一次審査、技術接続、運用支援のみを行い、登録済みアクワイアラが最終権限を保持している。 | PSP 役割自体には登録が不要な場合があるが、契約実態が重要である。 |
| 外国会社が日本で当該事業を行う。 | METI は、該当事業について国内営業所の登録が必要と述べている。 |

## 運用上の義務

METI の FAQ は、FinWiki 分析で重要となる三つの運用義務を強調している。

- **加盟店調査**: 問題のある加盟店を是正または排除するための初期および継続的な加盟店審査。
- **カード番号管理**: クレジットカード番号および関連する顧客 / 取引情報の漏えいを防ぐ管理。
- **不正防止**: PCI DSS、カードデータ非保持、EMV / 3-D Secure、オーソリゼーション監視、リスクベースの加盟店管理などの多層的対策。

会社分析では、PSP の堀は API の利便性だけではない。より価値があるのは、加盟店の引受、不正監視、チャージバック削減、アクワイアラ / ネットワーク / 規制上の期待への対応を担うコンプライアンス・リスクエンジンである。

## 事業モデルの見方

| 収益 / リスク項目 | 重要な理由 |
|---|---|
| 加盟店手数料 / 処理手数料 | 中核的な決済 take-rate pool だが、競争と大口加盟店交渉により圧縮されがちである。 |
| ゲートウェイおよび付加価値サービス | トークン化、継続課金、不正対策ツール、サブスクリプション、複数決済オーケストレーション、データサービス。 |
| 精算タイミング | 運転資金、信用、リザーブ、オペレーショナルリスクの論点を生む。 |
| チャージバック / 不正エクスポージャー | 引受基準、リザーブ方針、加盟店監視コストを左右する。 |
| 登録およびセキュリティ負担 | 固定費を引き上げ、コンプライアンスインフラを持つ大規模事業者に有利に働く。 |
