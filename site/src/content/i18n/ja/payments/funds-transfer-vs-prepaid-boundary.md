# 日本における資金移動と前払の境界

## ウィキ上の位置づけ

この項目は [[payments/INDEX|payments index]] 配下に置く。隣接する文脈は [[payments/card-acquiring-japan-stack|Japan card acquiring stack]]、より広い制度境界は [[fintech/japan-stablecoin-regulatory-landscape|日本 Stablecoin 法制度の三層構造（JPYC・USDC・Project Pax）]] と合わせて読む。

## 要約

実務上の境界は **移転可能性** である。資金移動サービスは、銀行以外が為替取引を取り扱い、価値を別の人へ移動できるようにする。前払式支払手段は主に、発行者または加盟店で支払うための蓄積価値であり、開かれた個人間送金レールとして設計されていない。

この区別は、コード決済、ウォレット、蓄積残高、ゲームクレジット、交通系電子マネー、加盟店ギフト価値、ポイント型商品にとって中心的である。商品ブランドは信頼できる判定材料ではない。同じアプリ内に、資金移動残高、前払価値、ポイント、カード連携決済が別々の法的レイヤーとして含まれることがある。

## 境界判定

| 質問 | 資金移動のシグナル | 前払のシグナル |
|---|---|---|
| ユーザーは別の人に価値を送れるか。 | はい。特に送金指図を受け付ける場合。 | 通常はいいえ。残高は主に本人の支払いに使われる。 |
| 銀行以外で為替取引を行っているか。 | 資金移動業登録を示唆する。 | 通常、中核機能ではない。 |
| 価値は発行者または加盟店から商品 / サービスを買うために使われるか。 | あり得るが、それだけでは十分でない。 | 中核的な用途。 |
| 未使用価値は、手段終了時に保護され返金されるか。 | 利用者資金の保全は資金移動の類型に依存する。 | 資金決済法ルールの下で、利用終了時の前払式支払手段払戻手続が適用される。 |
| 残高は高額送金か低額送金か。 | 類型が重要。第一種、第二種、第三種資金移動業。 | 送金上限より、前払ルールと発行者義務が重要。 |

## 資金移動業の類型

FSA 資料では、2021 後の資金移動業を三類型で説明している。

| 類型 | 機能 | 主な制約 |
|---|---|---|
| 第一種資金移動業 | JPY 1 百万を超える高額移転。 | 認可制。具体的な送金指図なしに資金を受け入れることは原則として制限される。 |
| 第二種資金移動業 | JPY 1 百万以下の移転。 | 登録制。送金と無関係の過大な残高は削減が必要になる場合がある。 |
| 第三種資金移動業 | JPY 50,000 以下の低額移転。 | 登録制。受取人アカウント残高は一時的にも JPY 50,000 を超えられず、条件付きで分別預金による保全が認められる。 |

この分類は重要である。ユーザーには「残高」に見えるウォレットでも、送金機能の有無や送れる・保有できる価値の大きさにより、規制上の扱いが異なるからである。

## 前払レイヤー

FSA は、前払式支払手段を、商品券、ギフトカード、磁気 / IC 型プリペイドカード、サーバー型前払式支払手段を含む広いカテゴリとして説明している。前払式支払手段の発行者は、届出または登録が必要になる場合があり、利用者への情報提供と、必要な額の顧客価値保全を行う必要がある。前払式支払手段を廃止する場合、発行者は資金決済法に基づく払戻手続を行う。

分析上のルールは次のとおり。

- **クローズド型または加盟店利用の蓄積価値** は前払分析に寄りやすい。
- **人と人の間で移転可能な価値** は資金移動分析に寄りやすい。
- **現金同等の払戻しまたは広い移転可能性** がある場合は、単純に「ポイント」や「前払」とラベル付けするのではなく、より高リスクの法務レビューを発動すべきである。

登録済み第三者型前払式支払手段発行者の母集団には、長い末端発行者ページを作るのではなく [[payments/prepaid-payment-instrument-issuers-japan-index]] を使う。

## 商品の分解

| 商品表面 | あり得る法的レイヤー |
|---|---|
| コード決済アプリ | 資金移動残高、前払残高、クレジットカード、銀行デビット、ポイント、クーポン (例: [[JapanFG/paypay|PayPay]]、[[JapanFG/kyash|Kyash]])。 |
| 小売系電子マネー | 前払式支払手段、ロイヤルティポイント、加盟店精算契約。 |
| マーケットプレイス・ウォレット | 売上金支払い用の資金移動残高、購入用の前払クレジット、エスクロー / 精算機能 ([[JapanFG/merpay|Merpay]] と比較)。 |
| ゲーム / アプリ内クレジット | サーバー型前払式支払手段、販促ポイント、プラットフォーム手数料経済。 |
| ステーブルコイン / EPI ブリッジ | 電子決済手段、資金移動ルート、信託ルート、暗号資産 / ECISB 境界 ([[fintech/japan-ecisb-license|Japan ECISB license]] 参照)。 |

## 重要性

- **消費者保護**: 払戻し、資金保全、開示、不正利用対策は法的レイヤーに依存する。
- **事業経済性**: 前払価値、送金残高、還元ポイント、加盟店精算はそれぞれ異なる負債・フロート構造を作る。
- **ライセンス戦略**: 会社は前払に近い構造にとどまるために P2P 移転を避けることも、ウォレットのネットワーク効果を実現するため資金移動業登録を取得することもある。
- **JapanFG ルーティング**: 銀行グループ、決済グループ、小売企業、ステーブルコイン案件は、アプリ UI ではなく法的な残高レイヤーで比較すべきである。
