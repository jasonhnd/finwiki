# 日本の決済不正事案タイムライン 2023-2025



## ウィキ上の位置づけ

この項目は、時系列の事案・執行ページとして [[payments/INDEX|payments index]] の下に位置づける。[[payments/japan-card-security-authentication-controls|Japan card security and authentication controls]] の J-CSC ガイドライン 6.0 / 6.1  管理フレームワークと組み合わせ、銀行レール固有の不正面については [[payments/japan-bank-api-incident-and-fraud-control|Japan bank API incident and fraud control]]、四分類の文脈については [[payments/japan-payment-scheme-economics-matrix|Japan payment scheme economics matrix]] とあわせて読む。各事案類型でどの当事者がチャージバックリスクを負うかは [[payments/japan-card-issuer-acquirer-processor-split|card role split]]、PSP オンボーディング失敗事案は [[payments/psp-merchant-settlement-risk|PSP merchant settlement risk]]、後払い悪用による現金化は [[payments/japan-bnpl-credit-purchase-boundary|BNPL / credit-purchase boundary]] と比較する。カードブランド文脈は [[JapanFG/visa-worldwide-japan|Visa Worldwide Japan]]、[[JapanFG/mastercard-japan|Mastercard Japan]]、[[JapanFG/jcb|JCB]]、主要カード issuer のアンカーは [[JapanFG/mufg-nicos|MUFG NICOS]]、[[JapanFG/rakuten-card|Rakuten Card]]、[[JapanFG/aeon-financial-service|AEON Financial Service]]。

## 要約

日本の 2023-2025 期の決済不正は、**四つの面が同時に収れんしたこと**で特徴づけられる。(1) 漏えいまたはフィッシングされたカード番号がオンライン加盟店で使われる EC クレジットカード不正。これにより MET 6.0 / 6.1 J-CSC ガイドラインの厳格化と、EC カード決済における **EMV 3-DS 義務化**、すなわち 2025-03からの必須化につながった。(2) 金融機関を装う SMS を受けた被害者がフィッシングサイトに認証情報を入力し、正規に見える銀行アプリまたは銀行 API の引落指図を通じて資金を失う **スミッシング起点の銀行 API 不正指図**。(3) PayPay / d払い / au PAY / 楽天ペイを横断する **コード決済の不正利用事案**。多くは credential stuffing やソーシャルエンジニアリングによるアカウント乗っ取りに結びつく。(4) KYM (know-your-merchant) 管理が不十分なため、不正加盟店がカード取引を獲得し、チャージバック前に消える **PSP 加盟店オンボーディング失敗**。金融庁、経済産業省、警察庁、全国銀行協会は公開注意喚起を連携して行い、累積的な規制対応は、管理基準と加盟店 / PSP オンボーディング厳格性の双方について、2025 基準を大きく引き上げた。

## 集計された不正損失の文脈

この時期の日本の決済不正損失の全体像は、類型ごとに異なる統計制度で報告されるため、注意して読む必要がある。

| 出所 | 範囲 | パターン |
|---|---|---|
| 日本クレジット協会 | クレジットカード不正利用被害額 (issuer 側) | 年間被害額を記録。期間を通じて非対面 (EC) 不正が支配的カテゴリ |
| 警察庁サイバー犯罪統計 | フィッシング、スミッシング、不正アクセス | 期間を通じてフィッシング関連サイバー犯罪報告が大幅に増加 |
| 全国銀行協会 | インターネットバンキング不正送金 | スミッシング起点の不正送金波に関する定期注意喚起 |
| 金融庁ニュースルーム | 事業者に対する行政処分 | 銀行、カード issuer、PSP、前払式発行者にまたがる複数の執行事案 |
| 消費者庁 | 消費者向け注意喚起 | フィッシング、偽加盟店、BNPL 現金化、コード決済のソーシャルエンジニアリングに関する定期警告 |

全体像としては、2023 および 2024, を通じて **日本のカード不正被害額は EC チャネルで歴史的に高い水準に達し**、同時に **スミッシング起点の銀行口座侵入も大きく増加した**。この二つが、J-CSC ガイドライン厳格化と、銀行 API 側の認証 / デバイス紐づけ執行を後押しした。

## 2023 の事象

| 事案類型 | 何が起きたか | 規制 / 業界対応 |
|---|---|---|
| EC カード不正損失が高止まり | 加盟店侵害やフィッシング経路を通じたカード番号流出が、EC 不正利用を継続的に押し上げた。非対面チャネルは主要な不正損失類型であり続けた | 経済産業省 / 日本クレジット協会は、すべての EC カード受入に EMV 3-DS を義務づける方向へ進み、J-CSC ガイドライン 5.0 → 6.0 改訂パイプラインが加速 |
| 主要銀行を狙うスミッシング波 | MUFG、SMBC、Mizuho、楽天銀行などを装う SMS が被害者を認証情報収集サイトへ誘導し、その後、不正な銀行アプリまたはインターネットバンキング送金が発生 | 全国銀行協会が継続的な公開警告を発出。金融庁は銀行に対し、デバイス紐づけ、アプリ導入確認、高額送金認証の強化を促進 |
| コード決済アカウント乗っ取り事案 | credential stuffing や SIM スワップ経路によるアカウント乗っ取りが、複数ウォレットでの不正コード決済利用を引き起こした | ウォレット事業者はデバイス変更時の再認証と高額取引 OTP 執行を強化 |
| 金融庁の執行措置 | AML / 不正管理の不備を理由に、銀行、カード issuer、前払式発行者への行政処分が定期的に発生 | 事業者ごとのニュースルーム公表 |

## 2024 の事象

| 事案類型 | 何が起きたか | 規制 / 業界対応 |
|---|---|---|
| 銀行 API 不正指図の波 | スミッシング起点のフィッシングが被害者を偽銀行ログインページへ誘導し、取得された認証情報で、攻撃者管理デバイス上の銀行 API 送金指図または銀行アプリ有効化が正規に見える形で行われる面が明確化 | 全国銀行協会が警戒度の高い注意喚起を発出。参加銀行はデバイス紐づけ、生体認証による step-up、新デバイスでの送金有効化に 24時間のクーリング期間を強化 |
| スミッシング面の拡大 | 金融機関に加え、税務当局、配送サービス、政府機関を装う SMS が偽決済画面へ誘導。量と巧妙さが大きく増加 | 警察庁の公開統計が増加を確認。携帯キャリアのフィルタリングは改善したが、面を完全には排除できず |
| カード issuer 侵害事案 | 特定のカード issuer 事案でカード会員データが侵害され、強制再発行とチャージバック波が発生 | J-CSC ガイドライン 6.0 が公表され、非保持化ルールと脆弱性スキャン義務が強化 |
| EMV 3-DS 導入加速 | EC 加盟店が EMV 3-DS 導入を、2025-03 義務化を前に加速。中堅加盟店と PSP は実行圧力に直面 | 経済産業省 / J-Credit のガイダンス資料と J-CSC 6.0 / 6.1 タイムラインが導入を促進 |
| PSP 加盟店オンボーディング失敗 | PSP が不十分な KYM 管理で不正加盟店をオンボードした事案が発生。加盟店は架空商品でカード取引を受け入れ、チャージバック裁定前に消え、acquirer / PSP / issuer が損失を抱えた | 経済産業省は PSP / 加盟店契約当事者の義務を厳格化。PSP 業界は内部 KYM 枠組みを強化 |

## 2025 の事象

| 事案類型 | 何が起きたか | 規制 / 業界対応 |
|---|---|---|
| EC カード決済で EMV 3-DS が義務化 | 2025-03, 以降、EC カード受入には EMV 3-DS 認証が必要になった。J-CSC ガイドライン 6.0 / 6.1 枠組みに基づく | 業界全体でコンプライアンス圧力。非対応 EC 加盟店はカード決済受入を失うリスク |
| 銀行 API スミッシングが継続 | 認証強化にもかかわらず、銀行 API 不正指図面は 2025 に入っても継続。攻撃手法は進化し、フィッシングサイトで取得した情報に AI 生成音声通話で追撃する例も含む | 銀行と金融庁は公開注意喚起を継続。銀行横断モニタリングを強化 |
| コード決済不正事案 | PayPay、d払い、au PAY、楽天ペイの各ウォレットに対するアカウント乗っ取り・ソーシャルエンジニアリング事案が継続。一部では同一被害者に対する複数ウォレット横断攻撃も発生 | ウォレット事業者はデバイス紐づけと高額取引管理を引き続き強化 |
| 複数事業者への金融庁執行 | 管理不備を理由に、銀行、カード issuer、PSP、前払式発行者への執行が継続 | 事業者ごとのニュースルーム公表 |
| 税務当局 / 政府機関なりすましスミッシング | 納税期限前後に特に量が増え、消費者向け警告が強化 | 消費者庁と警察庁が公開注意喚起を継続 |

## 横断的な攻撃パターン

| パターン | 仕組み | 狙われるスキーム |
|---|---|---|
| スミッシング → フィッシングサイトで認証情報収集 | SMS が信頼ブランドを装い、被害者が攻撃者管理ページに認証情報を入力する | 銀行口座、カード口座、ウォレット口座 |
| SMS ワンタイムコードの窃取 | 攻撃者が被害者に SMS OTP を共有させる、または SIM スワップで OTP を傍受する | 銀行 API、カード 3-D Secure、ウォレット OTP |
| リモート操作マルウェアによるデバイス紐づけ回避 | 被害者が、攻撃者に被害者端末を操作させるマルウェアをインストールする | 銀行アプリ、ウォレットアプリ |
| 加盟店侵害またはスキマーによるカード番号収集 | EC 加盟店侵害または POS スキマーからカード番号が流出し、他加盟店で再利用される | EC カード決済 (主に) |
| credential stuffing によるアカウント乗っ取り | 他サイト流出認証情報をウォレット / カードポータルに試す | ウォレット、カードポータル |
| 偽加盟店の PSP オンボーディング | 不正者が弱い KYM を通過し、架空商品でカード取引を受け、チャージバック解決前に逃げる | カード類型。PSP / acquirer が損失負担 |
| BNPL / 後払い悪用による現金化 | 不正者が盗用 ID で後払い口座を開設し、現金同等商品の購入後に不払い | BNPL、コード決済後払い、割賦 |

## 損失を負う主体

| 面 | 主な損失負担者 |
|---|---|
| 3-D Secure なしのカード EC 不正 | 加盟店 (チャージバック責任) |
| 3-D Secure が正常完了したカード EC 不正 | issuer (liability shift) |
| 消費者口座の銀行 API 不正引落 | 銀行 (消費者に過失がない場合の預金者保護法枠組み) |
| コード決済の不正利用 | ウォレット事業者の約款による。事案ごとの証拠により異なる |
| カード issuer 侵害に伴う再発行 | issuer |
| PSP 加盟店オンボーディング失敗 | PSP / acquirer / 場合によりブランドが一部吸収 |
| スミッシング起点の認証情報開示 | 自発的開示と判断される場合は消費者負担になりやすい。ただし精巧ななりすまし事案では銀行がより柔軟に解釈した例もある |

**liability shift のダイナミクス**は投資インセンティブを左右する。issuer が残余損失を負う場合、issuer は不正モニタリングと 3-D Secure 導入に投資する。加盟店が損失を負う場合、加盟店側の不正検知に投資する。消費者が損失を負う場合、金融サービス事業者に枠組み改定を求める規制・政治圧力が高まる。2023-2025 期には、この三つのダイナミクスが同時に動いていた。

## 規制対応アーキテクチャ

2023-2025 不正に対する日本の規制対応は、複数の当局と自主規制団体に階層化されている。

| 主体 | 役割 |
|---|---|
| 金融庁 (FSA) | 銀行 / カード issuer / 前払式発行者 / ウォレット事業者の監督と執行 |
| 経済産業省 (METI) | 割賦販売法の所管。カード決済 / EC 加盟店セキュリティガイドライン (J-Credit と連携) |
| 日本クレジット協会 | カード決済セキュリティガイドライン (J-CSC)、業界調整 |
| 警察庁 | サイバー犯罪統計、スミッシング / フィッシング捜査、刑事執行 |
| 消費者庁 | 消費者警告、公開注意喚起 |
| 全国銀行協会 | 銀行側調整、公開注意喚起 |
| 資金決済法上の PSP および電子決済等代行業枠組み | 銀行 API と口座情報アクセスのルール |

当局横断の連携は強まりつつある。2024-2025 の主要事案の多くでは、金融庁、警察庁、関連業界団体が数日以内に連携した公開注意喚起を行った。

## 消費者保護枠組みの比較

保護枠組みは面によって大きく異なる。そのため、同じ消費者でも、不正事案で使われたスキームによって結果が大きく変わり得る。

| スキーム | 主な消費者保護レバー | 強さ |
|---|---|---|
| カード EC (3-D Secure あり) | ブランドのチャージバック規則 + 割賦契約における割賦販売法の抗弁の接続 | 強い |
| カード EC (3-D Secure なし) | ブランドのチャージバック規則。加盟店への liability shift | 強いが加盟店ルートに依存 |
| 銀行 API 不正引落 | 消費者に重過失がない場合の預金者保護法枠組み | 適用される場合は強い |
| 消費者の認証情報開示による銀行アプリログイン侵害 | 銀行約款と預金者保護法の事案別解釈 | 可変。消費者が何を開示したかによる |
| コード決済不正利用 | ウォレット事業者の約款。法定チャージバック枠組みなし | カード / 銀行より弱い |
| 前払式電子マネー不正利用 | 発行者約款。払戻しは発行終了時のみ | 最も弱い |
| BNPL / 後払い ID 不正 | 提供者約款 + 一般的な消費者信用枠組み | 可変 |

面ごとの差異は、**カード / 銀行のような最も強い枠組みに約款をそろえるよう事業者へ構造的圧力**を生む。消費者信頼のギャップを避けるためである。ただし、コード決済と前払式の面では法定義務ではないため、事業者ごとにばらつきが残る。

## スミッシングの運用メカニクス

2024-2025 のスミッシング面は、消費者側損失の持続的な源泉である。攻撃チェーンのどの段階を対象にするかによって、事業者側で検知または遮断できる管理点が変わるため、その仕組みを理解することが重要である。

| 攻撃チェーンの段階 | 事業者側の検知 / 管理面 |
|---|---|
| 1. 攻撃者が電話番号を収集 | 事業者の範囲外。携帯キャリア監視は部分的 |
| 2. 銀行 / カード / ウォレット / 政府を装う SMS 送信 | 携帯キャリアのフィルタリング、SMS 送信者認証標準 |
| 3. 被害者がリンクをクリックし、フィッシングサイトへ到達 | URL ブロックリスト連携、消費者ブラウザ警告 |
| 4. 被害者が認証情報を入力 | この段階では事業者側は検知できない。フィッシングサイトが事業者を模倣する |
| 5. 攻撃者が認証情報で事業者システムへログイン | **事業者の検知点 1**: デバイス紐づけ、IP / 位置情報異常、行動モニタリング |
| 6. 攻撃者が高額取引を開始 | **事業者の検知点 2**: 取引パターン監視、step-up 認証 |
| 7. 資金が攻撃者管理の受け子口座へ移動 | **事業者の検知点 3**: 受入口座パターン、受入側 AML / CFT 監視 |
| 8. 受け子が資金を現金化 | 出金時点での銀行 / ウォレット AML 監視 |

事業者側の主要なレバーは 5-7の段階にある。主要銀行における 2024-2025 の強化は、デバイス紐づけ、すなわち二次確認なしに新デバイスから 5 を行うことを防ぐ管理と、取引パターン監視、すなわちカード会員 / 口座保有者の通常行動から逸脱した 6 を遅延または遮断する管理に集中してきた。継続的な課題は、攻撃者がマルウェアまたはソーシャルエンジニアリング通話で被害者端末を遠隔操作する場合、**被害者管理デバイスからの正規に見えるセッション**がデバイス紐づけ管理を回避することである。
