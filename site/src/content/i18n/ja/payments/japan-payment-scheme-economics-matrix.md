# 日本の決済スキーム経済マトリクス

## 要約

日本のリテール決済は、現在四つの並行するスキーム類型で動いている。**カード**（Visa / Mastercard / JCB / Amex / Diners）、**コード**（PayPay / d払い / au PAY / 楽天ペイ / メルペイ / FamiPay）、**口座間送金**（Bank Pay / J-Debit / Cotra）、**プリペイド電子マネー**（Suica / PASMO / WAON / nanaco / Edy）である。それぞれ免許スタック、精算場所、手数料経済が異なるため、消費者から見た UX が似ていても、誰がポイントを負担し、誰がチャージバックを負い、誰が加盟店関係を管理するかには重大な差がある。このエントリは、読者が四つの経済スタックを同時に見る必要があるときに、他の [[payments/INDEX|payments index]] エントリが参照する横並び比較ページである。

## ウィキ上の位置づけ

これは横断スキームの参照ページとして [[payments/INDEX|payments index]] の配下に位置する。市場シェアの見方には [[payments/cashless-jp-landscape|Japan cashless landscape]]、カード類型の詳細には [[payments/japan-card-issuer-acquirer-processor-split|card role split]] と [[payments/japan-interchange-and-merchant-fee-stack|interchange and merchant fee stack]]、ウォレット類型の詳細には [[payments/japan-code-payment-competitive-map|code-payment competitive map]]、銀行直結の詳細には [[payments/account-to-account-payment-japan|A2A payment route]]、ウォレット / プリペイド境界には [[payments/funds-transfer-vs-prepaid-boundary|funds-transfer vs prepaid boundary]] とあわせて読む。四類型すべての免許ルーティングは [[financial-licenses/payment-license-stack|Japan payment license stack]] にある。

## 四類型スキームマップ

四つの類型は、消費者がタップ、スキャン、カード提示のどれを行うかで主に異なるのではない。価値が通る**残高レイヤー**と、事業者が持つ**免許ルート**が異なる。

| 類型 | 例 | 残高レイヤー | 主な免許ルート |
|---|---|---|---|
| カード決済 | Visa, Mastercard, [[card-issuers/jcb\|JCB]], Amex, Diners | イシュア与信枠（後払い） | 割賦販売法（信販 / 包括信用購入あっせん）+ ブランドルール |
| コード決済 | [[payment-firms/paypay\|PayPay]], d払い, [[payment-firms/au-payment\|au PAY]], 楽天ペイ, メルペイ, FamiPay | 資金移動残高 / プリペイド副残高 / カード連携 / 銀行デビット（混在） | 資金移動業 + 前払式支払手段（資金決済法） |
| 口座間送金 (A2A) | Bank Pay, J-Debit, Cotra 連携アプリ | 銀行預金口座（直接引落） | 銀行法 + JEPPO インフラ + 第三者の場合は電子決済等代行業 |
| プリペイド電子マネー | Suica, PASMO, [[card-issuers/aeon-card\|WAON]], nanaco, [[payment-firms/rakuten-edy\|Edy]] | プリペイド蓄積価値（クローズドループまたは準オープン） | 前払式支払手段（資金決済法） |

各類型には、ネットワークルールと精算場所を所有する**スキーム運営者**と、消費者保護、資金保全、払戻しメカニクスを決める**日本法上の免許**がある。

## スキーム運営者と精算場所

| 類型 | スキーム運営者のパターン | 実際に資金が精算される場所 |
|---|---|---|
| カード | 国際ブランド（Visa / Mastercard / Amex / Diners）または国内ブランド（[[card-issuers/jcb\|JCB]]）。4者間スキームはイシュア / アクワイアラーを分離し、3者間スキーム（Amex / Diners / 一部 JCB カード）はそれらを一体化する | ブランドルールによる清算を通じたイシュア ↔ アクワイアラー精算。ネットポジションは最終的に銀行システムを通って移動する（円部分ではしばしば [[payments/japan-payment-clearing-and-settlement-infrastructure\|Zengin / BOJ-NET]] 経由） |
| コード | ウォレット運営者（PayPay Corp、NTT docomo、KDDI、Rakuten Pay、Mercari 等）。ウォレット運営者が通常は加盟店契約を持つか、Netstars のような QR ゲートウェイ PSP に集約を委託する | まず内部ウォレット台帳。加盟店銀行口座への定期ネット精算は全銀振込を通じて行う |
| A2A | Bank Pay / J-Debit は JEPPO。即時 P2P・加盟店送金は Zengin-Net + Cotra システム | 預金口座の直接引落。最終精算は全銀 / Cotra の 1 日 2 回のバッチ窓口で清算 |
| プリペイド電子マネー | 発行者（Suica は JR East、PASMO は PASMO Co.、WAON は AEON、nanaco は Seven Card Service、Edy は Rakuten Edy） | 発行者の蓄積価値台帳。加盟店支払いは発行者 ↔ 加盟店契約の精算スケジュールに基づき全銀経由で流れる |

4者間と 3者間のカード区別が重要なのは、3者間スキームにはインターチェンジ手数料がないためである。ブランド自身がイシュア兼アクワイアラーなので、アクワイアラー費用控除後の加盟店手数料全体をブランドが得る。

## 日本法上の免許

免許の問いは UX ではなく、残高レイヤーの下流にある。ページ読者は、取引がどの副残高に触れたかを特定せずに PayPay を「クレジットカードの競合」と呼ぶべきではない。

| 類型 | 免許スタック | 主な義務 |
|---|---|---|
| カード | 割賦販売法登録（包括信用購入あっせん業者 / 包括信用購入あっせん業等）。METI 115 一覧では、イシュア、加盟店契約主体（アクワイアラー）、PSP に別の登録がある | カード会員与信審査、セキュリティガイドライン（J-CSC）、加盟店 / PSP の PCI DSS、EC の EMV 3-DS、ブランドによるチャージバックルール |
| コード | 移転可能残高には資金移動業登録（第一種 / 第二種 / 第三種）。蓄積価値副残高には前払式支払手段発行者登録。同一事業者が複数免許を並行して持つことが多い | 利用者資金の保全（供託 / 信託 / 保証）、AML / CFT、資金決済法上の不正取引対応 |
| A2A | 口座そのものは銀行法。JEPPO インフラルール。第三者が銀行 API で支払指図を開始する場合は電子決済等代行業登録 | 銀行 API 契約に基づく認証 / デバイス紐付け、口座情報アクセスルール、不正引落補償（預金者保護法の枠組み） |
| プリペイド | 資金決済法上の前払式支払手段発行者届出（自家型）または登録（第三者型）。FSA 803行 [[payments/prepaid-payment-instrument-issuers-japan-index\|third-party prepaid registry]] | 未使用残高額の 50% を供託 / 信託 / 保証で保全する必要。発行終了時の払戻し手続き。有効期限 / 未使用残高会計 |

これらの境界は安定していない。事業者は P2P 送金機能を追加すると、残高をプリペイド副台帳から資金移動副台帳へ移すことが多く、そのたびに新たな免許分析が必要になる。判定については [[payments/funds-transfer-vs-prepaid-boundary|funds transfer vs prepaid boundary]] を参照。

## アクワイアラーモデル（単一アクワイアラーと複数アクワイアラー）

| 類型 | アクワイアラー構造 |
|---|---|
| カード | **複数アクワイアラー。** 加盟店はブランドごとに複数のアクワイアラー（SMBC Card、MUFG NICOS、JCB、AEON Financial Service、Rakuten Card Merchant Service、PF アクワイアラーとしての GMO-PG 等）から選べる。加盟店手数料は、アクワイアラー別、ブランド別、場合により MCC 別に交渉される。Amex / Diners（3者間）ではブランドが実質的に唯一のアクワイアラーである。 |
| コード | **ウォレットごとの単一アクワイアラー。** PayPay 加盟店は PayPay と契約する（または Netstars のようなゲートウェイ経由で接続を卸す）。d払い加盟店は NTT docomo と契約する。複数ウォレット受け入れは、複数のウォレット契約を結ぶか、一つの QR ゲートウェイアグリゲーターと契約することで実現する。 |
| A2A | **単一スキームアクワイアラー。** Bank Pay 加盟店は JEPPO と契約し、J-Debit 加盟店も JEPPO と契約する。同一スキームに代替アクワイアラーは存在しない。 |
| プリペイド電子マネー | **スキームごとの単一アクワイアラー。** Suica 受け入れは JR East（および PASMO グループ相互性）を通じて契約し、WAON は AEON、nanaco は Seven Card Service、Edy は Rakuten Edy を通じる。Suica / PASMO 間には鉄道会社間の 10 IC 相互利用スキームによる相互受け入れがある。 |

カード以外の類型における単一アクワイアラーパターンは、コード決済とプリペイドの加盟店手数料が、カード加盟店手数料よりも競争で規律されにくいと METI と JFTC が見る理由の一つである。

## 加盟店手数料レンジ（MDR + 取引ごとの固定費）

これらは公開報道ベースのレンジである。実際の加盟店手数料は、MCC、チケットサイズ、ECR と MPM QR、オンアスとオフアス、交渉力によって異なる。

| 類型 | 典型的な加盟店手数料（MDR） | 取引ごとの固定費 |
|---|---|---|
| カード | JFTC 2022: 単純平均 ~2.70%、加重平均 ~1.66%。Payments Japan 2022 ロードマップ: Category I 平均 2.63%、Category II 平均 2.89% | 古典的な MDR では通常なし。PSP レイヤーでは取引ごと手数料が加わる場合がある |
| コード（PayPay 等） | 小規模加盟店向け MPM QR は歴史的に 0%（PayPay 無料キャンペーン 2018-2021）→ 現在は一般加盟店で MPM 約 1.6% から 1.98%、CPM 約 1.98% から 2.95%。大手チェーンは交渉レートが低い | 一部ウォレットは 300円未満の小口取引に固定円手数料を加える |
| A2A (Bank Pay) | 公表レート範囲はコード決済と同等または低め。加盟店は JEPPO 料率表に基づく銀行側処理費を負担する | 口座引落処理費が取引ごとに発生 |
| プリペイド電子マネー | 通常 2-3% レンジ。交通圏の Suica 加盟店手数料は鉄道会社所有小売では歴史的に低め。WAON / nanaco 手数料は発行者が設定 | タップごとの処理費が組み込まれていることがある |

METI の加盟店手数料開示圧力（[最近の規制圧力](#最近の規制圧力)参照）は、まずカード類型を明示的に対象とする。カード類型では開示が最も進んでおり、JCB は 2023-06-01 の METI/JFTC リリース以降、イシュアとアクワイアラー間の配分率を開示している。

## イシュア手数料 / インターチェンジ — 誰が何を得るか

| 類型 | インターチェンジまたはイシュア側シェア | 位置づけ |
|---|---|---|
| カード（4者間） | Visa / Mastercard / JCB の日本標準インターチェンジ率は 2023 以降公表されている（Payments Japan ロードマップ 2023）。Payments Japan 2022 ロードマップは **Category I と Category II の双方でイシュア手数料 = 1.56%** と記録した | アクワイアラーは加盟店手数料からイシュアにインターチェンジを支払う。ブランドもスキーム手数料を取る |
| カード（3者間 Amex / Diners） | インターチェンジなし。ブランドがイシュア兼アクワイアラー | ブランドは PSP / プロセッサー費用控除後の加盟店手数料マージン全体を保持する |
| コード | カードの意味での正式なインターチェンジはない。代わりに、上流レイヤーへの**資金源手数料**がある。カード原資のコード決済はカード MDR を通し、銀行引落原資は銀行手数料を通し、プリペイド残高原資はマージンを内部に保持する | ウォレット運営者は残高原資のマージンを保持し、カード原資ではマージンがカード類型に流出する |
| A2A | 引落口座銀行へ支払われる銀行側処理費と JEPPO スキーム手数料 | 銀行が口座引落収益を保持し、加盟店関係は JEPPO が持つ |
| プリペイド | 外部インターチェンジなし。発行者が処理費控除後の加盟店手数料を保持する。価値がチャージされ未使用の場合は**マイナスフロート経済** | 発行者はフロート利回り + 未使用残高の失効益を得る |

4者間カード取引では、典型的な ~2.5% MDR のうち、約 1.5-1.6% のインターチェンジがイシュアに支払われ（日本の数値は Payments Japan 由来）、約 0.3-0.5% のブランド手数料があり、残りをアクワイアラーと PSP が分ける。ソースパックは [[payments/japan-interchange-and-merchant-fee-stack|interchange and merchant fee stack]] を参照。

## 精算サイクル

| 類型 | 加盟店精算 | 消費者請求 |
|---|---|---|
| カード | 伝統的には T+M（月次バッチ）。大手 PSP は追加料金で T+5 から翌日精算を提供する | カード会員は購入月の翌月中旬に請求される（典型的な 1-2 か月のイシュア信用フロート） |
| コード | ウォレットプランにより T+1 から T+M。PayPay Bank 連携精算は翌日も可能。ノンバンク精算は週次 / 月次 | 資金レールにより、銀行 / カード / 現金からのチャージは取引前または取引時に発生する |
| A2A | T+0 から T+1 （消費者側は即時引落、加盟店側は 1 日 2 回の全銀 / Cotra 精算を通じてバッチ入金） | 即時。取引時に銀行口座から引き落とされる |
| プリペイド | 発行者の加盟店契約により T+1 から T+M | 価値は事前チャージ済み。消費者はチャージ時点で「請求」される |

消費者が構造的な信用フロートを享受する唯一の類型はカードである。その他すべての類型は取引時またはその近くで引き落とされる。これが、コード決済が取引**件数**を支配しても、クレジットカードがキャッシュレス**金額**の過半をなお保つ大きな理由である。

## チャージバック対応

| 類型 | チャージバック / 紛争経路 | 消費者保護のレバー |
|---|---|---|
| カード | ブランドルールによるチャージバック（Visa Dispute、Mastercard Chargeback、JCB Dispute、Amex Dispute）。加盟店証拠はアクワイアラー → ブランド → イシュアを通る。理由コードは標準化されている | 強い。分割払いでは割賦販売法の抗弁の接続に加え、ブランドのチャージバックルールがある |
| コード | ウォレット運営者の紛争窓口。標準化されたウォレット間チャージバック枠組みはない。返金は通常、加盟店またはウォレットのカスタマーサポートを通じて交渉される | 弱い。ウォレット規約に依存し、法定チャージバック権はない |
| A2A | 銀行の不正引落枠組み（キャッシュカードは預金者保護法、Bank Pay は銀行 API ルール）。取消フローは銀行の紛争処理を通る | 不正には強い。加盟店紛争には弱く、消費者は加盟店へ直接請求する必要がある |
| プリペイド | 資金決済法上の終了時手続きに基づく発行者返金方針。加盟店紛争のチャージバックはない | 弱い。返金は個別の紛争購入ではなく、手段の終了時のみ |

チャージバックの強さは、カード決済が高額 EC、旅行、B2C サブスクリプションでなお標準である大きな理由である。加盟店が履行しない場合、消費者には戻る経路がある。

## 受け入れフットプリント

| 類型 | 国内オフライン | 国内オンライン | クロスボーダー |
|---|---|---|---|
| カード | 普遍的（POS、EMV 接触 / 非接触） | 普遍的 | あり。国際ブランドネットワーク |
| コード | 高い（コンビニ、飲食店、ドラッグストア、商店街）。MPM QR は導入が安い | 成長中だが、高額 EC ではカードほど普遍的ではない | 限定的。Alipay+、WeChat Pay との二者間連携はゲートウェイ PSP 経由 |
| A2A (Bank Pay / J-Debit) | 受け入れは限定的。スーパー、ドラッグストア、一部チェーンに集中 | 限定的 | なし。国内専用の銀行レール |
| プリペイド電子マネー | 交通（Suica / PASMO）、モール（WAON）、コンビニ（nanaco）で非常に高い。チャージ上限により低単価に偏る | 限定的（Suica internet、Edy の EC 利用） | なし。国内クローズドスキーム |

クロスボーダーレーンは、カード類型が実質的に唯一の地位を保つ領域である。国内専用の競争が多く存在しても、訪日観光客決済と海外向け消費者決済はいずれも国際ブランドネットワークに寄る。

## 認証 / 不正管理スタック

| 類型 | 取引時の主認証 | 二次管理 |
|---|---|---|
| カード | POS では EMV 接触 / 非接触 + PIN。EC では **EMV 3-DS**（2025-03 から METI / J-CSC ガイドライン 6.0により EC カード決済で義務化）。アプリ内ではトークン化 | 加盟店 / PSP の PCI DSS、非保持ルール、J-CSC 脆弱性スキャン、ブランド不正監視 |
| コード | ウォレットアプリへのデバイス紐付け + 生体認証（Face ID / Touch ID / Android 相当）+ 4桁 PIN。高額イベントでは OTP | ウォレット AML 監視、登録 / 上位階層移行時の KYC、取引速度制限、デバイス変更時の再認証 |
| A2A | 銀行アプリログイン + 生体認証 + 銀行側デバイス紐付け。銀行 API アクセスは電子決済等代行業契約で管理 | 銀行不正監視、日次引落上限、不正引落取消（預金者保護法） |
| プリペイド | 低額ではタップ時認証なし（Suica / PASMO の無人上限は通常 JPY 20,000）。高額またはチャージでは PIN | 発行者側不正監視、モバイルアプリ版（Mobile Suica、WAON app）の ID 紐付け |

EC における EMV 3-DS 要件は、単一で最大の 2025 管理変更である。[[payments/japan-card-security-authentication-controls|Japan card security and authentication controls]] で全体のガイドライン 6.0 タイムラインを参照。

## ロイヤルティ / ポイント経済

この次元は日本で特に複雑である。ポイントプログラムが、決済スキームレイヤーと重なりながらも一致しない**並行通貨レイヤー**として機能するためである。

| 類型 | 誰がポイントを負担するか | スキーム横断の交換可能性 |
|---|---|---|
| カード | イシュア負担（Rakuten Card → Rakuten Point、SMBC Card → V Point、JCB Card → Oki Doki Point、AEON Card → WAON POINT） | 共通ポイント（V Point、Rakuten Point、d Point、PayPay Point、Ponta）を通じて高い。[[loyalty/japan-points-landscape\|Japan points landscape]] を参照 |
| コード | プラットフォーム負担（PayPay 予算から PayPay Point、NTT docomo から d Point、KDDI から au PAY Point、Rakuten から楽天ポイント）。キャンペーン補助は歴史的に積極的 | V Point ↔ T Point（2024に統合）。PayPay Point と Rakuten Point はグループ内に閉じる。d Point と au PAY Point は選択的な横断加盟店受け入れを持つ |
| A2A | なし、または最小限（Bank Pay / J-Debit はポイント通貨を運営しない） | N/A |
| プリペイド | 発行者負担（WAON POINT、nanaco point、JRE POINT 経由の Suica リワード、Edy プログラム経由の Edy リワード） | 限定的。通常は所有グループ内で交換可能 |

ポイントレイヤーは**無料のマージンではない**。意図的な予算項目である。PayPay の長期補助フェーズ（2018-2022）と楽天のフライホイールはいずれも、決済スキームレベルの収益なしには持続しない可能性があるポイント経済に依存している。そのため [[payments/japan-code-payment-competitive-map|code-payment competitive map]] はキャンペーン費用比率を追跡する。

## スキーム横断の交換可能性と相互運用性

| レーン | 仕組み | 実態 |
|---|---|---|
| カードブランドの相互受け入れ | EMV + ブランドルール | 高い。あるブランドをアクワイアした端末は、そのブランドを世界的に受け入れる |
| コード決済のウォレット横断 | QR ゲートウェイ集約（Netstars JPQR、SBPS） | 部分的。加盟店は一つのゲートウェイで複数ウォレットを受け入れられるが、各ウォレットはなお自分のスキームを通る |
| プリペイド IC 相互利用 | 10 IC カード相互利用協定（Suica、PASMO、Kitaca、manaca、TOICA、SUGOCA、hayakaken、nimoca、ICOCA、PiTaPa） | 交通圏内では高い。加盟店側の受け入れは異なる |
| A2A 相互運用 | Cotra システムが預金取扱金融機関と資金移動業者を橋渡しする | 成長中。採用は参加アプリに依存する |
| ポイント横断償還 | 共通ポイント協定 + V Point ↔ T Point 統合 | 選択的。すべてのポイントが相互交換できるわけではない |

Suica / PASMO の 10 IC 相互利用スキームは、プリペイド類型で最も強い相互運用性の事例だが、交通圏のフットプリントに制約される。コード決済類型にはこれに匹敵するものがない。QR ゲートウェイは加盟店向けに受け入れを集約するが、ウォレット間の真の相互利用スキームは作らない。

## BOJ-NET / 全銀 / Cotra 依存

| 類型 | 精算インフラ依存 |
|---|---|
| カード | ネットポジション精算は最終的に銀行口座を通って移動する → 国内振込は全銀 → 銀行間最終決済は BOJ-NET |
| コード | ウォレット運営者は加盟店支払いを全銀振込で精算する。銀行口座からの利用者チャージは全銀または銀行 API を使う |
| A2A | 直接依存。Bank Pay / J-Debit は JEPPO サービスであり、Cotra は Zengin-Net が運営する |
| プリペイド | 加盟店支払いは全銀振込。チャージイベントは全銀または銀行 API |

すべての日本の決済スキームは、最終的に Zengin / BOJ-NET に依存するシステムである。どのスキームも最終的には商業銀行口座間で円を動かす必要があるためである。清算マップ全体は [[payments/japan-payment-clearing-and-settlement-infrastructure|Japan payment clearing and settlement infrastructure]] を参照。

## ステーブルコイン / EPI との重なり

トークン化されたマネーをめぐり、潜在的な第五のスキーム類型が出現しつつある。まだ大規模ではないが、法的ルートは定義された。

| トークン化マネー種別 | 隣接するもの | スキーム類推 |
|---|---|---|
| JPYC（EPI ルート上の電子決済手段） | プリペイド + 資金移動のハイブリッド | 経済性はプリペイドに近いが、オンチェーン移転可能性を持つ |
| Progmat coin（信託型ステーブルコイン） | 資金移動 / 銀行信託 | 銀行発行預金トークンの類似物 |
| SBI VC Trade 経由の USDC（電子決済手段等取引業 / ECISB ルート） | 外国発行 EPI、VASP 隣接取引所経由で取引 | クロスボーダー対応可能なスキーム類似物 |
| 預金トークン（DCJPY） | 銀行業 | トークン化された銀行預金 |

規制分類は [[fintech/japan-epi-three-types-overview|Japan EPI three types overview]] を参照。決済スキームの観点では、これらは第六の残高レイヤーを追加するが、加盟店受け入れフットプリントやチャージバック体制がまだ定義されていないため、上記の四類型マトリクスからは除外している。

## 最近の規制圧力

| 日付 | ソース | 変更点 |
|---|---|---|
| 2022-04-08 | JFTC | クレジットカード加盟店手数料に関する報告書。規模別の手数料ばらつきを記録し、インターチェンジ透明性の向上を求めた |
| 2023-06-01 | METI + JFTC | JCB がカード加盟店手数料のイシュア・アクワイアラー間配分率を開示（日本の大手ブランドで初） |
| 2023 | Payments Japan ロードマップ | Visa、Mastercard、UnionPay が日本のクレジットカード標準インターチェンジ率を公表 |
| 2024-2025 | METI 6.0 / 6.1 ガイドライン | EC カード決済で EMV 3-DS が義務化。非保持ルールが厳格化され、EC 加盟店の脆弱性スキャン義務が拡大 |
| 2025-2026 | METI キャッシュレス比率リリース | 2025 キャッシュレス比率 58.0% / JPY 162.7 trillion。手数料開示圧力はカード類型からコードおよびプリペイド類型へ広がる |
| 継続中 | JFTC | 加盟店手数料の不透明性、コードおよびプリペイド類型における単一アクワイアラーの価格支配力に関する定期調査 |

開示圧力は現在カード類型に集中している（公開データの深さも最大）が、キャッシュレス金額に占めるコード決済比率が上昇し続けるにつれ、コード決済手数料開示へも広がっている。

## 横断分析: 零細加盟店アクセス（商店街経済）

四つの類型は、市場の**3人未満の小規模加盟店**側、つまり商店街の個人商店で大きく異なる。

| 類型 | 小規模加盟店オンボーディング | ハードウェア費用 | JPY 500 取引の総コスト |
|---|---|---|---|
| カード | アクワイアラー / PSP 審査、KYC、一部プランの最低月額費、EMV 対応端末が必要 | JPY 0 （PSP 提供タブレットリーダー）から JPY 50,000+（フル POS） | MDR 約 JPY 12.50 から JPY 14.75  + 端末償却 + 月額費 |
| コード（MPM QR） | ウォレットアプリ口座 + 銀行口座、最小限の書類、印刷 QR ステッカー | JPY 0 （印刷 QR） | MDR 約 JPY 8 から JPY 14.75 。キャンペーン期間中は 0 の場合もある |
| A2A (Bank Pay) | 銀行口座 + JEPPO 加盟店契約 | JPY 0 （MPM 用印刷 QR） | 銀行側処理費 + 加盟店手数料 |
| プリペイド（Suica / WAON 等） | 発行者審査、FeliCa リーダーが必要 | JPY 30,000+ FeliCa リーダー | MDR 約 JPY 10 から JPY 15  + リーダーハードウェア |

コード類型の MPM QR 経済は、商店街の零細加盟店に**ほぼゼロのオンボーディング費用**を与える唯一のスキームである。このため、コード決済は過去のどのキャッシュレス化波よりも早く小規模加盟店セグメントに浸透した。カード類型は、ハードウェア費用と審査摩擦により、この端では構造的に不利である。プリペイド電子マネー類型は、ほとんどの零細加盟店が正当化できない FeliCa リーダーを必要とする。A2A の零細加盟店受け入れはまだ狭い。

この零細加盟店のコスト差は、**JFTC の加盟店手数料調査**が最も政治的に注目される領域でもある。METI のキャッシュレス政策は、小規模加盟店の手数料負担が、日本のキャッシュレス比率を 2025 基準値の 58.0% から 2030 目標の 65% へ動かすうえでのボトルネックであることを明示的に認めている。
