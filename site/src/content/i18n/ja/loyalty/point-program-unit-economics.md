# ポイントプログラムのユニットエコノミクス（日本のロイヤルティ原資・フロート・ブレッカジ・CPA）

## ウィキ上の位置づけ

この項目は [[loyalty/INDEX|loyalty index]] の配下に置く**ユニットエコノミクス／原資メカニクス**のページである。[[loyalty/point-liability-accounting-boundary|point liability accounting boundary]] の経済面における対をなすもので、あちらのページが*ポイントがどう分類されるか*を問うのに対し、このページは*プログラムが儲かるのか、そして誰が負担するのか*を問う。[[loyalty/japan-points-landscape|Japan points and loyalty landscape]] のプログラムマップ、[[loyalty/retail-media-points-data-loop|retail-media points data loop]] のデータ収益化フライホイール、[[loyalty/point-exchange-network-risk|point exchange network risk]] の事業者間移転メカニクスと合わせて読む。良好なポイント経済の川下にいる受益者はグループ金融である——[[payment-firms/rakuten-fg|Rakuten FG]]、[[megabanks/ndfg|NDFG]]、[[megabanks/paypay-fg|PayPay FG]]。

## 要約

ポイントプログラムの経済性は小さな方程式に帰着する——**原資 − 利用（償還）コスト − 運営コスト + ブレッカジ利得 + フロート利回り + 収益化（リテールメディア／金融クロスセル）= プログラムマージン。** 構造的に興味深い項は二つで、**ブレッカジ**（付与されたが決して利用されないポイントはほぼ純然たるマージン利得であり、その*タイミング*は ASBJ 企業会計基準第29号／IFRS 15 に支配される）と**フロート**（ポイント負債を付与してから利用に対して支払うまでのギャップは、ゼロ金利の調達残高である）である。プログラムが利益を生むかどうかは、目玉の還元率よりも、**誰がポイントを負担するのか**（加盟店か事業者か）、**ブレッカジがどれほど高く出るか**、そして**ID グラフが収益化されているか**にかかっている。「1% 還元」のプログラムは一つの経済的対象ではなく、それらの答え次第で少なくとも四つの非常に異なるものになる。

## ユニットエコノミクスの方程式

付与されたポイントの代表的な一単位について、事業者の経済性は次のとおりである。

| 項 | 符号 | ドライバー |
|---|---|---|
| 原資の流入 | + | 加盟店負担型ポイント：加盟店が事業者に発行料を支払う。事業者負担型：純然たるコスト。 |
| 利用（償還）コスト | − | 実際に利用されたポイントの価値（事業者はそれを履行しなければならない） |
| 運営コスト | − | プラットフォーム、決済、不正対策、データインフラ、パートナー管理 |
| ブレッカジ利得 | + | 決して利用されないポイント → 負債が収益に振り替えられる（タイミングは会計基準による） |
| フロート利回り | + | 未利用残高は、償還まで保有される無利息の残高である |
| 収益化 | + | ID グラフに帰属するリテールメディア＋金融クロスセル収益 |

目玉の「還元率」は*グロスの*利用コストを定めるにすぎない。マージンは残り五つの項によって決まる——だからこそ、同じ還元率を掲げる二つのプログラムが正反対の経済性を持ち得るのである。

## 原資：実際に誰がポイントを負担するのか

最大の経済的分岐は**原資（誰が負担するか）**であり、それは消費者が目にするものとは一致しない。

| 原資モデル | 誰が負担するか | 事業者の経済性 | 典型例 |
|---|---|---|---|
| 加盟店負担型 | パートナー加盟店が発行ポイントごとに事業者へ支払う | 事業者は発行時にマージンを得て、さらにブレッカジ／フロートを留保する | パートナー小売店での共通ポイント利用 |
| 事業者負担型（キャンペーン） | 事業者自身のマーケティング予算 | 純然たる獲得コスト。データ／クロスセル経由でのみ回収される | ウォレットの「100% 還元」プロモーション |
| イシュアー負担型（カード） | カードイシュアーがインターチェンジ／手数料から原資を出す | カード経済性を防衛する。リワードはインターチェンジのコスト | クレジットカードのリワードポイント |
| 自己負担型（グループ小売） | グループ自身の小売マージン | リテンション支出。リピート購入として回収される | 店舗グループのポイント（食品スーパー／コンビニ） |

共通ポイントは通常、**加盟店負担型**のコア（発行時にマージンがプラスで、*さらに*ブレッカジとフロート）を回しつつ、その上に**事業者負担型**のキャンペーンを重ねる。両者を混同するのは、[[loyalty/point-liability-accounting-boundary|the accounting-boundary page]] で指摘される典型的な読み違いである——キャンペーンの上乗せは即座に P&L を直撃してプログラムを赤字に見せる一方、加盟店負担型のベースは静かにマージンがプラスなのである。

## ブレッカジ：マージンのエンジン

**ブレッカジ**——付与されたが決して利用されないポイント——は、ロイヤルティにおいて純粋なマージン利得に最も近い存在である。仕組みとしては、

- 事業者が決して履行する必要のないポイントは、最終的に収益へ振り替えられる負債である。
- **ASBJ 企業会計基準第29号**および **IFRS 15** の下では、その振替は予想償還期間にわたる*償還パターンに比例して*認識される——失効時に一括ではなく——しかも、ブレッカジが履歴から**信頼できる見積り**となる場合に限られる。見積れない場合、収益はさらなる償還が見込み薄になるまで待つ。
- ブレッカジが高い＝構造的なマージンが高い。しかしそれはプログラム全体で最も**見積りに敏感で、レビューにさらされる**数値でもある。過大計上は収益を前倒しする。キャッシュレス推進協議会／ペイメントジャパンの開示規範（比較可能な償還率、失効、未利用残高の報告）が存在するのは、一つにはブレッカジが収益平準化の弁として使われるのを防ぐためである。

設計上の選択はブレッカジを直接押し上げる——短い有効期限、用途限定ポイント（期間・用途限定）、高い最低利用しきい値はいずれもブレッカジを*高める*——これは（ほとんど明言されないが）用途限定ポイントがこれほど広く普及している理由の一つである。これの会計上の枠組みは [[loyalty/point-liability-accounting-boundary|the accounting-boundary page]] のバケット 3 にあたる。

## フロート：無利息の調達残高

ポイントを付与してからその利用に対して支払うまでの間、事業者はまだ決済する必要のない**未利用負債**を保有する。総体としてこれは、保険フロートやプリペイド手段のフロートに類似する、大規模で、緩慢に動き、無利息の残高である。

- 平均償還までの時間が長いほど、発行に対するフロートは大きくなる。
- 大規模な共通ポイントやウォレットにとって、未利用のポイント負債は、ゼロ金利で完全に顧客によって賄われる定常残高である。
- フロートはブレッカジとは*別物*である——ブレッカジは決して戻ってこないポイント（マージン利得）、フロートはいずれ戻ってくるポイントに対する保有中のキャリー（資金面の便益）である。

このフロートの読み方こそ、金融グループにとってポイント負債を魅力的にするものである——銀行／通信グループ内のポイント事業者（[[payment-firms/rakuten-fg|Rakuten FG]]、[[megabanks/ndfg|NDFG]]、[[megabanks/paypay-fg|PayPay FG]]）は、顧客によって賄われる残高と、日次頻度の接点とを同時に貢献するのである。

## 収益化：薄いプログラムが利益化する場所

原資 − 償還 − コストだけで判断されたポイントプログラムは、限界的に見え得る。それをひっくり返す項が、**ブレッカジ**、**フロート**、そして **ID グラフの収益化**——[[loyalty/retail-media-points-data-loop|the retail-media data-loop page]] で示されるリテールメディア＋金融クロスセルのフライホイール——である。これが「赤字」ウォレットの整理にあたる。

- キャンペーン主体のウォレットは、事業者負担型の付与を即時費用として計上する（P&L は悪く見える）。
- 同じ支出が、最もリッチな ID グラフと最も深い日次頻度のファネルを買う。
- リテールメディアのマージンと金融クロスセル——ポイントが補助していた薄い小売／決済事業よりも高マージン——こそ、それが回収される場所である。

したがって収益性の問いは決して「還元率は何か」ではない。それは「加盟店負担型か事業者負担型か、ブレッカジはどれほど高いか、フロートはどれほど大きいか、グラフは収益化されているか」である。

## なぜこれが JapanFG／金融分析にとって重要か

- **還元率は見出しとして誤っている。** 二つの「1% 還元」プログラムは、原資、ブレッカジ、フロート、収益化の点で全く異なる。比べるべきはそれらであり、パーセンテージではない。
- **ブレッカジは収益の質を示すフラグである。** マージンを見せるために楽観的なブレッカジに頼るプログラムは収益を前倒ししている。「ポイント引当金」が「契約負債」へ移行したか、償還の前提がどう開示されているかを確認すること（[[loyalty/point-liability-accounting-boundary|accounting boundary]] に従う）。
- **フロート＋ID グラフこそ、金融グループがポイント事業者を欲しがる理由である。** グループ内のポイントは、無利息で顧客によって賄われる残高*と*クロスセルのファネルを供給する——[[megabanks/smfg|SMFG]] ／ V-Point、[[megabanks/ndfg|NDFG]] ／ dポイント、[[payment-firms/rakuten-fg|Rakuten FG]] の内部統合をマーケティングを超えて合理化する経済性である。
