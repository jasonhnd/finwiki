# T-Point + V-Point post-2024 merger — デュアルブランド移行、CCCMK Holdings、ポイント負債移転

## ウィキ上の位置づけ

この項目は [[loyalty/INDEX|loyalty index]] の配下に置かれる。T-Point + V-Point 統合の **post-2024-04-22 運用詳細ページ**であり、構造的な位置づけを記録する戦略項目 [[loyalty/v-point-smbc-ccc-case|V Point (SMBC × CCC) bank-led common-point case]] を補完する。エコシステムマップは [[loyalty/japan-points-landscape|Japan points and loyalty landscape]]、負債移転メカニクスは [[loyalty/point-liability-accounting-boundary|point liability accounting boundary]]、その IFRS 15 / ASBJ 29  の枠組みもあわせて読む。通信会社アンカー型ピアは [[loyalty/d-point-detailed-ecosystem|d Point detailed ecosystem]]、ウォレットアンカー型ピアは [[loyalty/sb-yahoo-paypay-points-unified-ecosystem|SoftBank / Yahoo / PayPay points unified ecosystem]]、第三の連合モデルは [[loyalty/ponta-points-deep-dive|Ponta points deep dive]]、親金融持株の文脈は [[JapanFG/smfg|SMFG]]、発行カード法人は [[JapanFG/smbc-card|SMBC Card (SMCC)]] とあわせて読む。

## 要約

**2024-04-22年の T-Point + V-Point merger** は、日本でこれまで最大の共通ポイント統合だった。**CCC Marketing の T-Point 共通ポイント事業と SMBC group の V-Point クレジットカード報酬プログラムは V-Point ブランドの下で統合**され、**CCCMK Holdings（2023-06に設立された SMFG-CCC 合弁）**が運営する。戦略的意図は、T-Point の約 70 百万枚規模の物理加盟店ネットワーク（TSUTAYA、FamilyMart 旧来網、ENEOS、ドラッグストア、飲食店）と、SMBC の銀行カード発行基盤（SMBC Card / SMCC）を組み合わせ、楽天（e-commerce）、dポイント（telco）、PayPay Points（wallet）に対する **銀行主導の共通ポイント対抗軸** を作ることだった。統合は実際に運用上の統合であり、T-Point の共通ポイント事業ブランドは 2024-04-22 に終了した。ただし **デュアルブランド移行の現実** は残る。TSUTAYA ブランドの T カードは移行期間中も V-Point が貯まるカードとして機能し続け、TSUTAYA 発行カードは参加加盟店で引き続き読み取れる。SMBC 発行の V カード（Olive一体型、SMBC Card）は、旧 CCC 発行カードと同じ V-Point ブランドの下で併存する。統合時の **ポイント負債移転メカニクス** は IFRS 15 / ASBJ 29  の枠組みに従った。旧 T-Point 負債は各発行パートナーのバランスシートにあり、変換比率は契約で定義され、受け入れ側運営者（CCCMK 下の V-Point）は旧残高の移行に伴い新たな負債を認識した。PayPay Points は構造的に異なるアンカー（カード / 銀行ではなくウォレット）から運営されており、この比較により V-Point の競争上限と下限が異なる形で規定される理由が見える。

## 統合とデュアルブランド移行のタイムライン

| 日付 | 事象 |
|---|---|
| 2003-10 | T Point が CCC の全国共通ポイントとして開始。TSUTAYA、FamilyMart、ENEOS がアンカーパートナー |
| 2020 | Vポイントが SMBC Card (SMCC) 独自のクレジットカード報酬ポイントとして開始（多くの保有者にとってワールドプレゼントを置換） |
| 2022-10 | CCC と SMFG / SMBC が、T Point と V Point を一つの共通ポイントブランドに統合する意向を発表 |
| 2023-03 | CCC が T Point 運営を SMFG-CCC 合弁ビークル内の CCC Marketing 構造へ再編 |
| 2023-06 | CCCMK Holdings 設立（SMFG 側が CCC Marketing 運営へ資本参加） |
| 2023-10 | Olive Account が V-Point を中核ロイヤルティ層として開始し、統合の土台を作る |
| 2024-01-09 | CCC / SMBC 共同プレスリリースが 2024-04-22 の統合日と V-Point へのブランド集約を確認 |
| **2024-04-22** | **T Point と Vポイントが「Vポイント」ブランドの下で統合**；T-Point 共通ポイントのブランドは終了；提携加盟店ネットワークは V-Point 受入へ移行 |
| 2024-06 onward | TSUTAYA は参加加盟店で V-Point が貯まる物理的な「Tカードデザイン」カードを発行継続 — デュアルブランドの現実 |
| 2025 | 旧 T カード保有者の V カードデザインへの段階的移行；提携加盟店での受入拡大 |
| 2026 (ongoing) | 長い裾野の T カード保有者は V-Point を獲得し続ける；全面的なデザイン移行は初期スケジュールを越えて続く |

2024-04-22 の日付は **ブランドレベルの統合** を示すが、**運用上の現実はデュアルブランドの長い裾野** である。TSUTAYA / CCC は T カードデザインにブランド価値を持つ小売事業を運営しているため、TSUTAYA ブランドは旧カードと新規発行の T カードデザインカードに残る。一方、SMBC 発行カード（V-Point ブランドの Olive一体型と SMBC Card）は V-Point ブランドを掲げる。どちらのカード類型も同じ統合 V-Point 台帳にポイントを貯める。

## CCCMK Holdings — 合弁ビークル

**CCCMK Holdings (CCC Marketing Holdings)** は、統合 V-Point 事業の運営ビークルである。公開構造は次のとおり。

| レイヤー | 役割 |
|---|---|
| CCC (Culture Convenience Club) | 創業者であり歴史的な共通ポイント運営者；旧 T-Point 運営者 |
| SMFG / SMBC | 銀行側の資本パートナー；SMBC Card (SMCC) 発行と Olive Account 連携を持ち込む |
| CCCMK Holdings | 統合 V-Point 共通ポイント運営を保有する合弁ビークル。2023-06 設立 |
| Vポイント株式会社 | CCCMK Holdings の下で V-Point を運営する具体的な事業会社 |

資本提携構造は、CCC のデータマーケティング事業の一部を SMFG-CCC 傘下へ移しつつ、CCC の小売（TSUTAYA など）におけるブランド存在感を維持する。SMFG による CCCMK Holdings への出資は SMFG の IR 資料で公開されている。見出しとしては、銀行側アンカーがデータマーケティング権益と並んでロイヤルティ事業の実質的な経済的利害を獲得したという点である。

## デュアルブランドの仕組み — TSUTAYA ブランドと SMBC ブランドの違い

2024-04-22 の統合は **ブランドレベル** の集約だったが、運用上は顧客向けカードが出自ブランドを保持している。

| カード種別 | 発行者 | デザイン / ブランド | 獲得面 |
|---|---|---|---|
| 旧 T カード (pre-2024) | CCC (TSUTAYA / FamilyMart 旧来網など) | T カードデザイン | 統合後は参加加盟店で V-Point が貯まる |
| 新 TSUTAYA T カード (post-2024) | CCC / TSUTAYA | T カードデザイン | V-Point が貯まる |
| V-Point Card (CCCMK 発行) | CCCMK Holdings | V-Point デザイン | V-Point が貯まる |
| Olive一体型 (SMBC 発行) | SMBC | SMBC / Olive デザイン | 銀行口座連携付きで V-Point が貯まる |
| SMBC Card / SMCC | SMBC Card | SMBC Card デザイン | クレジットカード料率で V-Point が貯まる |

含意として、顧客は上記いずれのカード形態を持っていても、同じ V-Point 台帳にポイントを蓄積できる。このデュアルブランドの現実は、CCC が TSUTAYA 小売を継続運営していることと、70百万枚規模のカード基盤では旧カードの置き換えに何年もかかるという実務上の事実を反映している。

## PayPay Points との比較

| 観点 | V-Point (post-2024 merger) | PayPay Points |
|---|---|---|
| アンカー類型 | 銀行主導 + カード主導 + 旧 CCC 小売ネットワーク | ウォレット主導 + LY データ + SoftBank telco |
| 運営者 | CCCMK Holdings (SMFG-CCC JV) | PayPay 株式会社 (SoftBank + LY) |
| ID の起源 | カード / 銀行口座 (SMBC Card, Olive, CCCMK card) | PayPay アプリログイン (SMS-OTP、しばしば telco 連携) |
| ウォレット連携 | V NEAR PAY (限定的 NFC), Olive 連携 | PayPay (支配的な QR ウォレット；[[payments/japan-code-payment-operator-2025-market-share-matrix|2025 share matrix]] 参照) |
| クレジットカード連携 | SMBC Card, V-Point Card, Olive一体型 | PayPay Card (SoftBank / LY) |
| 銀行連携 | SMBC + Olive | PayPay Bank (旧 JNB) |
| 証券連携 | SBI証券 cooperation (SMBC × SBI alliance) | PayPay Securities |
| 会員 ID 基盤 | 統合時点で 130M+ unified (CCC + SMBC sides) | 70M+ PayPay registered users |
| 小売受入 | CCC の物理加盟店ネットワークを継承 | PayPay 加盟店基盤による全国 QR 受入 |
| 自社商品での獲得経済性 | カード利用料率；Olive ボーナス階層 | ウォレット利用料率（キャンペーン資金） |
| 投資商品との接続 | SBI証券 の投信購入に V-Point を利用 | PayPay 証券で PayPay Points を利用 |
| キャンペーン様式 | 銀行型の構造化プロモーション（低めの見出し料率、持続的な経済性） | ウォレット型の頻繁なキャンペーン（高い見出し料率、breakage 資金） |
| データグラフ | CCC 小売データベース + SMBC 顧客フロー | LY (Yahoo Japan + LINE) データ資産 + PayPay 取引フロー |

構造比較から、二つの異なる金融流通アーキタイプが見える。**V-Point の強み** は、SBI alliance を通じた **投資 / NISA クロスセル** と、Olive による **銀行口座のメイン口座アンカー** である。**PayPay Points の強み** は、高い MAU を生む頻繁なキャンペーンを伴う **ウォレット活動の moat** である。両者が最も直接競合するのは、コンビニエンスストア、ドラッグストア、飲食店などの **小売加盟店でのポイント獲得面** であり、V-Point の CCC 由来ネットワークと PayPay の全国 QR 受入が重なる。

Yahoo ポイントおよび SoftBank Smart Login と PayPay Points のより広い統合については [[loyalty/sb-yahoo-paypay-points-unified-ecosystem|SB/Yahoo/PayPay unified points]] を参照。

## ポイント負債移転メカニクス — IFRS 15 / ASBJ 29  の枠組み

2024-04-22 の統合は、重要な **ポイント負債移転の問題** を生んだ。統合前の T-Point または V-Point 負債を持つ各運営者は、移行を処理する必要があった。

| 事象 | 会計処理 |
|---|---|
| 統合前の CCC 帳簿上の T-Point 負債 | CCC / パートナーのバランスシート上で契約負債 (IFRS 15) または引当負債 (ASBJ 29) として認識 |
| 統合前の SMBC Card 帳簿上の V-Point 負債 | SMBC Card / SMCC 帳簿上で契約負債として認識 |
| 統合イベント (2024-04-22) | 旧台帳から CCCMK Holdings の統合 V-Point 台帳へ負債が移行。変換比率と精算メカニクスを定義 |
| 受け入れ側運営者 (CCCMK) | 旧残高の移行に伴い新たな負債を認識し、以後の交換義務を負う |
| 移行期間 | 移転された負債残高について、CCC / パートナーと CCCMK / SMBC の間で二者間ネット精算 |
| プログラム間交換 | V-Point ↔ ANA mileage、V-Point ↔ partner programs は調整後の変換率で継続。精算レッグの仕組みは [[loyalty/point-liability-accounting-boundary]] に記載 |

移行のクリーンな形は、**1 T-Point = 1 V-Point で変換**され、受け入れ側の CCCMK / SMBC エンティティが交換義務を引き受けるというものである。摩擦のある形では、一部の旧パートナーが参加条件を再交渉し、長い裾野の加盟店の一部は移行せず、キャンペーン付与残高の一部は有効期限ルールが異なるためルール調整が必要になった。

統合後の SMCC / SMFG IR 資料を読むアナリストにとっての開示上の手がかりは、**「ポイント引当金」が JGAAP の科目として残っているのか、それとも「契約負債」(contract liability) に移行しているのか**を追うことである。この移行は、会社が IFRS 15 / ASBJ 29  の繰延収益モデルを完全に採用したかどうかを示す。CCCMK Holdings は個別上場会社ではないため開示は限定的だが、SMFG レベルのセグメント注記は部分的な可視性を提供する。

## SBI alliance — 銀行主導の差別化としての投資商品接続

**SMBC × SBI alliance** は、V-Point にとって最も特徴的なクロスセル経路である。顧客は貯めた V-Point を **SBI証券で投資信託の購入** に使うことができ、これは SMBC に紐づく顧客にとって dポイントや PayPay Points エコシステムには直接対応物がない。仕組みは次のとおり。

| 仕組み | 説明 |
|---|---|
| 対象顧客 | V-Point と連携した SBI証券口座保有者 |
| 対象商品 | 選定された投資信託口数 (主要な投資信託銘柄) |
| 変換 | 1 V-Point = 投信購入に充当する 1 円 |
| 税務処理 | 購入後は通常の NISA / 特定口座処理が適用 |
| 精算 | V-Point 負債は SBI証券の投信購入注文に対して消滅 |

戦略的意図は、V-Point を **ほぼ現金に近い投資資金通貨** にし、高 MAU のウォレットポイント層の外側にいる高所得 / 資産形成世帯にとっての V-Point の魅力を高めることにある。これは、通信会社主導およびウォレット主導のピアに対する、銀行主導の共通ポイントの構造的差別化である。
