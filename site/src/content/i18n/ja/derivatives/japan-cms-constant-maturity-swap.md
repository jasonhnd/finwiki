# 日本 CMS（コンスタント・マチュリティ・スワップ）市場

## 要約

コンスタント・マチュリティ・スワップ（CMS）は、一方のレッグが固定参照（または変動短期金利参照）を支払い、他方のレッグが定期的にリセットされる長期テナーのスワップレート — 最も一般的には各リセット日の 10年または 5年 JPY スワップレート — を支払う OTC 金利デリバティブである。CMS レッグの決定的な特徴は、各リセットで参照されるスワップレートが一定の満期（例：各リセットで 10年）を持つことであり、一方バニラ IRS では変動参照が一定の短期テナー（例：3カ月 TIBOR または複利 TONA）を持つ。

CMS は、日本で販売されるイールドカーブの形状にリンクした各種の仕組み商品の構造的な構成要素である。CMS リンク債（10年 JPY スワップレートにインデックスされたクーポンを支払う）、CMS スプレッド債（10年マイナス2年スプレッド、または類似のカーブスティープネス指標にインデックスされたクーポンを支払う）、CMS キャップ付き / フロア付きフローターである。CMS の価格設定は、CMS レートのペイオフが原資産であるフォワードスワップレートに対して非線形であるため、コンベキシティ調整を必要とする。コンベキシティの補正は原資産であるスワップレートのインプライドボラティリティに依存し、[[derivatives/japan-swaption-market|JPY swaption vol grid]] を直接利用する。

FinWiki にとって、このエントリは CMS のメカニクス、コンベキシティ調整、JPY のユースケース（CMS リンク債、CMS スプレッド商品）、価格設定インプット、そしてディーラーフランチャイズをカバーする。

## ウィキ上の位置づけ

このエントリは、金利デリバティブクラスターの [[derivatives/INDEX|derivatives index]] の配下にある。CMS レートが導出されるバニラ IRS のベーシスについては [[derivatives/japan-irs-market]]、コンベキシティ価格設定を駆動するボラグリッドのインプットについては [[derivatives/japan-swaption-market]]、ディスカウントカーブについては [[derivatives/ois-tona-curve]] と照らし合わせて読むこと。仕組み債の販売チャネルは [[derivatives/structured-bond-japan-retail-issuance]] と [[derivatives/structured-product-eb-knockin-japan-retail]] でカバーされる。

## 商品のメカニクス

標準的な JPY CMS は以下の構造を持つ：

| 要素 | 詳細 |
|---|---|
| 想定元本 | 定義される。交換されない |
| テナー | スワップの総テナー（例：5年の総満期） |
| リセット頻度 | 定期的（通常は四半期または半年） |
| 固定 / スプレッドレッグ | 固定レート（ディーラーが価格設定する「CMS スワップレート」）または変動短期金利 + スプレッドを支払う |
| CMS レッグ | 各リセット日に現行の N 年 JPY スワップレートを支払う（例：各リセットで観測される 10年 JPY スワップレート） |
| 日数計算 | JPY では通常 ACT/365  |
| 決済 | 各クーポン日のネット支払 |
| 文書 | ISDA マスター + CSA |

例：5年 CMS-10年スワップは、一方のレッグで 10年 JPY スワップレート（各四半期リセットで固定）を、他方のレッグの固定レートまたは 3カ月 TIBOR / 複利 TONA + スプレッドに対して支払う。

経済的内容：CMS レッグの支払者は、時間を通じた長期スワップカーブの水準（および形状）に対するビューを取っている。CMS レッグの受領者は、反対方向にヘッジまたは投機している。

## なぜコンベキシティ調整が必要か

将来の任意のリセット日における CMS レートは、その日に開始する N 年スワップのパースワップレートである。CMS レッグのペイオフの PV は、原資産であるフォワードスワップレートに対して非線形である。なぜなら、CMS レッグのクーポンで「支払われる」スワップレートは、それ自身の PV（その日にそのレートでエントリーした場合の原資産 N 年スワップの PV）がそれ自身のレートに非自明に関連するスワップを参照して計算されるからである。

標準的な価格設定アプローチは CMS ペイオフを分解する：

1. 現在のディスカウントカーブを用いて、リセット日 $T$ における N 年原資産スワップのフォワードスワップレート $S(t, T, N)$ を計算する。
2. フォワードスワップレートにコンベキシティ調整を適用する：$\hat{S} = S + \text{convex. adj.}$。調整は原資産であるスワップレートのインプライドボラティリティ（[[derivatives/japan-swaption-market|swaption vol grid]] から）と原資産スワップのテナー構造に依存する。
3. $\hat{S}$ を価格設定のための CMS クーポンの期待値として用いる。

クローズドフォームの近似（Hagan、Brigo-Mercurio など）が広く用いられる。マルチファクター短期金利モデル（Hull-White、LMM）は、複雑なカーブレジームにおいてより正確なコンベキシティ調整を与える。

実務上の帰結：CMS の価格設定は、特に参照されるスワップレートのテナー（例：CMS-10年商品に対する 10年ボラ）において、スワップションのボラサーフェスに実質的に敏感である。これが、CMS 市場とスワップション市場がディーラーのブックで緊密に結合している理由である。

## JPY のユースケース

JPY の CMS 商品はいくつかのエンドユーザー目的に役立つ：

| 商品 | 構造 | エンドユーザーの魅力 |
|---|---|---|
| CMS リンク債（クーポン = a + b × CMS-10年） | 10年 JPY スワップレートに連動する定期クーポン | バニラフローターに対する利回り向上。長期金利が上昇するというビュー |
| CMS スプレッド債（クーポン = a + b × (CMS-10年 − CMS-2年)） | カーブスティープネスに連動するクーポン | カーブがスティープ化すれば利回りピックアップ。カーブフラット化に対する保険 |
| CMS キャップ付きフローター | CMS レートを参照するキャップを持つ変動クーポン | 変動債の上限を定義する。利上げ懸念が存在するときに人気 |
| CMS フロア付きフローター | CMS レートを参照するフロアを持つ変動クーポン | 低金利環境での下限を定義する |
| レンジアクルーアル CMS | CMS レートが定義されたレンジ内にあるときのみクーポンが発生 | カーブレンジの方向性ビューでの利回り向上 |
| スノーボール CMS | 過去の CMS レート観測に基づいてクーポンが累積的に増加 | 高度に経路依存。散発的な販売 |

日本における CMS リンク債のエンドユーザーは以下にわたる：

- **地方銀行**：バニラクーポン収入が不十分なときの JPY 債券ポートフォリオの利回り向上。
- **信託銀行 / 年金基金**：より広範な債券マンデートの一部としてのカーブ形状連動配分。
- **富裕層リテール（プライベートバンキング経由）**：向上したクーポンを支払う CMS スプレッド債の販売。
- **生命保険会社**：ALM ポートフォリオの構成要素としての選別的な利用。
- **企業財務**：限定的な直接利用。バニラ IRS への依存がより大きい。

ボリュームは散発的であり、JPY スワップカーブがフラットイールドの代替に対して魅力的な形状を呈する期間に集中する。

## 価格設定インプット

JPY 向けの完全な CMS 価格設定エンジンには以下が必要である：

| インプット | ソース |
|---|---|
| JPY OIS-TONA ディスカウントカーブ | [[derivatives/ois-tona-curve|TONA-OIS curve]] |
| JPY スワップフォワードレートカーブ | [[derivatives/japan-irs-market|JPY IRS market]] からのブートストラップ |
| JPY スワップションインプライドボラティリティサーフェス | [[derivatives/japan-swaption-market|JPY swaption vol grid]]（エクスパイリー × テナーのマトリクス） |
| 相関の仮定（CMS スプレッド商品向け） | テナー間相関。ヒストリカルまたはオプションインプライドのデータから逆算可能 |
| 信用・資金調達スプレッド（ディーラー自身のバランスシート向け） | xVA フレームワーク（FVA、CVA、DVA） |

CMS スプレッド商品は、2 つのスワップレート（例：10年 vs 2年）間の相関に特に敏感である。なぜなら、スプレッドのペイオフはいずれのテナー単独よりもボラティリティが低いからである。相関の誤った価格設定は商品を実質的に誤評価しうる。

## ディーラーフランチャイズ

JPY CMS のマーケットメイクは、以下のより広範な金利仕組みデスクの中に位置する：

| ディーラーカテゴリ | 活動 |
|---|---|
| 日本のメガバンク証券系列（MUFG 証券、SMBC 日興、みずほ証券） | バンキングチャネルおよび証券チャネルの顧客への販売のための CMS リンク債と CMS スプレッド債の組成。スワップションブックに対する CMS リスクの在庫保有 |
| 独立系日本証券（野村、大和） | リテールおよび機関の販売を伴う仕組み商品の組成 |
| グローバル投資銀行（JPMorgan、Goldman、Citi、Deutsche、BNP、Barclays、HSBC） | 機関フロー向けの CMS マーケットメイク。クロスボーダー販売。クロスカレンシー CMS スプレッド商品 |
| インターディーラーブローカー | 限定的。CMS は IDB 取引よりもディーラー対エンドクライアントの傾向 |

CMS 商品からのディーラー損益は、IR におけるより広範な金利 / 仕組み商品ラインの一部であり、個別には開示されない。

## 流動性と市場の厚み

| テナー / 構造 | 流動性 |
|---|---|
| バニラ CMS（例：5年 CMS-10年） | 中程度。ディーラーはリクエストに応じて気配を出す。ビッドアスクはバニラ IRS より広い |
| CMS リンク債（発行体側） | 散発的。リテール / 機関の需要に依存 |
| CMS スプレッド債 | 散発的。相関に敏感な価格設定のため、ディーラーはエクスポージャーを厳密に管理する |
| バミューダンコーラブル CMS | 限定的。あつらえ。主にディーラー対発行体 |

JPY CMS 市場は、EUR CMS 市場（カーブスティープネス商品がはるかに大きくより発達した仕組み販売基盤を持つ）よりも実質的に小さく、USD CMS よりも小さい。散発的な販売パターンは、カーブの形状が魅力的なペイオフプロファイルを呈するときに CMS のボリュームが急増することを意味する。

## 清算

JPY CMS は主にバイラテラルである。JSCC は、バニラ IRS に対してほど広く清算スコープを CMS に拡張していない。非清算の CMS 取引は、対象カウンターパーティについて UMR フェーズの IM 要件と標準的な CSA 担保化の対象となる。
