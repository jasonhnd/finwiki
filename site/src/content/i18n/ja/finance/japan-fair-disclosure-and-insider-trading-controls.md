# 日本のフェア・ディスクロージャーとインサイダー取引管理

## 概要

日本のフェア・ディスクロージャーおよびインサイダー取引管理の記録は、選択的開示、適時開示、未公表の重要事実、発行体の情報管理、インサイダー取引規制、取引所開示システムを接続する。公開ルートには、FSA のフェア・ディスクロージャー資料、金融商品取引法本文、JPX / TSE の適時開示ページ、TDnet が含まれる。

このページは [[finance/INDEX|finance domain]] に属し、 [[securities/INDEX|securities domain]], [[securities/tokyo-stock-exchange|Tokyo Stock Exchange]], [[financial-regulators/japan-exchange-group|Japan Exchange Group]], [[finance/japan-large-shareholding-disclosure|large-shareholding disclosure]], [[finance/japan-tender-offer-process|Japan tender offer process]], [[securities/japan-ipo-listing-disclosure-route|IPO listing disclosure route]], [[securities/japan-market-infrastructure-map|Japan market infrastructure map]], and [[securities/japan-underwriting-market-structure|Japan underwriting market structure]].

## フェア・ディスクロージャー・ルールのフィールド

| フィールド | 公開事実ルート |
|---|---|
| 法定根拠 | FIEA 第 27-36条。 |
| FSA の説明 | 投資者への公平な情報開示を確保するために導入されたルール。 |
| トリガー・ルート | 上場会社が特定の業務関係者に未公表の重要情報を提供する。 |
| 同時開示ルート | 例外条件が適用されない限り、情報伝達と同時に開示する。 |
| 守秘 / 取引禁止の例外 | 受領者が法令、規則、契約上の守秘義務および取引禁止義務を負う。 |
| 速やかな開示ルート | 指定された意図しない伝達または非同時伝達を認識した後、速やかに開示する。 |
| 漏えい / 取引ルート | 守秘 / 取引禁止義務を負う受領者が開示前に漏えいまたは取引した場合、速やかに開示する。 |
| 重要情報フィールド | 有価証券の価値に重要な影響を及ぼす可能性がある未公表の具体的情報。 |
| モザイク情報フィールド | FSA 指針は、モザイク情報は通常それ自体では対象情報ではないと示す。 |
| 内部統制フィールド | FSA 指針は、実務上の統制手段として内部規程と従業員研修を挙げる。 |

## インサイダー取引の公表フィールド

FIEA 第 166 条は、会社関係者のインサイダー取引規制に関する公開された法令上のルートである。Japanese Law Translation の資料では、当該事実が指定開示書類に記載されている場合、政令で定める公表措置または第 25 条に基づく指定開示書類の公衆縦覧を通じて `Publicized` が定義されている。

JPX の TDnet ページは、Company Announcements Disclosure Service / `適時開示情報閲覧サービス` を通じた開示により、重要事実および一定の公開買付け等事実に関する法令上の公表措置が完了すると説明している。同ページによれば、TDnet の公表情報は開示日を含めて 31 日間、公衆縦覧サービスで閲覧でき、過去の開示資料は上場会社検索で 10 年間確認できる。

## TDnet の公開閲覧面

| 表面 | 公開フィールド |
|---|---|
| TDnet | 開示資料の TSE 提出、TSE への事前説明、公表、報道機関配信、データベース登録を扱う電子ワークフロー。 |
| 会社公告開示サービス | TDnet 掲載の適時開示を公衆縦覧するサイト。31日間の公開表示フィールド。 |
| 上場会社検索 | 過去の適時開示資料。JPX TDnet ページに記録される 10年フィールド。 |
| TDnet データベースサービス | 過去の開示データを取得する有料データベース・ルート。 |

## 法的公表と実務上の情報ギャップ

法定公表ルートは、FIEA / 政令の公表措置フレームワークおよび TDnet の公表措置フィールドと結びつく。JPX のインサイダー取引公開資料も、正式公表直後の期間について実務上の情報ギャップ注記を記録している。このフィールドは、法定公表措置フィールドとは別に、行為 / 公正性の注記として記録する。
