# 日本の大量保有報告制度

## 概要

日本の大量保有報告制度は、ブロック保有、影響力、市場需給に関する情報を開示させる FIEA の「5%ルール」である。保有者の保有割合が 5% を超えると、原則として 5 営業日以内に大量保有報告書の提出が必要となる。その後、保有割合が 1% 以上変動した場合などの重要な変更については、原則として 5 営業日以内に変更報告書を提出する必要がある。提出は EDINET を通じて行われる。

本ページは、1年2026 月以降の改正後制度を前提にしている。FSA の提出ページは 2026-05-01 に更新され、同日に改正大量保有報告制度が施行され、保有割合の算定、共同保有者の範囲、様式が変更されたと説明している。出典の日付は慎重に扱う。

本ページは [[finance/japan-tender-offer-process|Japan tender offer process]]、[[finance/japan-activist-investor-playbook|Japan activist investor playbook]]、[[finance/cross-border-m-a-japan|cross-border M&A Japan]]、[[securities/tokyo-stock-exchange|Tokyo Stock Exchange]]、[[financial-regulators/japan-exchange-group|Japan Exchange Group]]、[[megabanks/sbi-hd|SBI HD]]、[[regional-banks/sbi-shinsei-bank|SBI Shinsei Bank]]、[[payment-firms/wealthnavi|WealthNavi]] と併せて読む。

## 制度スナップショット

| 論点 | 実務上の目安 |
|---|---|
| トリガー | 上場株券等の保有割合が 5% を超えること。 |
| 当初報告 | 大量保有報告書。原則として 5 営業日以内。 |
| 変更報告 | 1% 以上の変動またはその他の重要な変更後、原則として 5 営業日以内。 |
| 提出面 | EDINET 電子提出。紙提出は通常ルートではない。 |
| 目的 | 経営への影響力と市場の需給に関する透明性。 |
| 2026 後の注意 | 2026-05-01施行の改革により、算定、共同保有者の範囲、様式が変更された。 |

## 何ではないか

大量保有報告は公開買付書類ではない。保有者、保有割合、保有目的、共同保有者、関連するブロック保有情報を開示するが、それ自体は買付価格、買付期間、買付条件、資金証明、対象会社取締役会の意見を開示するものではない。

| 書類 | 公開記録で確認する事項 |
|---|---|
| 大量保有報告書 | 誰が 5% を超えたのか、どの保有構造で、どの目的なのか。 |
| 変更報告書 | 直近報告後に何が変わったのか。 |
| 公開買付届出書 | 買付条件、資金、条件、TOB 後の方針は何か。 |
| TDnet 開示 | 適時開示を通じて公表された上場会社イベント。 |

[[finance/japan-tender-offer-process|Japan tender offer process]] と本ページは別ルートだが、相互に接続する。

## 保有者分析

2026 後の制度では、公衆縦覧される提出書類に次のような保有者分析項目が含まれる。

- 直接保有
- 投資権限
- 議決権または指図権限
- 共同保有者
- 改正ルールにより含まれる場合のデリバティブまたは株式連動エクスポージャー
- 保有目的
- 重要提案行為
- 開示される場合の担保、借入、資金調達の文脈
- 急速な買い集めまたは売却のパターン

集計項目には、ファンド、投資顧問会社、関連会社、デリバティブの相手方、共同して行動する者が含まれ得る。開示される範囲では、提出者、共同保有者、目的、変更履歴が一体として記録される。

## アクティビズム / 支配権の読み方

大量保有報告は [[finance/japan-activist-investor-playbook|activist investor playbook]] ページの一部である。参入タイミング、持分増加、共同保有者の変化、目的の変更、提出者が重要提案行為を示唆しているかを確認できる。関連する公開記録には次が含まれる。

- 株主提案
- 株主総会の議決結果
- 公開書簡 / キャンペーンサイト
- TDnet 上の会社回答
- 該当する場合の TOB 書類
- 発行会社のガバナンス改革
- 市場価格と出来高の文脈

銀行、証券、保険、fintech 企業については、公開ソースがエンティティレベルの関連性を裏づける場合に限り、関連する支配権事例を [[JapanFG/INDEX|JapanFG]] ページへ戻して接続する。

## EDINET ワークフロー

1. EDINET で発行会社と提出者を検索する。
2. 大量保有報告書と最新の変更報告書を取得する。
3. 通常報告、短期大量保有報告、訂正、補正のいずれかを確認する。
4. 保有割合、共同保有者、保有目的、重要提案行為、存在する場合の担保 / 資金調達開示を抽出する。
5. 日付つきの持分推移を作成する。
6. TDnet / 上場会社開示と照合し、会社側の反応または関連するコーポレートアクションを確認する。
7. TOB が存在する場合は、公開買付届出書を別途読む。

## 読解チェックリスト

| 項目 | 理由 |
|---|---|
| 提出者 / 保有者 | 法的な報告者を特定する。必ずしも最終的な経済的グループとは限らない。 |
| 共同保有者 | 集計と協調的影響力の可能性を判断する。 |
| 保有割合 | 主要な閾値と変更報告の基礎。 |
| 保有目的 | 純投資、政策投資、重要提案、支配権、その他の目的を区別する。 |
| 取得資金 / 借入 | レバレッジ、資金調達、取引意図を示し得る。 |
| 直近取引 | 買い集め / 処分の速度を示す。 |
| デリバティブ / 契約上の権利 | 改正後および隠れたエクスポージャー分析で重要。 |
| 訂正履歴 | 過去の提出を修正し、解釈を変える可能性がある。 |
