# 日本のIBリーグテーブル

## ウィキ上の位置づけ

この項目は [[finance/INDEX|finance index]] の下に位置づけられる。ピア / 対比文脈として [[finance/cross-border-m-a-japan|Cross-border M&A Japan]]、より広い制度 / 規制境界として [[securities/INDEX|securities index]] とあわせて読む。

## 要約

日本の投資銀行リーグテーブルは順位付けの道具であり、絶対的な事実そのものではない。結果は、ベンダー、商品区分、対象となる案件母集団、公表日 / 完了日、役割クレジット、通貨換算に左右される。競争上の位置づけやマンデートへのアクセスを把握するために使うが、必ず情報ベンダーと対象期間を記録する。

FinWiki では、このページは [[securities-firms/goldman-sachs-japan]]、[[securities-firms/nomura-hd]]、[[securities-firms/daiwa-sg]]、[[securities-firms/smbc-nikko]]、[[securities-firms/mizuho-securities]]、[[securities-firms/mufg-mums]] を解釈するためのレイヤーである。

## 何が順位付けされるか

| テーブル種別 | 典型的な指標 | 重要性 |
|---|---|---|
| M&A財務アドバイザー | 公表 / 完了案件の金額と件数 | 関係アクセス、クロスボーダー助言能力、取締役会からの信頼。 |
| ECM / エクイティ・ブックランナー | IPO、PO、CB、株式関連案件の金額と件数 | 資本市場での販売力と発行体アクセス。 |
| DCM / 債券ブックランナー | 債券発行額と件数 | 事業会社、金融機関、公的セクターの債務アクセス。 |
| IPO主幹事 | TSE / JPX 上場プロセス上の役割、推薦書、引受役割 | スタートアップ / 成長企業へのアクセスと個人 / 機関投資家向け販売。 |

## 読解ルール

1. **ベンダーを記録する**: LSEG、Dealogic、Bloomberg、Mergermarket、会社公表テーブルは案件分類が異なる場合がある。
2. **期間を記録する**: 会計年度、暦年、半期、四半期、YTD、公表ベース、完了ベース。
3. **商品を分ける**: M&A、ECM、DCM、IPO引受は異なる能力を評価する。
4. **アドバイザーとブックランナーのクレジットを分ける**: 銀行は取締役会に助言しても資金調達を引き受けない場合があり、逆もある。
5. **親会社 / JV 帰属に注意する**: MUFG / Morgan Stanley Japan 構造では、[[securities-firms/mufg-mums]] と Morgan Stanley MUFG 各社の間でクレジットが分かれ得る。

- [[securities-firms/goldman-sachs-japan]]、[[foreign-financial-institutions/bank-of-america-japan]]、[[foreign-financial-institutions/socgen-japan]]: 外国銀行 / 外資系証券との競争。

## 意思決定での使い方

このページは、次の問いを立てるときに使う。

- 誰が日本の発行体アクセスを持っているか。
- ある銀行がM&A助言に強いのか、引受だけに強いのか。
- ランキングが一件の大型案件を反映しているのか、幅広いフランチャイズの厚みを反映しているのか。
- クロスボーダー取引で国内銀行と外資系銀行のどちらがより良い位置にいるか。
- 総合証券会社とオンライン証券をどう比較するか。
