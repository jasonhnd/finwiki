# ERC-7702 vs ERC-4337 · Ethereum AA 双轨对照


## Wiki route

This entry sits under [[systems/INDEX|systems index]]. Read it against [[systems/erc-7702-overview|ERC-7702 概览 · EOA 临时获得 SCW 能力的 Pectra 升级]] for peer / contrast context and [[fintech/INDEX|fintech index]] for the broader system / regulatory boundary.

## Key facts

- 4337 = 新建 SCW(2023-03 主网应用层激活) ^[extracted]
- 7702 = 升级现有 EOA(2025-05 Pectra 协议层激活) ^[extracted]
- 两者共享 EntryPoint 基础设施 ^[extracted]
- 4337 渗透率 < 5%(2023-2025)· 7702 预计快速铺开 ^[extracted]
- 主推方分离:4337 = Privy / Coinbase CDP / 钱包供应商 · 7702 = Vitalik / Ethereum core dev / MetaMask ^[extracted]

## Mechanism / How it works

**双轨详细对照**:

| 维度 | ERC-4337 | ERC-7702 |
|---|---|---|
| 适合人群 | 新用户 / fintech embedded wallet | 现有 EOA 用户 / DeFi 老钱 |
| 迁移成本 | 新地址 + 资产迁移 | 零成本 · 原地升级 |
| 渗透速度 | 慢(用户需主动建 SCW) | 快(可一键升级) |
| 主推方 | Privy / Coinbase CDP / 钱包供应商 | Vitalik / Ethereum core dev / MetaMask |
| 协议层改动 | 无(纯应用层) | 有(新 tx type) |
| 适用场景 | merchant 给客户发新钱包 | 钱包应用给现有用户升级体验 |
| 风险模型 | SCW 合约本身的安全审计 | delegation 目标合约 + EOA 私钥双重风险 |

**互补性的根本原因**:Ethereum 已有数千万 EOA 用户 · 历史资产/ENS/NFT/DeFi positions 不可能轻易迁移——这是 4337 路径的死结。而 fintech 新接入用户(Stripe / PayPal / Revolut 终端客户)从未有 EOA · 4337 直接发 SCW 是最优路径。

**实际场景分工**:
- Stripe merchant 给新用户发 wallet → 4337 SCW(经 [[agent-economy/privy-embedded-wallet-overview|Privy]])
- MetaMask 用户在 DeFi 上想要 batch approve+swap → 7702 临时 delegation
- 机构 multi-sig → Safe(基于 4337 Modular)
- DeFi 高级用户用 session key 给 trading bot 授权 → 4337 或 7702 + [[agent-economy/erc-7715-overview|ERC-7715]]

## Origin & evolution

2022-2024 间 Ethereum AA 阵营内部争论"协议层 vs 应用层"——4337 押应用层先行落地 · 7702 后跟协议层补全。Vitalik 2024-05 主推 7702 是关键转折点 · 标志 Ethereum 核心开发者正式背书"协议层 AA 是长期方向"。

Pectra 2025-05 激活后 · 2026-Q1-Q2 主流钱包默认支持 · 7702 渗透速度预计远超 4337 历史曲线(因 EOA 用户无需迁移地址)。

## Counterpoints

**重叠风险**:Privy / Coinbase CDP 等 4337 阵营担心 7702 侵蚀其 UX 差异化壁垒。短期(2026-2027)4337 商业模式确实承压。但长期看 · 两者服务不同用户群 · 共存稳定。

**复杂度上升**:钱包应用现在需要同时支持 EOA + 4337 SCW + 7702 EOA-with-delegation 三种地址类型 · 这增加了开发者负担。dApp 端也需要兼容三种地址行为(例如 ERC-1271 签名验证)。
