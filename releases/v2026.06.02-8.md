# 静的サイト構造の第一段階整理

## 日本語

### 概要

FinWiki の静的サイト構造 Phase 1 release です。Astro が root domain corpus を直接読む構造へ移行し、generated mirror と build-time corpus sync への依存を外しました。

### 公開範囲

- Astro content loader と trilingual entry routing
- Domain landing pages
- Wikilink rendering and localization
- GitHub Pages / Vercel shared Bun build pipeline
- Architecture docs, README, CHANGELOG, release notes

### 主要変更

- `site/src/content/entries` mirror を deprecated 扱いにし、build input から外しました。
- `entryRoute()`、`entryDomain()`、`i18nEntryId()` を追加し、frontmatter `slug` による flat route 化を防ぎました。
- `/[lang]/domains/` と `/[lang]/domains/[domain]/` を追加し、domain navigation を静的に生成しました。
- `remark-wikilink` が静的 href を出力し、build hook が en / zh の wikilink href を現在言語へ localize します。
- Vercel / GitHub Pages build から corpus sync step を削除し、同じ Bun pipeline に揃えました。

### 検証結果

- `bun x tsc --noEmit --pretty false`: PASS
- `bun run build`: PASS, 4219 pages
- `bun tools/wiki_link_audit.ts --report wiki-link-improvement-plan.md --fail-on-issues`: entries_checked=1411 / entries_with_issues=0 / dead_wikilink_references=0 / dead_wikilink_targets=0
- `bun tools/vercel_build.ts`: PASS, duplicate_ids=0, Pagefind 4219 pages / 3 languages / 132788 words, static assembly astro_files=8846 / root_surface_files=2838
- en / zh rendered wikilinks no longer retain `/ja/` hrefs in checked samples.

### 既知の注意点

- 古い local worktree には `site/src/content/entries/` が残る場合がありますが、現在の build input ではありません。
- 過去 release notes / CHANGELOG の sync command は当時の作業記録として残します。

### 次の作業

- Push 後に remote HEAD、GitHub Actions deployment、public homepage、domain landing、representative entry、AI discovery surface、Pagefind asset を確認します。
- Vercel project 設定後、preview URL で同じ静的成果物を確認してから DNS cutover を行います。

## English

### Overview

This is the FinWiki static-site architecture Phase 1 release. Astro now reads the canonical root domain corpus directly, removing the generated mirror and build-time corpus sync dependency.

### Release Scope

- Astro content loader and trilingual entry routing
- Domain landing pages
- Wikilink rendering and localization
- Shared Bun build pipeline for GitHub Pages and Vercel
- Architecture docs, README, CHANGELOG, release notes

### Major Changes

- Deprecated `site/src/content/entries` as a generated mirror and removed it from build input.
- Added `entryRoute()`, `entryDomain()`, and `i18nEntryId()` to prevent frontmatter `slug` from collapsing nested routes.
- Added `/[lang]/domains/` and `/[lang]/domains/[domain]/` for static domain navigation.
- `remark-wikilink` now emits static hrefs, while the build hook localizes en / zh wikilink hrefs to the current language.
- Removed the corpus sync step from Vercel / GitHub Pages builds and kept both paths on the same Bun pipeline.

### Validation Results

- `bun x tsc --noEmit --pretty false`: PASS
- `bun run build`: PASS, 4219 pages
- `bun tools/wiki_link_audit.ts --report wiki-link-improvement-plan.md --fail-on-issues`: entries_checked=1411 / entries_with_issues=0 / dead_wikilink_references=0 / dead_wikilink_targets=0
- `bun tools/vercel_build.ts`: PASS, duplicate_ids=0, Pagefind 4219 pages / 3 languages / 132788 words, static assembly astro_files=8846 / root_surface_files=2838
- Checked en / zh rendered samples no longer retain `/ja/` wikilink hrefs.

### Known Notes

- Older local worktrees may still contain `site/src/content/entries/`, but it is no longer a current build input.
- Historical release notes / CHANGELOG sync commands remain as records of the commands used at the time.

### Next Work

- After push, verify remote HEAD, GitHub Actions deployment, the public homepage, domain landing pages, representative entries, AI discovery surfaces, and Pagefind assets.
- After Vercel project setup, validate the preview URL before DNS cutover.
