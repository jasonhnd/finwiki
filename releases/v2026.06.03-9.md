# 領域物理改名 governance→non-profit・manufacturing→manufacturer-finance

## 日本語

### 公開範囲

- root 領域ディレクトリ 2 件: `governance/`→`non-profit/`（4 entry）、`manufacturing/`→`manufacturer-finance/`（8 entry）。
- 対応する i18n（ja/zh/en）ディレクトリ、`.opinions/` 配下、約 460 件の path wikilink を含む 60+ ファイル。
- site build 設定 3 件（`site/src/content.config.ts`・`site/src/lib/siteIndex.mjs`・`site/src/i18n/groups.ts`）、`tools/wiki_link_audit.ts` の domain 配置、`INDEX.md` の domain 表。
- 再生成された discovery surface、README / CHANGELOG / release notes、GitHub Release metadata。

### 主要変更

- **物理改名（git mv）**: v2026.06.03-6 で識別し v2026.06.03-7 で INDEX disambiguation 暫定対処していた 2 領域の mislabel を、ディレクトリ名自体の改名で恒久解消。governance の実体は非営利・公益法人、manufacturing の実体は manufacturer-finance（captive / vendor / export finance）。root + i18n（ja/zh/en）+ `.opinions` を `git mv`。
- **path 参照の精確置換**: bun スクリプトで `governance/…`→`non-profit/…`、`](governance/...)`→`](non-profit/...)`、frontmatter `domain`（引用符付き `domain: "manufacturing"` を含む）等を置換。正文中の "governance"/"manufacturing" の語、および historical な `CHANGELOG.md` / `releases/*.md` の史実記述は意図的に不変とした（rename スクリプトが誤改変した分は `git checkout HEAD --` で復元）。
- **site build 設定の更新**: `ENTRY_DOMAIN_DIRS`（content.config.ts と siteIndex.mjs）、`DOMAIN_TITLES`（ja/en/zh の 3 言語マップ、表示名を「非営利 / Non-Profit / 非营利」「製造業金融 / Manufacturer Finance / 厂商金融」へ）、`i18n/groups.ts` の super-group（corporate に manufacturer-finance、governance group に non-profit）。
- **tool / INDEX の更新**: `wiki_link_audit.ts` の per-domain suggested-link map を新 domain key へ、`INDEX.md` の domain 表第一列を改名。

### 検証結果

- `bun tools/release.ts --check --strict`: EXIT=0、link audit entries=1420 / issues=0 → PASS、md=1484 / domains=23 / entries=1420、counts in sync、JSON / LF / duplicate id verify OK。
- 死リンク 0、residual `[[manufacturing/`・`[[governance/` 0。
- sitemap: non-profit 4 URL / manufacturer-finance 8 URL、旧 `governance/`・`manufacturing/` URL は 0、lastmod 汚染 0、敏感情報 0。

### 既知の注意点

- **公開 URL が不可逆に変わる**（`/{lang}/governance/x`→`/{lang}/non-profit/x`、`/{lang}/manufacturing/x`→`/{lang}/manufacturer-finance/x`）。旧 URL は 301 ではなく消滅するため、外部 backlink は失効する。
- `site/` に node_modules が無いため Astro build を本地検証できず、build 健全性は push 後の GitHub Actions に依存する。build 失敗時は本 commit を revert する。
- canonical_anchor の mirror path（README の v8 note 等）は新 path へ追従済み。

### 次の作業

- push 後に GitHub Actions（Astro build）、remote HEAD、GitHub Release、公開 `/non-profit/INDEX`・`/manufacturer-finance/INDEX` を確認。
- build 失敗時は本 commit を revert する。
