## English

### Release Scope

- AI-discovery surface generator extensions: `tools/generate_ai_discovery.ts`, `lib/markdown_helpers.ts`.
- Regenerated discovery surfaces: `ai-index.json` (new `entities[]` + per-entry `canonical_anchor` + counts `entity_anchors` / `entity_mirror_pages`), `llms-full.txt` (per-mirror canonical-anchor line), `llms.txt` (snapshot line + AI reader rule).
- `README.md` / `CHANGELOG.md` / `releases/v2026.06.04-2.md` / GitHub Release metadata.
- **Public wiki content (entries) is unchanged** (corpus entries=1466 held; md +1 from this release note).

### Major Changes

- **Landed the only remaining main P1 work — "entity edges in discovery output" — for canonical_anchor Phase 2.** This is the final stage after Phase 0 (SCHEMA field + pilot) → Phase 1 (report-only audit, v8) → **Phase 2 (machine-readable entity edges).**
- **Added an `entities[]` array to `ai-index.json`**: 12 entity edges grouping pages that point at the same real-world entity via `canonical_anchor`. Each edge carries `anchor` (vault-root path) / `anchor_url` (resolved URL) / `anchor_resolves` (whether the anchor page exists in the public corpus) / `member_count` / `mirror_count` / `members[]` (`relation: canonical | mirror`). All 12 are currently a JapanFG entity (canonical) + one other-domain mirror, with `anchor_resolves=true` and drift=0.
- **Added a `canonical_anchor` field to every entry in `ai-index.json`** (12 declaring pages; empty string otherwise). Extended the `Entry` type and `buildEntry` to extract it from frontmatter.
- **Added a `Canonical anchor: <anchor> -> <url>` line to each mirror page in `llms-full.txt`** (12 lines).
- **`llms.txt`**: a "Canonical entity anchors: 12" snapshot line and one AI reader rule explaining how to use `entities[]`.
- **Scope decision**: the per-entry API (`api/entries/*.json`) is intentionally left unchanged. Its frontmatter block is already a curated subset (no `related` / `note` / `type`), and the entity edges are fully exposed via `ai-index.json` (`entities[]` + `entries[].canonical_anchor`) and `llms-full.txt`. This avoids a mechanical 1459-file diff and keeps the commit focused (recorded as a future option in the backlog).

### Validation Results

- `bun tools/release.ts --check --strict`: EXIT=0, link audit entries=1466 / issues=0 → PASS, md=1537 / domains=23, counts in sync, JSON / LF / duplicate-id verify OK.
- Discovery-surface checks: `entities[].length=12`, `entity_anchors=12` / `entity_mirror_pages=12`, every edge `anchor_resolves=true`, 12 `Canonical anchor:` lines in `llms-full.txt`.
- Because no wiki `.md` content was edited, `sitemap.xml` lastmod pollution is 0 (byte-identical to committed) and `robots.txt` is unchanged. 0 sensitive data.

### Known Notes

- This release is primarily a discovery-generator extension; no new public wiki entries were added (corpus entries unchanged).
- `entities[]` only emits entities that have at least one mirror (a self-pointing-only page is not an edge). There are currently no self-pointing declarations.
