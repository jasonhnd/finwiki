# JapanFG split-leftover route の一括修復 — entity 页の旧 `[[JapanFG/INDEX]]` route を各域 INDEX へ（source + i18n mirror）

## 日本語

### 公開範囲

- v2026.06.04-4 の JapanFG 17 域物理拆分後、**非 thin の entity 页**にも開頭 Wiki-route が旧 `[[JapanFG/INDEX]]` を指す取り残しが多数残っていた（死链ではない——umbrella は健在——が、各実体は自域 INDEX を route にすべき）。backlog P2 の最後の項目を消化。
- 対象: 主仓 source entity `.md` **575 件** + その `ja/zh/en` i18n mirror **2,069 件** = **2,644 件**。決定的スクリプトで一括変換。

### 主要変更

- **link 変換**: 非 JapanFG の entity 页の `[[JapanFG/INDEX|…]]` / `[[JapanFG/INDEX]]`（route 行・footer・本文）を、その页が属する **各域 INDEX**（`[[<domain>/INDEX|<domain> INDEX]]`）へ置換。source と 3 言語 mirror で **link target を byte 一致**させた（v05-2 の i18n 整合性教訓）。footer の重複 bullet は dedupe。
- **mirror `source:` ポインタ修正**: 拆分後も旧 `source: japanfg/<slug>` のままだった mirror frontmatter を、実際の現行パス `source: <domain>/<slug>` へ **1,887 件**訂正（prep-translate は path で解決するため build には無影響、metadata の正しさ向上）。
- **`source_hash` は意図的に不変**: mirror は現 source に対し既に stale（`fresh=false`、拆分等の source 編集後に未再翻訳）。新 source hash へ書き換えると「fresh な翻訳」と偽証し prep-translate が再翻訳をスキップ → stale 翻訳を恒久凍結してしまう。古い hash を残し pipeline の自己修復シグナルを温存。
- **意図的に不変**: 各 domain `INDEX.md` の `[[JapanFG/INDEX|JapanFG]]`（"entity-umbrella discovery surface" への正当なナビ・cross-link）と `releases/` 史実は変更せず（gotcha #7）。

### 検証結果

- `bun tools/wiki_link_audit.ts`: `entries_with_issues=0`、`dead_wikilink_references=0`、`canonical_anchor_drift=0`（1485 entries）。非 JapanFG source に残る `[[JapanFG/INDEX]]`= 0（domain INDEX 32 + releases 4 は意図的保持）。mirror 残= 0。
- `bun tools/release.ts --check --strict`: **EXIT=0**、md=1564 / domains=40 / entries=1485 / counts in sync、JSON/LF/dup-id OK。
- ⚠️ `site/` はローカル Astro build 不可（gotcha #3）→ push 後 GitHub Actions「Deploy FinWiki」の "Build human site" / "Build static search index" を `gh run watch` で確認。i18n schema は `passthrough()` で source_hash/link を検証しないため build 自体は安全側。

### 既知の注意点 / 次の作業

- mirror の翻訳本文は依然 stale（route link だけ patch）。次回 translate pipeline 実行時、575 source は再翻訳され mirror が現 source 準拠に更新される（self-healing）。今回の patch はその間の表示一貫性を担保。
- **次の作業**: P3 = v12 双批の近重複マージ（loyalty / money-market）、工具 = `.txt` 入口を audit 化、i18n 機械翻訳フローの docs 化。

## English

### Release Scope

- After the v2026.06.04-4 physical split into 17 JapanFG domains, many **non-thin entity pages** still opened with the stale `[[JapanFG/INDEX]]` route (not a dead link — the umbrella survives — but each entity should route via its own domain INDEX). This clears the last P2 backlog item.
- Scope: **575** source entity `.md` + their **2,069** `ja/zh/en` i18n mirrors = **2,644** files, transformed by a deterministic script.

### Major Changes

- **Link rewrite**: in non-JapanFG entity pages, `[[JapanFG/INDEX|…]]` / `[[JapanFG/INDEX]]` (route line, footer, body) → each page's own domain INDEX (`[[<domain>/INDEX|<domain> INDEX]]`). Link **target kept byte-identical** across source and the 3 mirrors (v05-2 i18n-integrity lesson). Duplicate footer bullets deduped.
- **Mirror `source:` pointer fix**: mirror frontmatter still carried the pre-split `source: japanfg/<slug>`; corrected to the real current path `source: <domain>/<slug>` on **1,887** mirrors (prep-translate resolves by path, so this is build-neutral metadata hygiene).
- **`source_hash` intentionally untouched**: mirrors are already stale vs current source (`fresh=false`, never re-translated after the split etc.). Rewriting to the new source hash would falsely certify a fresh translation and make prep-translate skip re-translation, freezing stale content. Leaving the old hash preserves the self-healing re-translate signal.
- **Intentionally unchanged**: each domain `INDEX.md`'s `[[JapanFG/INDEX|JapanFG]]` (a legitimate cross-link to the "entity-umbrella discovery surface") and `releases/` history (gotcha #7).

### Validation Results

- `bun tools/wiki_link_audit.ts`: `entries_with_issues=0`, `dead_wikilink_references=0`, `canonical_anchor_drift=0` (1485 entries). Remaining `[[JapanFG/INDEX]]` in non-JapanFG source = 0 (32 domain INDEX + 4 releases kept intentionally); mirrors remaining = 0.
- `bun tools/release.ts --check --strict`: **EXIT=0**, md=1564 / domains=40 / entries=1485 / counts in sync, JSON/LF/dup-id OK.
- ⚠️ `site/` cannot be built locally (gotcha #3) → after push, confirm GitHub Actions "Deploy FinWiki" ("Build human site" / "Build static search index") via `gh run watch`. The i18n schema is `passthrough()` (no source_hash/link validation), so the build is on the safe side.

### Known Notes / Next Work

- Mirror translation bodies remain stale (only the route link was patched). The next translate-pipeline run will re-translate the 575 sources and bring mirrors up to current source (self-healing); this patch ensures display consistency in the meantime.
- **Next**: P3 = merge v12 near-duplicate topics (loyalty / money-market); tooling = bring `.txt` entrypoints into the audit; document the i18n machine-translation flow.
