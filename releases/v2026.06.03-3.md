# 日本語サイト表示層の英語残留修正

## 日本語

### 公開範囲

- `/ja/` の human-site 表示層。
- header / footer / root page / browse / domain / entry chrome。
- entry title fallback、confidence 表示、rendered wikilink label。
- README / CHANGELOG / release notes / GitHub Release metadata。

### 主要変更

- `/ja/` を日本語ユーザー目線で再確認し、残っていた英語 UI label を日本語化しました。
- `site/src/i18n/ui.ts` に footer、aria、skip link、theme toggle、AI surface link の翻訳を追加しました。
- `EntryLayout`、home、browse、domain pages で title fallback と breadcrumb / jump label を現在言語へ寄せました。
- `site/src/lib/siteIndex.mjs` に localized title map を追加し、ja i18n frontmatter title または先頭 H1 を fallback title として使えるようにしました。
- `site/src/plugins/localize-wikilinks.mjs` が build 後の ja HTML で `data-wl` label を ja i18n title へ置換します。
- root homepage を日本語 first-view として書き換え、CSS の見出し / brand letter-spacing を 0 に揃えました。

### 検証結果

- `bun run build`: PASS、4,219 pages。
- build log: `localized wikilink hrefs: en=0 zh=0 ja-labels=17603`。
- `bun run index:search`: PASS、Pagefind 4,218 pages / 3 languages / 128,744 words。
- `bun tools/check_duplicate_html_ids.ts site/dist`: checked=4219 / duplicate_id_pages=0 / duplicate_ids=0。
- `/ja/` build output で `AI surface`、`Skip to content`、`Featured domains`、`Machine-readable`、`Last public snapshot`、`Toggle color theme`、`Jump to domain`、`Breadcrumb` の通常 UI 残留がないことを確認しました。
- 代表 entry で H1、confidence chip、domain link、関連 wikilink label が日本語表示へ変わることを確認しました。

### 既知の注意点

- proper noun、abbreviation、product name、official English name は意図的に英語のまま残します。
- raw code block 内の `[[...|...]]` 表示や、本文中に source として残る英語語句は今回の表示層修正の対象外です。
- content-level の長いタイトルや本文の語調 QA は、必要に応じて別作業で進めます。

### 次の作業

- push 後に GitHub Actions、remote HEAD、公開 `/ja/` の代表ページを確認します。
- 必要に応じて ja content-level の残存英語を、UI ではなく翻訳本文 QA として追加監査します。

## English

### Release Scope

- The human-site display layer for `/ja/`.
- Header, footer, root page, browse / domain / entry chrome.
- Entry title fallback, confidence labels, and rendered wikilink labels.
- README / CHANGELOG / release notes / GitHub Release metadata.

### Major Changes

- Reviewed `/ja/` from a Japanese-reader perspective and localized remaining English UI labels.
- Added footer, aria, skip-link, theme-toggle, and AI-surface translations in `site/src/i18n/ui.ts`.
- Routed entry, home, browse, and domain pages through localized title fallback and localized navigation labels.
- Added localized title maps in `site/src/lib/siteIndex.mjs`, using ja i18n frontmatter titles or first H1 values as fallback display titles.
- Extended `site/src/plugins/localize-wikilinks.mjs` so rendered ja HTML rewrites `data-wl` labels to ja i18n titles after build.
- Rewrote the root homepage as a Japanese-first entry surface and normalized heading / brand letter spacing to 0.

### Validation Results

- `bun run build`: PASS, 4,219 pages.
- Build log: `localized wikilink hrefs: en=0 zh=0 ja-labels=17603`.
- `bun run index:search`: PASS, Pagefind 4,218 pages / 3 languages / 128,744 words.
- `bun tools/check_duplicate_html_ids.ts site/dist`: checked=4219 / duplicate_id_pages=0 / duplicate_ids=0.
- Rendered `/ja/` output no longer has normal UI residue for `AI surface`, `Skip to content`, `Featured domains`, `Machine-readable`, `Last public snapshot`, `Toggle color theme`, `Jump to domain`, or `Breadcrumb`.
- A representative ja entry now shows Japanese H1, confidence chip, domain link, and related wikilink labels.

### Known Notes

- Proper nouns, abbreviations, product names, and official English names intentionally remain in English.
- Raw `[[...|...]]` labels inside code blocks and English terms preserved as source prose are outside this display-layer fix.
- Longer content-level title or body-style QA can be handled as a separate translation-body review.

### Next Work

- After push, verify GitHub Actions, the remote HEAD, and representative public `/ja/` pages.
- If needed, run a separate ja body-content QA pass for residual English that is not part of the UI layer.
