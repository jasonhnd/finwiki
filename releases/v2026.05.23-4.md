## English

### Overview

FinWiki now keeps the homepage readable for humans while adding machine-readable entry points that let AI agents and crawlers discover and traverse the full public corpus. Future wiki content, index, domain-count, or public-snapshot updates must regenerate the AI discovery files in the same work.

### Scope

- `robots.txt`
- `sitemap.xml`
- `llms.txt`
- `llms-full.txt`
- `ai-index.json`
- `tools/generate_ai_discovery.py`
- `index.html`
- `INDEX.md`
- `README.md`
- `AGENTS.md`
- `CHANGELOG.md`
- `releases/v2026.05.23-4.md`

### Major Changes

- Added `tools/generate_ai_discovery.py` to generate AI discovery surfaces from the Markdown corpus, root `INDEX.md` domain map, and `wiki-link-improvement-plan.md` audit summary.
- Added `robots.txt` and `sitemap.xml` so crawlers, search engines, and AI browsers can discover the public site.
- Added `llms.txt`, `llms-full.txt`, and `ai-index.json` so AI readers can consume entry points, URL rules, domain routes, page URLs, source paths, summaries, headings, and resolved wikilinks.
- Added ordinary Markdown web links beside the core Obsidian wikilinks in `INDEX.md`, making domain routes easier for crawlers that do not understand wikilinks.
- Updated `README.md`, `AGENTS.md`, and the homepage with a maintenance rule requiring AI discovery files to stay synchronized with future content updates.
