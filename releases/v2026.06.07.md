# 開発文書体系の要件・仕様階層化

## 日本語

### 公開範囲

- `docs/` を「治理 / 戦略 / プロダクト / 要件 / アーキテクチャ / 機能仕様 / 実装 / 品質 / 運用 / アーカイブ」の階層に再編しました。
- 本リリースは開発文書体系の再構築であり、wiki 本文、公開金融エントリー、サイト UI、公開 API の仕様は変更していません。

### 主要変更

- 既存の平置き文書を新構造へ移動しました。例: `docs/04-architecture/system-architecture.md`、`docs/06-implementation/toolchain.md`、`docs/08-operations/release-process.md`。
- 旧 site-level の architecture / development-plan 文書を `docs/04-architecture/astro-site-architecture.md` と `docs/99-archive/site-development-plan.md` へ統合しました。
- `BRD`、`PRD`、`NFR`、`RTM`、`ARD`、`FSD` と各子仕様、品質文書、運用 runbook を追加しました。
- `docs/README.md` を唯一の開発文書入口として再構成しました。
- 高推論の仕様モデルが仕様・検収・task packet を作り、高速コード実装モデルが許可ファイルだけを実装する [Model-Agent Workflow](../docs/06-implementation/model-agent-workflow.md) を追加しました。
- active subagent 上限 10、完了 subagent の close/retire、コード agent の allowed-file review を `parallel-development`、RTM、QA checklist、acceptance criteria へ反映しました。
- [Code/Docs Alignment Audit](../docs/07-quality/code-doc-alignment-audit.md) と [Next Development Plan](../docs/01-strategy/next-development-plan.md) を追加し、次期開発順序を `.txt` audit、docs link checker、active-doc stale scan、i18n freshness report、deployment runbook、content dedup / selected expansion に固定しました。
- active docs と実装事実の drift を修正しました。`domains.md` は現在の 40 domains へ、`astro-site-architecture.md` は現在の Astro/content/i18n/build 架構へ、`toolchain.md` / wikilink spec は canonical drift が release gate の hard gate である説明へ更新しました。
- UI/UX を FSD の実行入口として追加しつつ、横断テーマとして [UI/UX Principles](../docs/02-product/ui-ux-principles.md)、[Theme System](../docs/04-architecture/theme-system.md)、[UI/UX Functional Spec](../docs/05-functional-specs/ui-ux.md)、[Visual QA Checklist](../docs/07-quality/visual-qa-checklist.md) を新設しました。日語読者優先、低ノイズ金融リファレンス、非 purple 方向、CSS token source-of-truth、light/dark、responsive、Japanese UI chrome、visual QA を PRD / ARD / FSD / NFR / RTM / QA に接続しました。
- [Documentation Drift Audit](../docs/07-quality/documentation-drift-audit.md) を追加し、active docs、root control docs、source `canonical_anchor`、generated discovery / API surface を同じ基準で監査しました。`JapanFG/<entity>` の旧 canonical alias を現行 entity path へ修正し、`japanfg-split-design.md` を 17 新領域、40 domains、`JapanFG/INDEX` umbrella retained の現状へ同期しました。
- `AGENTS.md` と root `README.md` に、`docs/` が内部開発文書であり corpus から排除されるが、公開 repo として秘密情報・個人情報は禁止であることを明記しました。
- `lib/markdown_helpers.ts` の frontmatter 境界検出を LF / CRLF 両対応にし、CRLF の corpus entry が API 生成時に frontmatter なしとして扱われる既存不具合を修正しました。
- `lib/markdown_helpers.ts` は `docs/` への相対 Markdown links を AI-surface `markdown_links` から除外し、`tools/generate_ai_discovery.ts` は `api/entries/` を清掃してから現行 API JSON を再生成するようにしました。

- stale API residue check: `api/entries/JapanFG/` は retained umbrella の `INDEX.json` のみです。

- `bun tools/wiki_link_audit.ts --fail-on-issues`: EXIT=0。`entries_checked=1485`、`entries_with_issues=0`、`dead_wikilink_references=0`、`canonical_anchor_drift=0`。
- `cd site && bun run build`: EXIT=0、4441 pages built。

### 既知の注意点

- `docs/` は引き続き公開 wiki corpus、sitemap、llms、ai-index、api、Astro サイト、dead-link audit から除外されます。
- 旧 release note の説明文も新パスへ更新し、現在の文書入口と矛盾しないようにしました。

### 次の作業

- 今後の開発は `next-development-plan.md` の Phase A→E の順で進めます。
- 今後の開発で要求・仕様・実装の関係が変わる場合は、`docs/03-requirements/rtm.md` を同時に更新します。
- コード実装を委譲する場合は、まず `model-agent-workflow.md` の task packet を作り、完了後に allowed-file diff と subagent closeout を確認します。

## English

### Release Scope

- Reorganized `docs/` into governance, strategy, product, requirements, architecture, functional specs, implementation, quality, operations, and archive layers.
- This release restructures developer documentation only. It does not change wiki body content, public financial entries, site UI, or public API behavior.

### Major Changes

- Moved the existing flat docs into the new structure, including `docs/04-architecture/system-architecture.md`, `docs/06-implementation/toolchain.md`, and `docs/08-operations/release-process.md`.
- Folded the former site-level architecture / development-plan docs into `docs/04-architecture/astro-site-architecture.md` and `docs/99-archive/site-development-plan.md`.
- Added BRD, PRD, NFR, RTM, ARD, FSD, child functional specs, quality docs, and operations runbooks.
- Rewrote `docs/README.md` as the single developer-docs entry point.
- Added [Model-Agent Workflow](../docs/06-implementation/model-agent-workflow.md): high-reasoning specification models write specs, acceptance, and task packets; fast code-implementation models implement only allowed files.
- Reflected the active-subagent limit of 10, completed-agent closeout, and allowed-file code review in parallel-development, RTM, QA checklist, and acceptance criteria.
- Added [Code/Docs Alignment Audit](../docs/07-quality/code-doc-alignment-audit.md) and [Next Development Plan](../docs/01-strategy/next-development-plan.md), setting the next development order to `.txt` audit, docs link checker, active-doc stale scan, i18n freshness report, deployment runbook, and then content dedup / selected expansion.
- Fixed active-doc drift against implementation facts: `domains.md` now reflects the current 40 domains, `astro-site-architecture.md` now reflects the current Astro/content/i18n/build architecture, and `toolchain.md` / the wikilink spec now state that canonical drift is a hard gate through release checks.
- Added UI/UX as the FSD execution entry while keeping it cross-linked across product, architecture, functional specs, and quality: [UI/UX Principles](../docs/02-product/ui-ux-principles.md), [Theme System](../docs/04-architecture/theme-system.md), [UI/UX Functional Spec](../docs/05-functional-specs/ui-ux.md), and [Visual QA Checklist](../docs/07-quality/visual-qa-checklist.md). Connected Japanese-reader-first UX, restrained financial-reference presentation, non-purple visual drift rules, CSS token source of truth, light/dark behavior, responsive behavior, Japanese UI chrome, and visual QA into PRD / ARD / FSD / NFR / RTM / QA.
- Added [Documentation Drift Audit](../docs/07-quality/documentation-drift-audit.md) to audit active docs, root control docs, source `canonical_anchor`, and generated discovery / API surfaces under one baseline. Updated old `JapanFG/<entity>` canonical aliases to current entity paths and synchronized `japanfg-split-design.md` to the current 17 new domains, 40 total domains, and retained `JapanFG/INDEX` umbrella.
- Updated `AGENTS.md` and root `README.md` to state that `docs/` is internal developer documentation excluded from the corpus, while still prohibited from containing secrets, personal information, customer information, non-public conversations, or internal case details.
- Updated `lib/markdown_helpers.ts` so frontmatter boundary detection supports both LF and CRLF, fixing an existing generator bug where CRLF corpus entries could be emitted as if they had no frontmatter.
- Updated `lib/markdown_helpers.ts` to filter relative `docs/` Markdown links out of AI-surface `markdown_links`, and updated `tools/generate_ai_discovery.ts` to clear `api/entries/` before writing current API JSON files.
