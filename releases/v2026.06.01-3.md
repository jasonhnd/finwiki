# GPT i18n 翻訳バッチ継続・1013/1013 進捗

## 日本語

### 概要

2026-06-01 JST の GPT/Codex i18n 継続バッチです。zh/en 883 の release 後、次の 130 source entries を GPT/Codex worker で処理し、累計を zh 1013 / en 1013 へ進めました。

### 公開範囲

- 対象: `site/src/content/i18n/{zh,en}/`
- 作業 batch: 130 source entries, 260 language outputs
- 記録更新: `README.md`, `CHANGELOG.md`, this release note
- AI discovery / counts: `tools/release.py --write` により同期

### 主要変更

- 旧 w4 single-file agents が `.cache/jobs` を lock していたため、該当 idle agents を archive して cleanup を完了しました。
- `bun scripts/prep-parallel.mjs --workers 10 --size 13` で次 batch の 130 jobs を準備しました。
- 10 本の GPT/Codex workers で `.cache/jobs/w0..w9/` を処理しました。
- 未生成 9 entries は recovery workers に分割し、最後の 3 entries は single-file recovery で補完しました。
- `bun scripts/commit-translate.mjs` で placeholder verify + unmask を全件再実行しました。

### 検証結果

- `bun scripts/commit-translate.mjs`: ok=260, needs_review=0, missing=0.
- Cumulative i18n count: zh 1013 / en 1013.
- `bun run build`: passed, 4147 pages built.
- `python tools/release.py --check --strict`: passed.

### 既知の注意点

- Windows では idle agent が worker directory を握ると `.cache/jobs` cleanup が EBUSY になります。
- 今後も batch 終了ごとに worker archive を確認します。

### 次の作業

- 次の GPT/Codex batch で未翻訳分を継続します。
- `prep-parallel` が 0 jobs を返すまで同じ verify-first flow を繰り返します。
