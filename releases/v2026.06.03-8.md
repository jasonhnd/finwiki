### 公開範囲

- mirror page 6 件（`manufacturing/sony-group-finance-arm`、`banking/ui-bank`、`banking/daiwa-next-bank`、`banking/sony-bank`、`banking/gmo-aozora-net-bank`、`JapanFG/japan-post-bank`）。

- 再生成された discovery surface、README / CHANGELOG / release notes。
- GitHub Release metadata。

### 主要変更

- **canonical_anchor back-fill（6 件）**: 「operating-profile companion to the entity anchor」等の自己記述を機械可読シグナルとして確認した mirror page に `canonical_anchor` を設定。`sony-group-finance-arm`→`JapanFG/sony-fg`、net bank 4 件（`ui-bank` / `daiwa-next-bank` / `sony-bank` / `gmo-aozora-net-bank`）→各 JapanFG entity anchor、`japan-post-bank`→`JapanFG/yucho`。Phase 0 pilot 2 件と合わせて計 8 件。case page・anchor を body link しない peer・captive 不在の platform page は保守的に未設定。
- **Phase 1 audit 拡張（report-only）**: `tools/wiki_link_audit.ts` に `auditCanonicalAnchors()` を追加。各 `canonical_anchor` 宣言に対し (a) anchor が実在 entry に resolve するか、(b) 宣言ページが core body で anchor を cross-link するかを検証し、`canonical_anchor_checked` / `canonical_anchor_drift` counter と `--report` 専用セクションを出力。`AuditRow.issues`・`entries_with_issues`・dead counter・exit code には一切触れない厳格な report-only 実装で、既存 gate の挙動は不変。

### 検証結果

- 直接 audit: entries_checked=1420 / entries_with_issues=0 / dead_wikilink_references=0 / dead_wikilink_targets=0 / **canonical_anchor_checked=8 / canonical_anchor_drift=1** / EXIT=0。
- `bun tools/release.ts --check --strict`: count sync 後 EXIT=0、link audit entries=1420 / issues=0 → PASS、counts in sync、JSON / LF / duplicate id verify OK。
- 工具改動の隔離 regression test（content edit を HEAD に戻し工具改動のみ適用）でも `--check --strict` EXIT=0 を確認。

### 既知の注意点

- drift=1 は Phase 0 pilot の `insurance/saison-automobile-fire`（anchor を `## Related` footer のみで参照、core body には未記載）。report-only のため gate を阻害しない。将来 body link を追加するか warning として残すかは後続判断。
- Phase 2+（discovery 出力への entity edge、残る mirror pair の back-fill、hard requirement 化）は後続タスク。
