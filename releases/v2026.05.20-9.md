# v2026.05.20-9 グローバルヘルスチェックの実施とフロントマターおよび壊れたリンクの修正 / 全局健康检查实施与前置元数据及损坏链接修复

## 日本語 🇯🇵

### 概要

この release は、知識庫（Wiki）全体の完全な整合性を保証するためのグローバルヘルスチェック（整合性スキャン）を実行し、不整合が検出されたフロントマター（frontmatter）のスキーマ修正、および壊れたウィキリンク（broken wikilinks）の完全な解決を行ったスナップショットです。また、参照関係の欠落を補うために、2つのコアフレームワークに関する新規Wikiページを追加しました。

### 対象範囲

- 530以上のWiki Markdownファイルおよびインデックス。
- `JapanFG/` フォルダ配下の27コンテンツファイル。
- 壊れたウィキリンク修正（`JapanFG/sompo.md`）。
- 新規Wikiページ追加（`fintech/protocol-renewal-trigger-as-event-anchor.md`、`security/fork-and-rebrand-5-layer-audit-framework.md`）。
- ルートおよびドメインインデックスの件数・リンク同期。

### 主要変更

1. **フロントマターの自動スキーマ適合修正（27ファイル）**:
   - `last_tended` 日付フィールドが欠落していたファイルに対し、`last_updated` または `created` を元に自動挿入しました。
   - 非推奨または無効な `confidence` / `status` 値を一括で正規値へマッピング（`high` -> `likely`、`confirmed`/`extracted` -> `certain`、`tentative` -> `possible`、`planned` -> `candidate`）。
2. **壊れた内部ウィキリンクの完全解決**:
   - `JapanFG/sompo.md` にあった誤った内部リンク `[[JapanFG]]` を正規インデックス `[[JapanFG/INDEX|JapanFG]]` に修正。
3. **新規コアフレームワークWikiページの追加**:
   - `fintech/protocol-renewal-trigger-as-event-anchor.md`: 商業共生関係における「契約更新」が市場全体の排期と製品リリースに及ぼす影響を説明するフレームワーク。
   - `security/fork-and-rebrand-5-layer-audit-framework.md`: DeFi等で頻発するフォーク＆リブランドプロジェクト向けの、上流差分から法規・部署指紋に及ぶ「5層監査フレームワーク」。
4. **インデックスおよび集計件数の同期**:
   - ルート `INDEX.md` および `fintech/INDEX.md` を更新し、総数・カテゴリ別数を同期し、新規リンクを追加。

### 検証

- **整合性再スキャン**: 自作の整合性スキャンツール `vault_health_check.py` を再実行し、制御文書（元々フロントマターを持たないREADMEやCHANGELOGなど）以外の全Wikiコンテンツファイルで、スキーマエラーおよび壊れた内部リンクの発生数が完全に **0** であることを確認しました。
