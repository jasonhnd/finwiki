# Wikiリンク実在性ゲート追加と死リンク修復

## 日本語

### 概要

2026-06-02 JST の FinWiki graph integrity リリースです。本文内 `[[wikilink]]` が存在しない target を指す場合も release-blocking issue として検出するよう、wiki-link audit と CI gate を強化しました。

### 公開範囲

- 対象: 公開 wiki graph、`tools/wiki_link_audit.py`、`.github/workflows/deploy.yml`
- 修復範囲: 既存 entry の alias 追加、非金融背景リンクの plain text 化、`wiki-link-improvement-plan.md`
- 同期 surface: `README.md`、`CHANGELOG.md`、`releases/v2026.06.02-4.md`、release-generated discovery surfaces

### 主要変更

- `tools/wiki_link_audit.py` に dead wikilink target 実在性の検出と report 出力を追加しました。
- fenced code / inline code 内の `[[...]]` を audit 対象から外し、表内 escaped pipe、case-insensitive alias、root public files、末尾 slug fallback を解決するようにしました。
- GitHub Actions に `python tools/wiki_link_audit.py --fail-on-issues` を追加し、Astro build 前に graph integrity を止めます。
- 100 entry に 144 件の明示 alias を追加し、旧 slug / 表記ゆれ / 統合前 link を既存実体へ解決しました。
- 非金融背景 entity 7 件は、新規 entry を捏造せず plain text に戻しました。
- agent が過大生成した `*/INDEX` 置換は revert し、semantic route を保つ形で修復しました。
