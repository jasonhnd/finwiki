# v2026.05.20 FinWiki 初回公開

## 日本語 🇯🇵

### 概要

FinWiki の初回 GitHub Release です。
この release は、金融・フィンテック・暗号資産・日本金融機関を中心とする公開情報ベースの知識ベースを、GitHub 上で参照可能な公開 snapshot として固定するものです。📚💴🧭

### 対象範囲

- リポジトリ: この GitHub repository
- ブランチ: `main`
- Snapshot 種別: 知識ベース初回公開
- 主な形式: Markdown / Obsidian `[[wikilink]]`
- 主要入口:
  - `README.md`
  - `INDEX.md`
  - `CHANGELOG.md`
  - `SCHEMA.md`
  - `OBSIDIAN-SETUP.md`

### 主な内容

- 🏦 `JapanFG/`: 日本の金融グループ、銀行、保険、証券、政策金融、決済会社、外資金融機関。
- 💳 `fintech/`: ステーブルコイン、CBDC、EPI、MiCA、GENIUS、MAS、HKMA、オンチェーン金融。
- 🪙 `exchanges/`: 日本・海外 CEX、DEX、VASP 規制、JVCEA、カストディ、IEO、セキュリティ事案。
- ⚙️ `systems/`: Canton、CCTP、EigenLayer、Hyperlane、BFT、クロスチェーン、L1/L2 基盤。

### この release で固定した運用ルール

- `README.md` は日本語を先、中文を後に置く。
- `README.md` では絵文字を積極的に使い、ナビゲーションと領域識別を分かりやすくする。
- `README.md` と `CHANGELOG.md` は常に日本語・中文を同時に維持する。
- `CHANGELOG.md` は詳細なタイムライン、作業背景、対象範囲、検証結果を記録する。
- GitHub Release title は日本語のみとし、body は日本語を先、中文を後に置く。
- release notes にも個人情報、ローカルパス、非公開案件情報を含めない。

### 検証

- `gh release list` で release 未作成状態を確認。
- `README.md` の日本語セクションが中文セクションより前にあることを確認。
- `CHANGELOG.md` に本 release 準備作業の詳細記録を追加。
- `AGENTS.md` に release title は日本語のみ、body は日本語先・中文後とするルールを追加。
- `git diff --cached --check` で新規文書に trailing whitespace がないことを確認。

### 既知の注意点

- 既存 Markdown には過去作成時点の trailing whitespace が残っています。初回同期の原文保持を優先したため、この release では一括整形していません。
- 本リポジトリは知識ベースであり、ソフトウェアパッケージやバイナリ成果物は含みません。
- 時事性の高い金融・規制情報は、各エントリーの `last_updated`、`confidence`、出典、または後続更新で確認してください。

### 次の作業

本 release 将以金融、金融科技、加密资产和日本金融机构为核心的公开信息知识库，固定为 GitHub 上可引用的公开快照。📚💴🧭

### 范围

- 仓库：本 GitHub repository
- 分支：`main`
- 快照类型：知识库首次公开
- 主要格式：Markdown / Obsidian `[[wikilink]]`
- 主要入口：
  - `README.md`
  - `INDEX.md`
  - `CHANGELOG.md`
  - `SCHEMA.md`
  - `OBSIDIAN-SETUP.md`

### 主要内容

- 🏦 `JapanFG/`：日本金融集团、银行、保险、证券、政策金融、支付公司和外资金融机构。
- 💳 `fintech/`：稳定币、CBDC、EPI、MiCA、GENIUS、MAS、HKMA、链上金融。
- 🪙 `exchanges/`：日本与海外 CEX、DEX、VASP 监管、JVCEA、托管、IEO、安全事件。
- ⚙️ `systems/`：Canton、CCTP、EigenLayer、Hyperlane、BFT、跨链、L1/L2 基础设施。

### 本 release 固化的维护规则

- `README.md` 必须日文在前，中文在后。
- `README.md` 可以积极使用 emoji，让导航和领域识别更清楚。
- `README.md` 与 `CHANGELOG.md` 必须永远同时维护日文和中文。
- `CHANGELOG.md` 必须记录详细时间线、工作背景、影响范围和验证结果。
- GitHub Release 标题只使用日文，body 必须日文在前、中文在后，并写清楚每个公开快照的意义。
- release notes 也不能包含个人信息、本地路径或非公开案件信息。
