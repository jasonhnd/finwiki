# v2026.05.20-4 GitHub履歴プライバシーrewrite

## 日本語 🇯🇵

### 概要

この release は、公開 repository 上の個人情報残留リスクを下げるため、FinWiki の Git history と release tags を privacy rewrite した snapshot です。

### 対象範囲

- `main` branch の Git history。
- 既存 release tags: `v2026.05.20`, `v2026.05.20-2`, `v2026.05.20-3`。
- Commit author / committer metadata。
- GitHub Release body の privacy scan。
- README / CHANGELOG の release 運用ルール。

### 主要変更

- 公開不適合ページと残存 privacy text を history 全体から除去。
- Commit author / committer を generic maintainer identity に統一。
- Local `refs/original/*` と reflog を削除し、garbage collection を実行。
- `main` と release tags を GitHub へ force push。
- README に history rewrite 時の cached view / dangling commit purge 確認ルールを追加。

### 検証

- Local full-history grep: privacy pattern no match。
- Deleted-path history scan: no reachable commit。
- Fresh mirror clone: remote refs 内の privacy pattern no match。
- Fresh shallow clone: current tree / commit metadata no match。
- GitHub Releases body scan: privacy pattern no match。
- GitHub remote refs: rewrite 後の `main` と release tags のみを指す。

### 既知の注意点

- GitHub API では、force push 後もしばらく unreferenced / dangling commit が旧 SHA で直接参照できる場合があります。
- GitHub 上の cached views / dangling commits を完全に purge するには、GitHub Support への依頼が必要です。
- Fork や第三者 clone は repository owner 側だけでは削除できません。

### 次の作業

- GitHub Support に cached views / dangling commits の purge を依頼する。
- 既存 fork がある場合は、fork owner に削除または history rewrite を依頼する。

- 删除本地 `refs/original/*` 和 reflog，并执行 garbage collection。
