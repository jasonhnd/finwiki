# Vercel shadow deployment 準備

## 日本語

### 概要

2026-06-02 JST の FinWiki hosting migration 準備リリースです。`finwiki.zksc.io` の本番配信は GitHub Pages のまま維持し、Vercel ecosystem へ移行するための shadow deployment 設定と静的出力 assembly を追加しました。

### 公開範囲

- 対象: Vercel 設定、root build scripts、static publish assembly、Vercel 出力除外ルール
- 同期 surface: `README.md`、`CHANGELOG.md`、`releases/v2026.06.02-5.md`、release-generated discovery surfaces
- 本番状態: DNS cutover は未実施。GitHub Pages を現行本番・rollback 先として維持

### 主要変更

- root `vercel.json` を追加し、Install: `cd site && bun install --frozen-lockfile`、Build: `node tools/vercel_build.mjs`、Output: `_vercel_public` を定義しました。
- root `package.json` に Vercel build と static publish assembly 用 script を追加しました。
- `tools/vercel_build.mjs` を追加し、wiki-link audit、strict release check、corpus sync、Astro build、duplicate HTML id check、static assembly を同じ順序で実行します。
- `tools/assemble_static_publish.mjs` を追加し、Astro HTML (`site/dist`) と raw Markdown / AI discovery surface を `_vercel_public` に組み立てます。
- `_vercel_public`、`_site`、`.vercel`、`app` を sync / audit / discovery から除外し、ローカル出力の再帰混入を防止しました。
- README の Hosting section を GitHub Pages 単体から Vercel migration 前提へ更新しました。

### 検証結果

- `node tools/vercel_build.mjs`: PASS
- `python tools/wiki_link_audit.py --fail-on-issues`: PASS。entries_checked=1411 / entries_with_issues=0 / dead_wikilink_references=0 / dead_wikilink_targets=0
- `python tools/release.py --check --strict`: PASS。counts in sync
- `bun run build`: PASS。4147 pages を生成
- `python tools/check_duplicate_html_ids.py site/dist`: PASS。checked=4147 / duplicate_id_pages=0 / duplicate_ids=0
- `node tools/assemble_static_publish.mjs --out _vercel_public`: PASS。astro_files=4149 / root_surface_files=2837

### 既知の注意点

- DNS はまだ Vercel へ切り替えていません。
- Vercel 側で repository root、install command、build command、output directory、production URL、custom domain、certificate provisioning を確認するまで、GitHub Pages workflow と `CNAME` file を残します。
- Vercel preview / production URL で `/`、`/ja/`、`/en/`、`/zh/`、`/llms.txt`、`/ai-index.json`、`api/entries/index.json` を確認してから cutover します。

### 次の作業

- Vercel project を repository root で import し、設定値が `vercel.json` と一致することを確認します。
- Vercel production URL の 200 / SSL / AI discovery surface を確認します。
- 確認後、Cloudflare の `finwiki` CNAME を Vercel 指定 target へ変更します。
