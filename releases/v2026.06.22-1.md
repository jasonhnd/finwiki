# AI surface Phase 1: per-entry API frontmatter enrichment

## 日本語

### 公開範囲

- #42 の Phase 1 として、`api/entries/*.json` の per-entry API frontmatter subset を富化しました。
- Wiki corpus 本文は変更していません。
- `api/entries/` は release generation で clean rewrite され、AI discovery outputs も同期されます。
- `llms-tasks.txt` の stale partial-spinoff route を current canonical route へ更新しました。

### 主要変更

- `api/entries/<domain>/<slug>.json` の `frontmatter` object に `type`、`canonical_anchor`、`related`、`note` を追加しました。
- 既存の `sources`、`title`、`aliases`、`domain`、`created`、`last_updated`、`last_tended`、`review_by`、`confidence`、`tags`、`status` は保持しています。
- Scalar field が存在しない場合は `null`、list field が存在しない場合は `[]` を出力します。
- `docs/05-functional-specs/ai-discovery-surface.md` と `docs/04-architecture/ai-discovery-architecture.md` に Phase 1 の API boundary を明記しました。
- `llms-tasks.txt` の `corporate-strategy/partial-spinoff-tax-deferral` link を `corporate-strategy/japan-kabushiki-bunpai-spinoff-regime` へ更新し、AI task guide の route audit を通る状態にしました。

### 検証結果

- `bun tools/release.ts --write`
- `bun tools/release.ts --check --strict`
- `bun run surface:drift`
- `bun run ai:audit`
- Targeted JSON inspection for `canonical_anchor`, `related`, `note`, `type`, and `sources`
- `git diff --check`

### 既知の注意点

- #42 は typed entity graph exposure ではありません。
- `api/entries/*.json` には typed `entity_node` / `entity_edges` を追加していません。
- `ai-index.json` の typed graph fields are existing generator fields synchronized by release generation; #42 does not add per-entry entity-edge API fields.

### 次の作業

- Maintainer review and merge of the #42 PR.
- After #42 is merged, continue the serial queue with #37, then #38.
