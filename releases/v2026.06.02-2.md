# deploy-pages 警告抑制と CI ログ仕上げ

## 日本語

### 概要

2026-06-02 JST の FinWiki CI ログ仕上げリリースです。`v2026.06.02` で Node.js 20 runtime annotation は消えましたが、`actions/deploy-pages@v5` の内部依存が Node.js 24 上で `DEP0040 punycode` deprecation warning を出したため、deploy step に限定して抑制しました。

### 公開範囲

- 対象: `.github/workflows/deploy.yml`
- 同期 surface: `README.md`, `CHANGELOG.md`, `releases/v2026.06.02-2.md`, discovery / footer snapshot
- wiki 本文と i18n corpus は変更なし

### 主要変更

- `actions/deploy-pages@v5` step に `NODE_OPTIONS: --no-deprecation` を設定しました。
- 抑制範囲は deploy action process のみです。build、HTML id gate、artifact upload には適用していません。
- checkout / upload / deploy actions は Node.js 24 runtime 世代のまま維持します。

### 検証結果

- `bun run build`: PASS。4147 pages を生成。
- `python tools/check_duplicate_html_ids.py site/dist`: PASS。`checked=4147 duplicate_id_pages=0 duplicate_ids=0`。
- `python tools/release.py --check --strict`: PASS。counts sync、JSON valid、LF endings OK。
- `git diff --check`: PASS。
- CRLF check: PASS。`w/crlf` count は 0。
- GitHub Actions log の deprecation warning は push 後に確認します。

### 既知の注意点

- この release は CI ログの警告ノイズ整理です。公開 wiki 本文の内容変更はありません。

### 次の作業

- push 後に GitHub Actions run が成功し、`Node.js 20` / `DEP0040` / `deprecation` のログ命中がないことを確認します。
- `https://finwiki.zksc.io/` と `llms.txt` の反映を確認します。
