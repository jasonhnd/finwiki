### 主要変更

- **roadmap P1 = canonical_anchor Phase 2 を完了**。残っていた「hard requirement + audit を gate 化」を落地。
- **評価結論（ADR-007）**: 「すべての mirror が `canonical_anchor` を必須宣言」は**機械的に gate できない**——mirror か否かは編集判断であり列挙不能。実測で 12 件の宣言済み mirror のうち anchor と**同 slug は 4 件のみ**（残り 8 件は `toyota-financial-services→toyota-financial` 等の異 slug）、かつ同 slug 跨域対は 4/4 すべて宣言済み。slug 启发式は recall≈33% かつ現状 violation 0 で gate にならない。よって**機械 gate は「宣言の有効性」（drift）に置き、「mirror 必須」は規範 + review で担保**する。
- **drift を硬 gate 化**: `tools/wiki_link_audit.ts` に `--fail-on-canonical-drift` を追加（`canonical_anchor_drift>0` で EXIT=1）。`tools/release.ts` の `runWikiLinkAudit` が常にこの flag を渡すため、`--check` / `--write` の各リリースで drift=0 を gate する。`release.helpers.ts` の `AuditResult` に `canonicalDrift` を追加し、失敗時に「broken links」と「canonical drift」を区別したメッセージを出す。
- **default は report-only を維持**: flag 無しでは drift は従来どおり report-only（ADR-002 の隔離・既定挙動を保持）。`canonical_anchor` 計数は `entries_with_issues` に入らない。
- **規範を hard requirement 化**: `SCHEMA.md` の `canonical_anchor` を「mirror page では必須・drift は gate」と明記、`docs/06-implementation/entry-authoring.md` に新規 mirror page の必須宣言と drift 硬 gate を追記。

### 検証結果

- 正例: `wiki_link_audit.ts --fail-on-canonical-drift`（drift=0）→ EXIT=0。
- 負例: 壊れた `canonical_anchor` を持つ一時ページを置くと drift=1 → **EXIT=1**（gate が効く）、確認後に削除。
- 対照: flag 無しでは同じ drift=1 でも EXIT=0（report-only 既定を保持）。
- `bun tools/release.ts --check --strict`: EXIT=0、link audit entries=1466 / issues=0 / canonical_drift=0 → PASS、md=1538 / domains=23、counts in sync、JSON / LF / duplicate id verify OK。
- wiki 本文 `.md` 未編集のため `sitemap.xml` lastmod 汚染は 0（規範文書 `SCHEMA.md` 等の編集分のみ）。敏感情報 0。

- 「mirror 必須」は機械 enforcement ではなく authoring/review-time のルール（ADR-007 の理由）。drift gate は宣言済み anchor の rot を防ぐ。

### Major Changes

- **Completed roadmap P1 (canonical_anchor Phase 2)** by landing the remaining "hard requirement + gate the audit" item.
- **Evaluation outcome (ADR-007)**: "every mirror must declare `canonical_anchor`" **cannot be a mechanical gate** — mirror-ness is an editorial judgment and not enumerable. Empirically, of the 12 declared mirrors only **4 are same-slug** as their anchor (the other 8 are editorial, e.g. `toyota-financial-services→toyota-financial`), and all 4 same-slug cross-domain pairs are already declared. The slug heuristic has ~33% recall and finds 0 violations, so it can't gate "find undeclared mirrors." Therefore the **mechanical gate enforces declaration validity (drift)**, while "mirror pages must declare" is enforced by the authoring standard + review.
- **Promoted drift to a hard gate**: added `--fail-on-canonical-drift` to `tools/wiki_link_audit.ts` (`canonical_anchor_drift>0` → EXIT=1). `runWikiLinkAudit` in `tools/release.ts` always passes the flag, so every `--check` / `--write` release gates `drift=0`. Added `canonicalDrift` to `AuditResult` (`release.helpers.ts`) and a failure message that distinguishes broken links from canonical drift.
- **Default stays report-only**: without the flag, drift remains report-only (preserving ADR-002's isolation and default behavior). `canonical_anchor` counters still never feed `entries_with_issues`.
- **Standards now state the requirement**: `SCHEMA.md` marks `canonical_anchor` "required on mirror pages; drift-gated"; `docs/06-implementation/entry-authoring.md` documents the required declaration on new mirror pages and the hard drift gate.

### Validation Results

- Positive: `wiki_link_audit.ts --fail-on-canonical-drift` (drift=0) → EXIT=0.
- Negative: planting a temp page with a broken `canonical_anchor` yields drift=1 → **EXIT=1** (the gate bites); removed afterward.
- Control: without the flag the same drift=1 still gives EXIT=0 (report-only default preserved).
- `bun tools/release.ts --check --strict`: EXIT=0, link audit entries=1466 / issues=0 / canonical_drift=0 → PASS, md=1538 / domains=23, counts in sync, JSON / LF / duplicate-id verify OK.
- Because no wiki `.md` content was edited, `sitemap.xml` lastmod pollution is 0 (only the standards docs like `SCHEMA.md` changed). 0 sensitive data.

### Known Notes

- This release is tooling (the gate) + standards docs; no new public entries (corpus entries unchanged; md +1 from this release note).
- "Mirror pages must declare" is an authoring/review-time rule, not mechanical enforcement (see ADR-007 rationale). The drift gate prevents declared anchors from rotting.
