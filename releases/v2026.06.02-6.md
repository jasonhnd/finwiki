# Pagefind静的全文検索追加

## 日本語

### 概要

2026-06-02 JST の FinWiki 静的検索追加リリースです。FinWiki は引き続き完全静的サイトとして運用し、データベース、検索 API、外部検索サービスを追加せず、Pagefind による build-time index を `/pagefind/*` として生成します。

### 公開範囲

- 対象: Astro human site、GitHub Pages build、Vercel shadow build、root cover homepage、三語 UI label
- 同期 surface: `README.md`、`CHANGELOG.md`、`releases/v2026.06.02-6.md`、release-generated discovery surfaces
- 出力: `/pagefind/*` search index / component assets

### 主要変更

- `pagefind` を site devDependency に追加し、`bun run index:search` を追加しました。
- GitHub Pages workflow と Vercel build script の両方で、Astro build と duplicate-id check の後に Pagefind index を生成します。
- 共通 layout と root cover homepage に Pagefind modal trigger を追加し、header の `Search` control または `Ctrl/Cmd+K` で検索できるようにしました。
- main content に `data-pagefind-body` を付け、header / footer / browse list に `data-pagefind-ignore` を付け、本文中心の検索 index にしました。
- root cover homepage と `/ja/`、`/en/`、`/zh/` の human pages で検索 UI を読み込むようにしました。

### 検証結果

- `node tools/vercel_build.mjs`: PASS
- `python tools/wiki_link_audit.py --report wiki-link-improvement-plan.md --fail-on-issues`: PASS。entries_checked=1411 / entries_with_issues=0 / dead_wikilink_references=0 / dead_wikilink_targets=0
- `python tools/release.py --check --strict`: PASS。counts in sync
- `bun run build`: PASS。4147 pages
- `python tools/check_duplicate_html_ids.py site/dist`: PASS。checked=4147 / duplicate_id_pages=0 / duplicate_ids=0
- `bun run index:search`: PASS。Pagefind indexed 4147 pages / 3 languages (ja, zh, en) / 136151 words
- Local browser check: `/`、`/en/`、`/zh/` で modal を開き、`銀行` 561 件、`bank` 1063 件、`银行` 941 件を確認。中国語検索結果から `/zh/banking/paypay-bank/` への遷移も確認しました。

### 既知の注意点

- Pagefind は日本語・中国語の stemming を行わない warning を出しますが、検索 index と UI は生成されます。
- 検索は静的 index のみで動作します。database / backend / external search API は追加していません。
- 検索対象は現在表示中の言語ページを中心にしています。

### 次の作業

- GitHub Pages deployment 後に public URL の `/pagefind/pagefind-component-ui.js`、`/`、`/zh/` を確認します。
- Vercel shadow deployment 作成後に `_vercel_public` 内の `/pagefind/*` と検索 UI を同じ手順で確認します。
- 必要になれば将来、domain filter や type filter を追加します。

## English

### Overview

This is the 2026-06-02 JST FinWiki static-search release. FinWiki remains a fully static site. No database, search API, or external search service was added; Pagefind now generates a build-time search index under `/pagefind/*`.

### Release Scope

- Target: Astro human site, GitHub Pages build, Vercel shadow build, root cover homepage, trilingual UI labels
- Synced surfaces: `README.md`, `CHANGELOG.md`, `releases/v2026.06.02-6.md`, release-generated discovery surfaces
- Output: `/pagefind/*` search index / component assets

### Major Changes

- Added `pagefind` as a site devDependency and added `bun run index:search`.
- Both the GitHub Pages workflow and Vercel build script now generate the Pagefind index after Astro build and duplicate-id validation.
- Added a Pagefind modal trigger to the shared layout and root cover homepage, reachable from the header `Search` control or `Ctrl/Cmd+K`.
- Added `data-pagefind-body` to main content and `data-pagefind-ignore` to headers, footers, and browse lists so the index focuses on article content.
- Loaded the search UI on the root cover homepage and the `/ja/`, `/en/`, and `/zh/` human pages.

### Validation Results

- `node tools/vercel_build.mjs`: PASS
- `python tools/wiki_link_audit.py --report wiki-link-improvement-plan.md --fail-on-issues`: PASS. entries_checked=1411 / entries_with_issues=0 / dead_wikilink_references=0 / dead_wikilink_targets=0
- `python tools/release.py --check --strict`: PASS. counts in sync
- `bun run build`: PASS. 4147 pages
- `python tools/check_duplicate_html_ids.py site/dist`: PASS. checked=4147 / duplicate_id_pages=0 / duplicate_ids=0
- `bun run index:search`: PASS. Pagefind indexed 4147 pages / 3 languages (ja, zh, en) / 136151 words
- Local browser check: opened the modal on `/`, `/en/`, and `/zh/`; confirmed 561 results for Japanese `銀行`, 1063 results for English `bank`, and 941 results for Chinese `银行`; confirmed navigation from a Chinese result to `/zh/banking/paypay-bank/`.

### Known Notes

- Pagefind warns that Japanese and Chinese stemming is not available, but the index and UI are still generated.
- Search is static-index only. No database, backend, or external search API was introduced.
- Search is centered on the currently displayed language pages.

### Next Work

- After GitHub Pages deployment, verify `/pagefind/pagefind-component-ui.js`, `/`, and `/zh/` on the public URL.
- After Vercel shadow deployment, verify `/pagefind/*` inside `_vercel_public` and repeat the same search UI checks.
- Add domain or type filters later if the site needs them.
