# v2026.05.20-2 Release運用ルール更新

## 日本語 🇯🇵

### 概要

この release は、FinWiki の GitHub Release 運用を明確化するための更新 snapshot です。
主な変更は、**Release title を日本語のみ**にすること、および **今後すべての push で README / CHANGELOG / Releases を同時更新する**ことです。

### 対象範囲

- `README.md`: Release title、release body、push 連動ルールを更新。
- `CHANGELOG.md`: 本変更の詳細タイムラインを追加。
- `AGENTS.md`: 今後の agent が守る release title / body / push 連動ルールを追加。
- `releases/v2026.05.20.md`: 初回 release notes の見出しと運用ルール表現を更新。
- `releases/v2026.05.20-2.md`: 本 release notes を追加。

### 主要変更

- GitHub Release の title は日本語のみとする。
- Release body は引き続き日本語を先、中文を後に置く。
- `origin/main` へ push するたびに、詳細な `README.md`、`CHANGELOG.md`、release notes、GitHub Release を更新する。
- 既存 release `v2026.05.20` の title から中文タイトルを外す。
- 作業中に存在した大量の wiki 本文変更・削除は、本 release の対象外として触らない。

### 検証

- `gh release view v2026.05.20` で既存 release title を確認。
- `README.md`、`AGENTS.md`、`releases/v2026.05.20.md` の該当文言を確認。
- `git diff --cached --check` で今回 stage する文書に trailing whitespace がないことを確認する。
- push 後、`v2026.05.20-2` を通常 release として作成する。

### 既知の注意点

- この release は運用ルール更新です。wiki 本文の大量変更・削除は含めません。
- 既存の未コミット本文変更は、別作業として扱います。

### 次の作業

- 今後の push では、README / CHANGELOG / Releases の三点更新を必ず同時に行う。
- 重要な内容整理や公開面クリーニングを行う場合は、別 release として範囲を明確化する。

## 中文 🇨🇳

### 概要

本 release 是为了明确 FinWiki 的 GitHub Release 运用规则。
主要变化是：**Release 标题只使用日文**，以及 **今后每次 push 都必须同时更新 README / CHANGELOG / Releases**。

### 范围

- `README.md`：更新 release title、release body 和 push 联动规则。
- `CHANGELOG.md`：追加本次变更的详细时间线。
- `AGENTS.md`：追加后续 agent 必须遵守的 release title / body / push 联动规则。
- `releases/v2026.05.20.md`：更新首次 release notes 的标题和运用规则表述。
- `releases/v2026.05.20-2.md`：新增本 release notes。

### 主要变更

- GitHub Release 的 title 只使用日文。
- Release body 继续保持日文在前、中文在后。
- 每次 push 到 `origin/main`，都必须更新详细的 `README.md`、`CHANGELOG.md`、release notes 和 GitHub Release。
- 从既有 release `v2026.05.20` 的 title 中移除中文标题。
- 当前工作区存在的大量 wiki 正文修改和删除，不属于本 release 范围，本次不处理。

### 验证

- 用 `gh release view v2026.05.20` 确认既有 release title。
- 检查 `README.md`、`AGENTS.md`、`releases/v2026.05.20.md` 的相关表述。
- 执行 `git diff --cached --check`，确认本次 staged 文档没有尾随空格。
- push 后创建正式 release `v2026.05.20-2`。

### 已知注意事项

- 本 release 是运用规则更新，不包含 wiki 正文的大量修改或删除。
- 当前未提交的正文修改应作为另一项工作单独处理。

### 下一步

- 后续每次 push 都必须同时更新 README / CHANGELOG / Releases。
- 如果进行重要内容整理或公开面清理，应单独创建 release 并写清楚范围。
