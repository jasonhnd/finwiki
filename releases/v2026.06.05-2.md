# P4 i18n — v12 + P3 の 55 entry に ja/zh/en mirror を生成（165 file、鍵不要）

## 日本語

### 公開範囲

- `site/src/content/i18n/{ja,zh,en}/` 配下に **165 個の翻訳 mirror**（55 source entry × 3 言語）を新規生成。対象は v12 並列追加の 43 entry + P3（v05-1）の 12 entry（mirror 未生成だった content entry）。
- 三語 README/CHANGELOG/`releases/v2026.06.05-2.md`、backlog（P4 を 🟢 へ）。corpus（root .md）は不変——i18n は `site/` 配下で corpus 計数・discovery には不算入。

### 主要変更

- **鍵不要で P4 を実施**。v05-1 では「i18n パイプライン `translate.mjs` が `ANTHROPIC_API_KEY` 必要 → 未実行」としたが、これは誤った前提だった。`translate.mjs` は **standalone Node script が別の Claude Haiku を API 経由で呼ぶ**ため鍵が要るだけ。本 release は **subagent（Claude 自身）が直接翻訳**して mirror を生成——P3 entry を書いたのと同じやり方。
- 8 並列 subagent（domain 別 file-scope 隔離）が各 entry を ja/zh/en へ翻訳。mirror frontmatter は pipeline 形式を再現: `source` / `source_hash`（= `sha256(body).slice(0,16)`、`translate.mjs` と同一算法）/ `lang` / `model` / `status: machine` / `fidelity: ok` / `title` / `translated_at`。
- **ガードレール**: i18n は link audit の対象外なので、各 agent + 全体チェックで全 165 mirror の `[[link]]` target と数値が source と byte 一致することを検証（mismatch=0）。`[[…]]` は target/label とも verbatim 保持（label の現地化は build の `localize-wikilinks` が担当）。数値・日付・URL・コード・`^[…]` marker も verbatim。
- 17 新ドメイン INDEX は mirror **不要**（domain landing は config から生成。banking も INDEX mirror 無し）。DOMAIN_TITLES は v05-1 で追加済み。

### 検証結果

- 全体 link-integrity チェック: `checked=165 link_mismatches=0`。i18n collection schema は `.passthrough()`（`model:` 等の追加 key 許容、pipeline 自身も `model:` を書く）。
- `bun tools/release.ts --check --strict`: **EXIT=0**（corpus 不変、entries=1485 / issues=0 / dead=0 / canonical_drift=0 / domains=40 / counts in sync）。
- 最終検証は **CI の Astro build**（i18n mirror を実処理）——push 後に確認。

### 既知の注意点 / 次の作業

- 翻訳は Haiku ではなく強モデル（subagent）による直接翻訳のため、既存 pipeline mirror（Haiku + mask）とは生成経路が異なるが、形式・hash は互換（将来 `translate.mjs` を鍵付きで回しても増量 skip が効く）。
- ja layer は元々 pipeline の別工程だったが、本 release では ja も含め 3 言語同時生成。

## English

### Release Scope

- Generated **165 translation mirrors** (55 source entries × 3 langs) under `site/src/content/i18n/{ja,zh,en}/`. Targets: the 43 v12 parallel-added entries + the 12 P3 (v05-1) entries that lacked mirrors.
- Trilingual README/CHANGELOG/`releases/v2026.06.05-2.md`, backlog (P4 → 🟢). The corpus (root `.md`) is unchanged — i18n lives under `site/` and is excluded from corpus counts / discovery.

### Major Changes

- **Did P4 without an API key.** v05-1 stated "the i18n pipeline `translate.mjs` needs `ANTHROPIC_API_KEY`, so P4 is blocked" — that framing was wrong. `translate.mjs` needs the key only because it's a **standalone Node script that calls a separate Claude Haiku over the API**. This release instead had **subagents (Claude itself) translate directly** and write the mirrors — the same way the P3 entries were written.
- 8 parallel subagents (per-domain, file-scope-isolated) translated each entry to ja/zh/en. Mirror frontmatter replicates the pipeline format: `source` / `source_hash` (= `sha256(body).slice(0,16)`, identical to `translate.mjs`) / `lang` / `model` / `status: machine` / `fidelity: ok` / `title` / `translated_at`.
- **Guardrail**: because i18n is outside the link audit, each agent plus a global pass verified that every one of the 165 mirrors has `[[link]]` targets and numbers byte-identical to its source (mismatches=0). `[[…]]` wikilinks were kept verbatim (target + label; the build's `localize-wikilinks` handles label localization). Numbers/dates/URLs/code/`^[…]` markers also verbatim.
- The 17 new-domain INDEX pages need **no** mirror (domain landings are generated from config — banking has no INDEX mirror either). DOMAIN_TITLES was added in v05-1.

### Validation Results

- Global link-integrity check: `checked=165 link_mismatches=0`. The i18n collection schema is `.passthrough()` (accepts extra keys like `model:`; the pipeline writes `model:` too).
- `bun tools/release.ts --check --strict`: **EXIT=0** (corpus unchanged, entries=1485 / issues=0 / dead=0 / canonical_drift=0 / domains=40 / counts in sync).
- Final validation is the **CI Astro build** (which actually processes the mirrors) — verified after push.

### Known Notes / Next Work

- These were translated by a strong model (subagents) directly rather than Haiku-via-`translate.mjs`, so the generation path differs from existing pipeline mirrors, but the format + hash are compatible (a future keyed `translate.mjs` run would incrementally skip them).
