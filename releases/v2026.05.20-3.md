# v2026.05.20-3 公開面プライバシー監査

## 日本語 🇯🇵

### 概要

この release は、FinWiki の公開面から個人情報、ローカル環境情報、非公開会話、内部案件由来の内容を取り除くための privacy cleanup snapshot です。

### 対象範囲

- 公開 wiki 本文全体の privacy pattern scan。
- 非公開・個人由来の方法論、writing、strategy、lifestyle、internal-case 系ページの削除。
- `README.md`, `INDEX.md`, `SCHEMA.md`, `OBSIDIAN-SETUP.md`, `AGENTS.md`, `CHANGELOG.md` の公開面ルール更新。
- `finance/INDEX.md`, `systems/INDEX.md` の削除後の routing 更新。
- 初回 release notes 内の account handle / 非公開方法論表現の除去。

### 主要変更

- 503 knowledge entries / 20 domains の公開面に再整理。
- 非公開 business case、個人所得・資産例、ユーザー原話、ローカル Obsidian path を含むページを削除。
- README / SCHEMA / AGENTS に「公開情報のみ」「個人情報禁止」「local path 禁止」を明文化。
- 削除済みページへの wikilink を除去。

### 検証

- local path、email、account handle、ユーザー原話、内部案件、私募貸付、個人 FIRE 関連の pattern scan を実施。
- 削除済みディレクトリと削除済みファイルへの残存リンクを scan。
- `git diff --check` で whitespace エラーを確認。
- push 後に remote HEAD と release 状態を確認する。

### 既知の注意点

- この release は current tree の公開面 cleanup です。過去の Git history や既存 GitHub release body の完全消去は、別途 history rewrite / release edit の対象です。
- 公開情報であっても時事性の高い金融・規制情報は、各エントリーの `last_updated`, `confidence`, `sources` を確認してください。

### 次の作業

- 必要に応じて、残存ページに public source URL を追加する。
- 完全な履歴消去が必要な場合は、Git history rewrite と release body 更新を別作業として行う。
