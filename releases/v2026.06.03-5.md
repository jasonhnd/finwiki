# AI タスク入口の死リンク修正とスナップショット同期

## 日本語

### 公開範囲

- 手動管理の AI タスク入口 `llms-tasks.txt`。
- `README.md` / `CHANGELOG.md` / `index.html` / release notes。
- 再生成された discovery surface（`sitemap.xml` / `ai-index.json` / `llms.txt` / `llms-full.txt` / `api/entries/*`）。
- GitHub Release metadata。

### 主要変更

- `llms-tasks.txt` の死リンク 10 件を、entry の改名・再編後の現存 entry へ再マッピング（money-market / derivatives / payments / policy-finance / securities / systems）。
- 例: `boj-open-market-operations-overview`→`boj-open-market-operations`、`japan-money-market-structure`→`japan-money-market`、`hokkaido-credit-guarantee-corporation`→`hokkaido-credit-guarantee-corp`、`japan-pts-liquidity-data`→`japan-pts-liquidity-data-guide`、`japan-cashless-payment-market-overview`→`cashless-jp-landscape`、`hyperlane-permissionless-interop-overview`→`hyperlane-overview`。
- Snapshot 行を `1,449 files / ~10.43M chars / 2026-05-25 JST` から `1,472 files / 1,411 link-audited entries / 23 domains / ~9.99M non-space chars / 2026-06-03 JST` へ更新。
- `releases/v2026.06.03-5.md` の追加で corpus が 1,471 → 1,472 files になったため discovery surface を再生成し、`README.md` / `index.html` / `ai-index.json` / `llms.txt` / `llms-full.txt` の counts を同期。
- 再生成で `systems/cross-chain-five-pole-comparison-matrix` の壊れていた frontmatter（全 null）も正しく再解析・修復。
- `README.md` に三語の AI 入口 note を追加。

### 検証結果

- link target 再照合: unique=160 / missing=0。
- `bun tools/release.ts --check --strict`: EXIT=0、link audit entries=1411 / issues=0 → PASS、canonical counts md=1472 / domains=23 / entries=1411、counts in sync、JSON / LF / duplicate id verify OK。
- mtime 復元により既存 1,469 ページの sitemap lastmod は不変。変更は 12 ファイルで、`sitemap.xml` は v2026.06.03-5 の URL 追加のみ。

### 既知の注意点

- corpus は release notes 追加で 1,471 → 1,472 files、領域数 23 は不変。
- fresh clone は fs mtime を一律 reset し、`generate_ai_discovery.ts` は mtime から lastmod を生成する。今回は commit 済み `sitemap.xml` の lastmod を真値として各ページ mtime を復元してから再生成した。この復元を省くと lastmod が全件 clone 日付へ汚染される。
- `.txt` は `tools/wiki_link_audit.ts`（`.md` のみ対象）のスコープ外のため死リンクが潜在していた。今後 entry を改名・再編する際は `llms-tasks.txt` の link も併せて更新が必要。

### 次の作業

- push 後に remote HEAD、GitHub Actions、GitHub Release、公開 `/llms-tasks.txt` を確認。
- 将来的には `.txt` 系入口の link target 実在性も release gate に組み込み、`.md` 以外の AI surface でも死リンクを自動検出することを検討。

- Because `.txt` files are outside the `tools/wiki_link_audit.ts` (`.md`-only) scope, these dead links stayed latent. Future entry renames / reorganizations must update the `llms-tasks.txt` links as well.
