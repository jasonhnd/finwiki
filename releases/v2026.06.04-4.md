# JapanFG 分割 — 巨大ドメインを 17 の機関類型別ドメインへ物理分割（23→40 領域）

## 日本語

### 公開範囲

- **公開 URL の不可逆変更**: かつて平坦だった `JapanFG/`（約 620 entity）を **17 個の機関類型別トップレベルドメインへ物理分割**。entity の URL が変わる（例 `JapanFG/sumitomo-mitsui` → `megabanks/sumitomo-mitsui`）。旧 `JapanFG/<entity>` URL は 404（非 301、v9 の改名と同様）。`JapanFG/INDEX` のみ umbrella hub として存続。
- 移動: 628 entity/license ファイル（root）+ 1,887 i18n ミラー（ja/zh/en）+ 6 sub-INDEX→ドメイン INDEX + 4 registry。
- 書換: 約 53,600 件の `[[JapanFG/…]]` wikilink target を新ドメインへ（root + i18n）、frontmatter `domain` と JapanFG/ alias を一括。
- 17 ドメイン INDEX landing page（10 並列 agent が執筆/整形）、site 設定 3 ファイル、root `INDEX.md` ドメイン表、discovery surface 全再生成、三語 README/CHANGELOG/release notes、`docs/`（ADR-008・domains・japanfg-split-design）。

### 主要変更

- **17 新ドメイン（角色復数名）**: megabanks(19) / regional-banks(135) / cooperative-banks(45) / trust-banks(47) / life-insurers(46) / non-life-insurers(56) / securities-firms(36) / asset-managers(14) / payment-firms(53) / card-issuers(23) / leasing-firms(23) / consumer-finance(5) / trading-company-finance(7) / financial-conglomerates(7) / foreign-financial-institutions(57) / financial-regulators(52) / financial-licenses(7)。**領域数 23 → 40**。
- **分類**: 既存 9 sub-INDEX の membership を一次シグナル + frontmatter tag で 3 大類（保険→生/損、証券-運用→証券/運用、決済-カード-リース-金融→決済/カード/リース）を分割。データが示した未収まり群のため `consumer-finance` / `trading-company-finance` / `financial-conglomerates` を追加（評価過程は ADR-008、`docs/japanfg-split-design.md`）。628 entity 中 617 を tag で自動マップ、11 を手動裁定。
- **ADR-004 の知情推翻**: 旧来は「移動コストが極大ゆえ sub-INDEX で代用」だったが、ユーザー判断で物理分割を選択（コスト承知）。
- **`JapanFG/INDEX`** を 17 ドメインへ案内する umbrella hub に書換（数千の `[[JapanFG/INDEX]]` link を有効に保持）。

### 検証結果

- `bun tools/release.ts --check --strict`: **EXIT=0**、link audit entries=1473 / issues=0 / **dead_wikilink=0** / **canonical_anchor_drift=0** → PASS、**md=1545 / domains=40**、counts in sync、JSON/LF/dup-id OK。
- 移動後の corpus に `[[JapanFG/` で残るのは umbrella INDEX 参照と、別ドメインへ alias で解決する既存 cross-ref のみ（dead=0 が保証）。
- 並列 10 agent は各ドメインに file-scope 隔離で INDEX 執筆 + 35 件の missing_peer_link 修復。

### 既知の注意点

- **公開 URL が約 620 ページ分変わる**（旧 JapanFG/ URL は消滅、非 301）。これは承知の上の不可逆変更。
- DOMAIN_TITLES（site の localized 領域名）に 17 新ドメインは未追加 → 暫定で slug 表示（次手で補完予定、build は graceful）。
- `aeon-financial-service-detail`（実体は電通系金融アーム）等の軽微な命名は将来 re-slug 候補。

### 次の作業

- push 後 GitHub Actions「Deploy FinWiki」build 緑を確認（site/ 設定変更のため必須）。
- DOMAIN_TITLES に 17 ドメインの ja/zh/en 名を追加。wiki_link_audit の domain route map を 17 ドメインへ拡充（非 gating）。

## English

### Release Scope

- **Irreversible public-URL change**: the formerly flat `JapanFG/` domain (~620 entities) was **physically split into 17 institution-type top-level domains**. Entity URLs change (e.g. `JapanFG/sumitomo-mitsui` → `megabanks/sumitomo-mitsui`); old `JapanFG/<entity>` URLs 404 (no 301, like the v9 rename). Only `JapanFG/INDEX` survives, as an umbrella hub.
- Moves: 628 entity/license files (root) + 1,887 i18n mirrors (ja/zh/en) + 6 sub-INDEX→domain INDEX + 4 registries.
- Rewrites: ~53,600 `[[JapanFG/…]]` wikilink targets to their new domains (root + i18n), plus frontmatter `domain` and JapanFG/ aliases.
- 17 domain INDEX landing pages (authored/refined by 10 parallel agents), 3 site-config files, root `INDEX.md` domain map, full discovery-surface regen, trilingual README/CHANGELOG/release notes, `docs/` (ADR-008, domains, japanfg-split-design).

### Major Changes

- **17 new role-plural domains**: megabanks(19) / regional-banks(135) / cooperative-banks(45) / trust-banks(47) / life-insurers(46) / non-life-insurers(56) / securities-firms(36) / asset-managers(14) / payment-firms(53) / card-issuers(23) / leasing-firms(23) / consumer-finance(5) / trading-company-finance(7) / financial-conglomerates(7) / foreign-financial-institutions(57) / financial-regulators(52) / financial-licenses(7). **Domain count 23 → 40.**
- **Mapping**: the existing 9 sub-INDEX memberships were the primary signal, with frontmatter tags splitting the 3 large/mixed categories (insurance→life/non-life, securities-AM→securities/asset-managers, payments-cards-leasing→payments/cards/leasing). The data forced three extra domains — `consumer-finance`, `trading-company-finance`, `financial-conglomerates` — for entities that fit none of the three (evaluation in ADR-008 / `docs/japanfg-split-design.md`). 617 of 628 entities were auto-mapped by tag; 11 were resolved by hand.
- **Informed override of ADR-004**, which had used sub-INDEX navigation precisely to avoid this move cost; the owner chose the physical split with the cost accepted.
- **`JapanFG/INDEX` rewritten as an umbrella hub** to the 17 domains, keeping the thousands of `[[JapanFG/INDEX]]` links valid.

### Validation Results

- `bun tools/release.ts --check --strict`: **EXIT=0**, link audit entries=1473 / issues=0 / **dead_wikilink=0** / **canonical_anchor_drift=0** → PASS, **md=1545 / domains=40**, counts in sync, JSON/LF/dup-id OK.
- The only `[[JapanFG/` references left in the corpus are the umbrella-INDEX link and pre-existing cross-refs that resolve by alias to other domains (dead=0 guarantees they resolve).
- 10 parallel agents authored each domain's INDEX under file-scope isolation and fixed 35 `missing_peer_link` entries.

### Known Notes

- **~620 public URLs change** (old `JapanFG/` URLs disappear, not 301). This is the accepted irreversible cost.
- DOMAIN_TITLES (localized domain names on the site) does not yet include the 17 new domains → interim slug display (build degrades gracefully; titles are a fast follow-up).
- Minor naming (e.g. `aeon-financial-service-detail`, actually a Dentsu-group finance arm) is a future re-slug candidate.

- Add ja/zh/en names for the 17 domains to DOMAIN_TITLES; extend the wiki_link_audit domain route map to the 17 domains (non-gating).
