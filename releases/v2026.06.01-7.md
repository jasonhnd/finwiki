# 三語語彙品質監査と専門語彙 polish

## 日本語

### 概要

2026-06-01 JST の FinWiki 三語 i18n 語彙品質リリースです。GPT/Codex worker を 10 並列で使い、日文原文、簡体中文訳、English 訳の語彙・語調・専門表現を監査し、大学学部水準を超える金融・制度・インフラ語彙へ寄せました。

### 公開範囲

- 対象: 公開 wiki 本文と `site/src/content/i18n/{zh,en}/`
- i18n 累計: zh 1380 / en 1380
- Release surfaces: `README.md`, `CHANGELOG.md`, `releases/v2026.06.01-7.md`
- Discovery / counts: `tools/release.py --write` により `llms.txt`, `llms-full.txt`, `ai-index.json`, `sitemap.xml`, `api/entries/*`, `index.html` を同期

### 主要変更

- 10 並列 GPT/Codex worker で三語語彙 audit を実施し、低レジスター表現、曖昧表現、機械翻訳由来の不自然表現を抽出しました。
- 10 並列 repair worker と residual repair / final polish worker で、専門語彙・制度語彙・金融実務語彙へ修正しました。
- 最終 scan で残った `Placeholder VC` は固有名詞として維持し、東洋経済新報社「すごいベンチャー100」は公式媒体名として維持しました。
- `systems/cross-chain-five-pole-comparison-matrix.md` の body wikilink を補強し、link audit の route / peer / system link 条件を再び満たしました。
- `.cache/` の agent 作業報告が link audit 対象に混入しないよう、未追跡作業 cache を削除しました。

### 検証結果

- `python tools/wiki_link_audit.py --report wiki-link-improvement-plan.md --fail-on-issues`: entries_checked=1411 / issues=0
- `python tools/release.py --check --strict`: PASS, counts in sync, JSON valid, LF endings OK
- i18n count: zh=1380 / en=1380
- bad Unicode scan: bad_files=0
- mojibake pattern scan: 0 hits
- low-register scan: 残存は公式名・固有名詞のみ
- `git diff --check`: whitespace error なし

### 既知の注意点

- Astro build は既存の duplicate id warning を出しますが、今回の語彙修正による release-blocking error ではありません。
- English 記事には、日本語の正式社名・法令名・登録簿名・出典名が意図的に残る箇所があります。

### 次の作業

- 公開後に `https://finwiki.zksc.io/` の反映と GitHub Pages deployment を確認します。
- 次回は訳語表の固定化、domain 別の用語統一、重複 id warning の整理を候補にします。
