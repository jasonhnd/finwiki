## 日本語

### 公開範囲

- 新規 `docs/`（内部開発文書、5 件）: `README.md` 索引 / `architecture.md` / `toolchain.md` / `release-process.md` / `gotchas.md`。

- `AGENTS.md`（`docs/` 非公開ルールを三語追記）、`README.md`、`CHANGELOG.md`、再生成された発見面。
- GitHub Release metadata。

### 主要変更

- **docs/ 新設（5 件）**: 仓库结构 / corpus 模型 / 双排除机制 / 新增 entry 手順（architecture）、各 tool・lib の職責と命令速査（toolchain）、リリース SOP と自検チェックリスト（release-process）、clone-mtime 汚染・self-ref count・`site/` 盲推・假死链・字面 `[[ ]]` 等の落とし穴集（gotchas）、および索引（README）。**内部開発者・agent 向けで、公開 wiki 本文とは分離。**
- **2 系統の排除に登録**: `lib/markdown_helpers.ts` の `EXCLUDED_WALK_DIRS`（`iterMarkdownFiles` → `release.ts` 計数 + `generate_ai_discovery.ts` 発見面を制御）と `tools/wiki_link_audit.ts` の `IGNORED_DIRS`（死リンク監査）。両方に `docs` を追加したため、`docs/` は corpus・`sitemap.xml` / `llms.txt` / `ai-index.json` / `api/`・Astro サイト・死リンク監査のいずれにも含まれない。`site/` の `ENTRY_DOMAIN_DIRS` は allowlist のため `docs/` は自動的に非対象（**site 設定は無変更**）。
- **AGENTS.md 追記**: `docs/` は内部開発文書・非公開・排除済み・解除禁止である旨を三語で明文化。

- 排除は 2 系統（`EXCLUDED_WALK_DIRS` と `IGNORED_DIRS`）あり、片方だけでは漏れる。両方を維持すること。

## English

### Release Scope

- New `docs/` (internal developer docs, 5 files): `README.md` index / `architecture.md` / `toolchain.md` / `release-process.md` / `gotchas.md`.
- `lib/markdown_helpers.ts` (`docs` added to `EXCLUDED_WALK_DIRS`), `tools/wiki_link_audit.ts` (`docs` added to `IGNORED_DIRS`).
- `AGENTS.md` (trilingual rule that `docs/` is unpublished), `README.md`, `CHANGELOG.md`, regenerated discovery surfaces.
- GitHub Release metadata.

### Major Changes

- **docs/ created (5 files)**: repo structure / corpus model / dual-exclusion mechanism / how to add an entry (architecture), each tool's & lib's responsibility and a command cheat-sheet (toolchain), the release SOP and a pre-flight checklist (release-process), and a gotchas set — clone-mtime pollution, self-ref count, `site/` blind-push, false dead links, literal `[[ ]]`, etc. (gotchas), plus the index (README). **For internal maintainers / agents, kept separate from the public wiki body.**
- **Registered in two exclusion lists**: `EXCLUDED_WALK_DIRS` (`lib/markdown_helpers.ts`, which `iterMarkdownFiles` drives for `release.ts` counts and `generate_ai_discovery.ts` discovery surfaces) and `IGNORED_DIRS` (`tools/wiki_link_audit.ts`, dead-link audit). Adding `docs` to both keeps `docs/` out of the corpus, `sitemap.xml` / `llms.txt` / `ai-index.json` / `api/`, the Astro site, and the audit. `site/`'s `ENTRY_DOMAIN_DIRS` is an allowlist, so `docs/` is auto-excluded (**no site config change**).
- **AGENTS.md note**: documents trilingually that `docs/` is internal, unpublished, excluded, and must not be re-included.

### Validation Results

- `bun tools/release.ts --check --strict`: EXIT=0, audit entries=1420 / issues=0 → PASS, **md=1485 unchanged (the 5 docs files are not counted)**, domains=23 / entries=1420, counts in sync, JSON / LF / duplicate-id verify OK.
- 0 `docs/` URLs in `sitemap.xml` / `llms.txt` / `ai-index.json`. 0 sensitive data.

### Known Notes

- `docs/` lives in a public GitHub repo, so even internal docs must not contain secrets / personal data (local paths and internal-flow examples are fine).
- There are two exclusion systems (`EXCLUDED_WALK_DIRS` and `IGNORED_DIRS`); one alone leaks. Keep both.

### Next Work

- After push, verify GitHub Actions (including that `docs/` does not affect the Astro build), remote HEAD, GitHub Release.
- Expanding `docs/` (CI / Vercel details, i18n translation flow, Obsidian usage) is future work.
